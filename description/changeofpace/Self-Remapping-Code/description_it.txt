Questo progetto è una proof of concept per Windows che dimostra il codice self-remapping come tecnica anti-manomissione. Crea più mapping virtuali delle stesse pagine fisiche, esegue il codice da un mapping mentre verifica l'integrità di un altro, rendendo difficile per debugger e strumenti di patching modificare il codice in esecuzione senza essere rilevati. L'implementazione in C mostra come i file mapping basati su section possano creare viste aliasate. È rivolto ai ricercatori di protezione software che studiano tecniche anti-patching e anti-debugging tramite memory aliasing.
