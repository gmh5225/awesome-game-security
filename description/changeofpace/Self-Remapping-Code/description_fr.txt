Ce projet est une preuve de concept Windows qui démontre le code auto-remappé comme technique anti-altération. Il crée plusieurs mappages virtuels des mêmes pages physiques, exécute le code depuis un mappage tout en vérifiant l'intégrité d'un autre, rendant difficile pour les débogueurs et les outils de patching de modifier le code en cours d'exécution sans être détectés. L'implémentation en C montre comment les mappages de fichiers adossés à des sections peuvent créer des vues aliasées. Il s'adresse aux chercheurs en protection logicielle qui étudient les techniques anti-patching et anti-débogage via l'aliasing mémoire.
