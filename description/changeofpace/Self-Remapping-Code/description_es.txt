Este proyecto es una prueba de concepto para Windows que demuestra el código con reasignación (self-remapping) como técnica anti-manipulación. Crea múltiples asignaciones virtuales de las mismas páginas físicas, ejecuta código desde una asignación mientras comprueba la integridad de otra, lo que dificulta que los depuradores y las herramientas de parcheo modifiquen el código en ejecución sin ser detectados. La implementación en C muestra cómo los mapeos de archivo respaldados por secciones pueden crear vistas aliasadas. Está dirigido a investigadores de protección de software que estudian técnicas anti-parcheo y anti-depuración mediante aliasing de memoria.
