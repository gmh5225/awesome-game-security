Dieses Projekt ist ein Windows-Proof-of-Concept, das selbst-remappenden Code als Anti-Tampering-Technik demonstriert. Es erzeugt mehrere virtuelle Mappings derselben physischen Seiten, führt Code aus einem Mapping aus und prüft gleichzeitig die Integrität eines anderen — wodurch Debugger und Patch-Tools den laufenden Code nur schwer unbemerkt ändern können. Die C-Implementierung zeigt, wie section-gestützte File-Mappings aliasierte Views erzeugen können. Es richtet sich an Software-Schutz-Forscher, die Anti-Patching- und Anti-Debugging-Techniken über Memory-Aliasing untersuchen.
