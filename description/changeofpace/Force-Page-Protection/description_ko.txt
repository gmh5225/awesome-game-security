이 프로젝트는 NtProtectVirtualMemory가 실패하는 경우 메모리 매핑된 뷰의 페이지 보호를 강제로 설정하는 x64dbg 플러그인입니다.
SEC_NO_CHANGE 할당 유형으로 매핑된 뷰, 호환되지 않는 초기 보호 설정을 가진 뷰를 처리하며, 원하는 보호로 뷰를 다시 매핑하여 이러한 NtProtectVirtualMemory 제한을 악용하는 안티패칭 메커니즘을 무력화합니다.
이 플러그인은 동적 분석 중 메모리 보호 제한을 재정의하기 위해 x64dbg에 ForcePageProtection(fpp) 명령을 추가합니다.
메모리 매핑된 뷰 제한을 악용하는 안티탬퍼 기법으로 보호된 메모리 영역을 패치해야 하는 리버스 엔지니어와 안티치트 분석가에게 주로 유용합니다.
