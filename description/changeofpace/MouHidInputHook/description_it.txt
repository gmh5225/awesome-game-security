Questo progetto è un driver kernel di Windows che intercetta l'oggetto CONNECT_DATA all'interno degli oggetti dispositivo MouHid per filtrare, modificare e iniettare pacchetti di input del mouse senza modificare gli stack dei dispositivi mouse HID USB.
Emula la strategia Moufiltr intercettando il callback ClassService usato per trasferire i dati del mouse alle code di dati di classe, supportando un unhooking sicuro senza scaricare gli stack dei dispositivi e le notifiche PnP per i cambiamenti di dispositivo.
La tecnica è sicura rispetto a PatchGuard e relativamente stealth, poiché evita di collegare oggetti dispositivo filtro e usa un'euristica per risolvere gli offset non documentati dei campi CONNECT_DATA.
È soprattutto utile per ricercatori anti-cheat e analisti della sicurezza degli input che studiano l'intercettazione dell'input mouse a livello kernel e il rilevamento della simulazione di input.
