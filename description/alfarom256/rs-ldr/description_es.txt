rs-ldr es una biblioteca Rust para la resolución dinámica de WinAPI basada en hashes en Windows x86_64, sin tabla de importación, cadenas de API visibles ni dependencia de kernel32 para la carga. Recorre el PEB para encontrar módulos cargados por nombre hasheado, analiza directorios de exportación (incluidas las exportaciones reenviadas) y puede cargar DLL mediante LdrLoadDll de ntdll a través de una interfaz DynApi. Las macros de ofuscación XOR de cadenas en tiempo de compilación decodifican a búferes en la pila que se ponen a cero al hacer drop, lo que ayuda a evadir el escaneo estático de cadenas y firmas. Las funciones opcionales incluyen un resolvedor SSN de syscall enchufable (con una referencia DirectResolver para técnicas al estilo Hell's Gate), un DynApi a nivel de proceso detrás de un spinlock, hashes djb2 con sal por build e intrínsecos de memoria en Rust puro para builds NODEFAULTLIB. Está orientada a herramientas Windows no_std de red team, ingeniería inversa y seguridad de juegos que necesitan una resolución de API sigilosa.
