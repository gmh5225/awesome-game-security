rs-ldr est une bibliothèque Rust pour la résolution dynamique WinAPI basée sur des hachages sous Windows x86_64, sans table d'importation, sans chaînes d'API visibles, ni dépendance à kernel32 pour le chargement. Elle parcourt le PEB pour trouver les modules chargés par nom haché, analyse les répertoires d'exportation (y compris les exportations transférées) et peut charger des DLL via LdrLoadDll de ntdll à travers une interface DynApi. Les macros d'obfuscation XOR des chaînes à la compilation décodent vers des tampons sur la pile qui s'effacent à la destruction (drop), aidant à éluder les scans de chaînes et de signatures. Les fonctionnalités optionnelles incluent un résolveur SSN de syscall enfichable (avec une référence DirectResolver pour les techniques de type Hell's Gate), un DynApi à l'échelle du processus derrière un spinlock, des hachages djb2 salés par build, et des intrinsèques mémoire purement Rust pour les builds NODEFAULTLIB. Elle cible les outils Windows no_std pour red team, reverse engineering et sécurité des jeux nécessitant une résolution d'API discrète.
