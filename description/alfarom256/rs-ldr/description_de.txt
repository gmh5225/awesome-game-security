rs-ldr ist eine Rust-Bibliothek für hashbasierte dynamische WinAPI-Auflösung unter Windows x86_64 ohne Importtabelle, sichtbare API-Zeichenketten oder kernel32-Abhängigkeit zum Laden. Sie durchläuft das PEB, um geladene Module anhand gehashter Namen zu finden, parst Exportdverzeichnisse (einschließlich weitergeleiteter Exporte) und kann DLLs über ntdll LdrLoadDll mittels einer DynApi-Schnittstelle laden. Compile-time-XOR-String-Obfuskationsmakros decodieren in Stack-Puffer, die sich beim Drop selbst nullen und so statische String- und Signaturscans erschweren. Optionale Features umfassen einen austauschbaren Syscall-SSN-Resolver (mit DirectResolver-Referenz für Hell's-Gate-ähnliche Techniken), eine prozessweite DynApi hinter einem Spinlock, buildweise gesalzene djb2-Hashes sowie reine Rust-Speicher-Intrinsics für NODEFAULTLIB-Builds. Zielgruppe sind no_std-Windows-Red-Team-, Reverse-Engineering- und Gamesicherheits-Tools, die stealthige API-Auflösung benötigen.
