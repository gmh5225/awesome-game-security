rs-ldr è una libreria Rust per la risoluzione dinamica WinAPI basata su hash su Windows x86_64, senza tabella di importazione, stringhe API visibili né dipendenza da kernel32 per il caricamento. Percorre il PEB per trovare i moduli caricati per nome hashato, analizza le directory di esportazione (incluse le esportazioni inoltrate) e può caricare DLL tramite LdrLoadDll di ntdll attraverso un'interfaccia DynApi. Le macro di offuscamento XOR delle stringhe a compile-time decodificano in buffer sullo stack che si azzerano al drop, aiutando a eludere la scansione statica di stringhe e firme. Le funzionalità opzionali includono un resolver SSN di syscall collegabile (con un riferimento DirectResolver per tecniche in stile Hell's Gate), un DynApi a livello di processo protetto da spinlock, hash djb2 con salt per build e intrinsici di memoria pure-Rust per build NODEFAULTLIB. È pensata per strumenti Windows no_std di red team, reverse engineering e game security che necessitano di una risoluzione API furtiva.
