This project is a technical educational write-up explaining how Valve Anti-Cheat (VAC) works in Counter-Strike 2, based on publicly available reverse-engineering research. It covers VAC’s usermode architecture, including the Trusted Launch boot checks in cs2.exe, the in-process scanners inside client.dll, and the external memory scanner running from steam.exe. Topics include module and PE hashing, interface CRC checks, thread inspection, vectored exception handling for hardware breakpoints, protobuf-based evidence reports to Valve servers, and server-directed diagnostics. The material is aimed at game security and anti-cheat researchers who want a forensic overview of VAC’s capabilities and limitations rather than a runtime tool or bypass.
