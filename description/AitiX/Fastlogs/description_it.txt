FastLogs è un motore open source di debug e segnalazione bug per build di gioco in cui la console del motore è difficile o impossibile da raggiungere, ad esempio WebGL, portali, mobile e console. Da una build in esecuzione, un singolo gesto o una chiamata agent può inviare le righe di log recenti, le info dispositivo, i breadcrumb di contesto, gli screenshot e gli snapshot di scena a un server Node.js self-hosted che memorizza i report in SQLite e restituisce un breve link condivisibile al viewer. Esistono client per Unity (pacchetto C# UPM) e GameMaker (GML) sotto un contratto di ingest JSON condiviso, con funzioni tra cui cattura automatica dei crash e outbox offline, un catalogo navigabile con ricerca full-text e triage, sink opzionali verso Slack, Discord, webhook e issue tracker, oltre allo scrubbing PII di default. Espone anche un percorso di invio headless e un canale di comandi remoti così che QA, sviluppatori o agenti IA possano ispezionare e guidare una build live senza debugger. Il caso d’uso principale è accorciare il ciclo da un bug in una build distribuita al momento in cui uno sviluppatore ne vede la causa in una console browser filtrabile.
