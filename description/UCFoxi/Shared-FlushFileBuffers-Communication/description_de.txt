Dieses Projekt stellt eine Kernel-zu-User-Kommunikationsmethode vor, die auf Shared Buffers und durch FlushFileBuffers ausgelöster IRP-Verarbeitung basiert. Es demonstriert das Hooking von IRP_MJ_FLUSH_BUFFERS zur Befehlsverarbeitung ohne Abhängigkeit von einem persistenten Worker-Thread. Der Code ist in C++-Kernel- und User-Mode-Komponenten aufgeteilt, die mit Visual Studio erstellt werden. Es eignet sich zur Erforschung verdeckterer Treiberkommunikationsmuster in der Spielsicherheit und bei Anti-Cheat-Bypass-Experimenten.
