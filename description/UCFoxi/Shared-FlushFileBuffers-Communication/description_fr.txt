Ce projet présente une méthode de communication noyau-utilisateur basée sur des tampons partagés et un traitement d'IRP déclenché par FlushFileBuffers. Il démontre le hooking de IRP_MJ_FLUSH_BUFFERS pour traiter des commandes sans dépendre d'un thread de travail persistant. Le code est réparti en composants C++ noyau et mode utilisateur construits avec Visual Studio. Il est utile pour étudier des schémas de communication de pilotes plus furtifs dans le cadre de la sécurité des jeux et d'expériences de contournement d'anti-cheat.
