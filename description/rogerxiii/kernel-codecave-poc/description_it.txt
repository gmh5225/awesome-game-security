Questo progetto è una proof of concept per il kernel Windows che nasconde codice eseguibile all'interno di code cave nei driver kernel legittimi. Analizza i driver caricati alla ricerca di regioni di padding inutilizzate (code cave) nelle loro sezioni .text, vi copia dello shellcode e lo esegue dal contesto di un modulo driver attendibile. In questo modo si evita di allocare nuova memoria kernel eseguibile, facilmente rilevabile dagli scan anti-cheat. È rivolto ai ricercatori di sicurezza del kernel che studiano l'abuso delle code cave e le tecniche di esecuzione furtiva di codice in modalità kernel.
