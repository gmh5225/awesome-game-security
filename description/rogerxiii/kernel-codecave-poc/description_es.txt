Este proyecto es una prueba de concepto del kernel de Windows que oculta código ejecutable dentro de code caves en controladores de kernel legítimos. Escanea los controladores cargados en busca de regiones de relleno no utilizadas (code caves) en sus secciones .text, copia shellcode en esos huecos y lo ejecuta desde el contexto de un módulo de controlador de confianza. Así se evita asignar nueva memoria ejecutable del kernel, que sería detectable por los análisis anti-cheat. Está dirigido a investigadores de seguridad del kernel que estudian el abuso de code caves y técnicas furtivas de ejecución de código en el kernel.
