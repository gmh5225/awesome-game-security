Dieses Projekt ist ein Windows-Kernel-Proof-of-Concept, das ausführbaren Code in Code Caves innerhalb legitimer Kernel-Treiber versteckt. Es durchsucht geladene Treiber nach ungenutzten Padding-Bereichen (Code Caves) in ihren .text-Abschnitten, kopiert Shellcode in diese Lücken und führt ihn im Kontext eines vertrauenswürdigen Treibermoduls aus. Dadurch wird die Allokation neuen ausführbaren Kernel-Speichers vermieden, der von Anti-Cheat-Scans erkannt werden könnte. Es richtet sich an Kernel-Sicherheitsforscher, die den Missbrauch von Code Caves und steganthafte Kernel-Codeausführungstechniken untersuchen.
