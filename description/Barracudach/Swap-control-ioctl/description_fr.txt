Ce projet est une preuve de concept pour le noyau Windows visant à intercepter la routine de dispatch IRP_MJ_DEVICE_CONTROL d'un pilote. Il démontre la redirection du pointeur de dispatch avec une approche trampoline et achemine le trafic ioctl via une logique de traitement personnalisée. L'exemple inclut des gestionnaires de requêtes de copie/allocation/protection de mémoire de processus ainsi qu'un comportement de recherche de base de module. Le cas d'usage principal est l'étude des hooks de communication des pilotes et des méthodes de détection anti-triche autour de l'interception ioctl.
