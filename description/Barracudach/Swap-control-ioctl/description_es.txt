Este proyecto es una prueba de concepto del kernel de Windows para interceptar la rutina de dispatch IRP_MJ_DEVICE_CONTROL de un controlador. Demuestra la redirección del puntero de dispatch con un enfoque trampoline y reenvía el tráfico ioctl mediante una lógica de manejo personalizada. El ejemplo incluye controladores de solicitudes de copia/asignación/protección de memoria de proceso y el comportamiento de búsqueda de la base del módulo. El caso de uso principal es estudiar los hooks de comunicación de controladores y los métodos de detección anti-trampas en torno a la interceptación de ioctl.
