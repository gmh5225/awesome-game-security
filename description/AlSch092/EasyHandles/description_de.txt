Dieses Projekt ist eine Driver-plus-DLL-Technik zum Erhalt von Prozess-Handles unter Umgehung von Kernel-Handle-Callbacks. Es ist in C und C++ implementiert und nutzt ObOpenPointerToObject im Kernelmodus, wobei ein usermodiger OpenProcess-Hook Anfragen über IOCTLs weiterleitet. So können Tools wie Debugger an callback-geschützte Prozesse anbinden, bei denen normale Handle-Erstellungspfade blockiert sind; Einschränkungen wie PPL-Ziele werden ebenfalls genannt. Es dient vor allem der Windows-Sicherheitsforschung zu Anti-Cheat- oder EDR-Handle-Schutzmechanismen.
