Este proyecto es una prueba de concepto de comunicación en el kernel de Windows que ejecuta código de kernel sin crear hilos del sistema visibles. Abusa de contextos de hilos de kernel existentes o de devoluciones de llamada de temporizador para ejecutar código personalizado, evitando así la detección de sistemas anti-trampas que enumeran hilos del sistema para hallar controladores mapeados manualmente. El controlador en C demuestra técnicas de ejecución en el kernel sin hilos. Está dirigido a investigadores de kernel que estudian la ejecución furtiva de código y la evasión de la detección de hilos anti-trampas.
