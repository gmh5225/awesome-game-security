Questo progetto è una proof of concept di comunicazione nel kernel Windows che esegue codice kernel senza creare thread di sistema visibili. Abusa di contesti di thread kernel esistenti o di callback del timer per eseguire codice personalizzato, evitando i sistemi anti-cheat che enumerano i thread di sistema per individuare driver mappati manualmente. Il driver in C dimostra tecniche di esecuzione kernel senza thread. È rivolto ai ricercatori del kernel che studiano l'esecuzione furtiva di codice e l'evasione del rilevamento dei thread anti-cheat.
