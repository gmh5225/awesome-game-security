Ce projet est une preuve de concept de communication noyau Windows qui exécute du code noyau sans créer de threads système visibles. Il abuse des contextes de threads noyau existants ou des rappels de minuteur pour exécuter du code personnalisé, contournant ainsi les systèmes anti-triche qui énumèrent les threads système pour détecter les pilotes mappés manuellement. Le pilote C démontre des techniques d'exécution noyau sans thread. Il s'adresse aux chercheurs noyau qui étudient l'exécution furtive de code et le contournement de la détection de threads anti-triche.
