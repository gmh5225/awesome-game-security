Этот проект — proof of concept внедрения в процессы Windows, который выполняет полезную нагрузку, комбинируя существующие системные ресурсы вместо создания новых.
Он реализован на C++ и сосредоточен на поиске утёкших дескрипторов процессов и потоков, обнаружении уже существующей RWX-памяти и постановке в очередь special user APC через NtQueueApcThreadEx2.
Рабочий процесс избегает распространённых «шумных» API вроде VirtualAllocEx, VirtualProtectEx и CreateRemoteThread, а также включает опциональные компоненты шифрования shellcode.
Он предназначен для исследований в области offensive security и для оценки того, как продукты EDR или античита обнаруживают пути внедрения с низким следом.
