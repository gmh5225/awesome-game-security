Este proyecto es una prueba de concepto de inyección de procesos en Windows que ejecuta cargas útiles combinando recursos del sistema existentes en lugar de crear nuevos.
Está implementado en C++ y se centra en encontrar handles de procesos e hilos filtrados, localizar memoria RWX preexistente y encolar special user APC mediante NtQueueApcThreadEx2.
El flujo de trabajo evita API ruidosas habituales como VirtualAllocEx, VirtualProtectEx y CreateRemoteThread, e incluye también componentes opcionales de cifrado de shellcode.
Está pensado para investigación de seguridad ofensiva y para evaluar cómo los productos EDR o anti-cheat detectan rutas de inyección de baja huella.
