本プロジェクトは、新しいシステムリソースを作成するのではなく、既存のリソースを組み合わせてペイロードを実行する Windows プロセスインジェクションの概念実証です。
C++ で実装されており、漏洩したプロセス／スレッドハンドルの発見、既存の RWX メモリの特定、および NtQueueApcThreadEx2 による特殊ユーザー APC のキューイングに焦点を当てています。
ワークフローは VirtualAllocEx、VirtualProtectEx、CreateRemoteThread などの一般的な高ノイズ API を避け、オプションのシェルコード暗号化コンポーネントも含みます。
攻撃的セキュリティ研究、および EDR やアンチチート製品が低フットプリントなインジェクション経路をどう検出するかを評価することを想定しています。
