Questo progetto è una proof of concept di iniezione di processi Windows che esegue payload combinando risorse di sistema esistenti invece di crearne di nuove.
È implementato in C++ e si concentra sul trovare handle di processi e thread trapelati, individuare memoria RWX preesistente e mettere in coda special user APC tramite NtQueueApcThreadEx2.
Il flusso di lavoro evita API rumorose comuni come VirtualAllocEx, VirtualProtectEx e CreateRemoteThread, e include anche componenti opzionali di crittografia dello shellcode.
È destinato alla ricerca di sicurezza offensiva e alla valutazione di come i prodotti EDR o anti-cheat rilevano percorsi di iniezione a basso impatto.
