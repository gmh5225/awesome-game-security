이 프로젝트는 새로운 시스템 리소스를 만들지 않고 기존 리소스를 조합하여 페이로드를 실행하는 Windows 프로세스 인젝션 개념 증명입니다.
C++로 구현되었으며, 유출된 프로세스 및 스레드 핸들 탐색, 기존 RWX 메모리 위치 파악, NtQueueApcThreadEx2를 통한 special user APC 큐잉에 중점을 둡니다.
워크플로는 VirtualAllocEx, VirtualProtectEx, CreateRemoteThread와 같은 일반적인 고노이즈 API를 피하며, 선택적 셸코드 암호화 구성 요소도 포함합니다.
공격적 보안 연구와 EDR 또는 안티치트 제품이 낮은 흔적의 인젝션 경로를 어떻게 탐지하는지 평가하기 위한 용도입니다.
