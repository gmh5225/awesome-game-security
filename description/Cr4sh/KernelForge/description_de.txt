Dieses Projekt ist eine Windows-Bibliothek zum Aufrufen von Kernel-Routinen aus dem Benutzermodus auf Systemen, die durch VBS und HVCI gehärtet sind.
Es ist in C++ mit einem geteilten Design umgesetzt: Eine Komponente stellt Kernel-Speicherprimitive über einen Wrapper für signierte Treiber bereit, eine andere baut höherwertige Funktionsaufruffähigkeiten auf.
Das Repository enthält Header, statische Bibliotheken, DLL-Bindungen sowie ein Beispiel, das Techniken zur DLL-Injektion vom Kernel in den Benutzermodus demonstriert.
Der primäre Anwendungsfall ist die fortgeschrittene Kernel-Sicherheitsforschung und das Prototyping von Exploits unter modernen Plattformschutzmaßnahmen.
