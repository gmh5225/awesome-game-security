Доказательство концепции, демонстрирующее технику внедрения в процесс, которая злоупотребляет Windows API форкинга процессов (RtlCreateProcessReflection), чтобы клонировать целевой процесс и перенаправить стартовый адрес форкнутой копии на внедрённый shellcode.
Она избегает традиционной инъекции через WriteProcessMemory, записывая shellcode в исходный процесс и позволяя форку унаследовать эту память, обходя распространённые эвристики обнаружения EDR.
В первую очередь полезна исследователям безопасности, изучающим новые техники внедрения кода, и инженерам античита, оценивающим методы обхода на основе форка.
