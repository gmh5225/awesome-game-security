Una prueba de concepto que demuestra una técnica de inyección de procesos que abusa de la API de forking de procesos de Windows (RtlCreateProcessReflection) para clonar un proceso objetivo y redirigir la dirección de inicio de la copia bifurcada hacia shellcode inyectado.
Evita la inyección tradicional basada en WriteProcessMemory escribiendo el shellcode en el proceso original y dejando que el fork herede esa memoria, eludiendo así heurísticas comunes de detección de EDR.
Resulta especialmente útil para investigadores de seguridad que estudian técnicas novedosas de inyección de código y para ingenieros de anti-cheat que evalúan métodos de evasión basados en fork.
