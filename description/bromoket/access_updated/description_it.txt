Questo progetto è un fork aggiornato di btbd/access che fornisce operazioni sui processi in modalità kernel senza handle, agganciando il puntatore xKdEnumerateDebuggingDevices per la comunicazione kernel–usermode.
Sostituisce gli offset hardcoded con la ricerca dinamica di pattern basata su Zydis per scoprire le funzioni del kernel a runtime, supportando Windows 10 (1607+) fino a Windows 11 (24H2) in un unico binario.
Il driver kernel esegue operazioni PROCESS_ALL_ACCESS senza creare handle reali, usando un hook pulito della sezione .data senza patch inline.
È soprattutto utile per i ricercatori di sicurezza del kernel che studiano l’accesso ai processi senza handle, l’hooking delle syscall e le tecniche del kernel Windows indipendenti dalla versione.
