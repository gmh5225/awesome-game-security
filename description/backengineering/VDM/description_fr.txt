Ce projet est VDM (Voyager Driver Manager), une bibliothèque C++ destinée à exploiter des pilotes signés vulnérables afin d'obtenir une lecture/écriture arbitraire de la mémoire physique et une exécution de code en mode noyau sous Windows. Elle fournit une API claire au-dessus de plusieurs backends de pilotes vulnérables (gdrv, cpuz, etc.) pour lire/écrire la mémoire physique, traduire les adresses virtuelles et exécuter du shellcode en mode noyau. La bibliothèque est conçue comme un backend modulaire pour des outils tels que kdmapper, msrexec et bluepill. Elle s'adresse aux chercheurs en exploitation noyau qui étudient les primitives d'attaque BYOVD et construisent des outils de niveau noyau.
