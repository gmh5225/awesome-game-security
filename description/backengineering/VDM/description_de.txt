Dieses Projekt ist VDM (Voyager Driver Manager), eine C++-Bibliothek zum Ausnutzen anfälliger signierter Treiber, um unter Windows beliebigen physischen Speicherzugriff (Lesen/Schreiben) und Kernel-Codeausführung zu erlangen. Sie stellt eine saubere API über mehrere anfällige Treiber-Backends (gdrv, cpuz usw.) bereit, um physischen Speicher zu lesen/schreiben, virtuelle Adressen zu übersetzen und Shellcode im Kernelmodus auszuführen. Die Bibliothek ist als modulares Backend für Tools wie kdmapper, msrexec und bluepill konzipiert. Sie richtet sich an Kernel-Exploitation-Forscher, die BYOVD-Angriffsprimitiven untersuchen und Tools auf Kernel-Ebene entwickeln.
