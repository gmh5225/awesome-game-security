Концептуальный инжектор shellcode, который использует Windows Debugging API и общую память (отображение файлов), чтобы внедрять полезную нагрузку без вызовов WriteProcessMemory, VirtualAllocEx или ReadProcessMemory.
Он подключается как отладчик, ставит аппаратные точки останова и манипулирует регистрами контекста потока, чтобы выстроить цепочку вызовов API внутри целевого процесса для отображения и выполнения shellcode.
В основном полезен исследователям безопасности, изучающим продвинутые техники инъекции, которые обходят обнаружение EDR, избегая традиционных API манипуляции памятью.
