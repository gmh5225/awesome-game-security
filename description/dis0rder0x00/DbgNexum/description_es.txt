Inyector de shellcode de prueba de concepto que usa la Windows Debugging API y memoria compartida (mapeo de archivos) para inyectar cargas útiles sin llamar a WriteProcessMemory, VirtualAllocEx ni ReadProcessMemory.
Se adjunta como depurador, establece puntos de interrupción de hardware y manipula los registros del contexto del hilo para orquestar una cadena de llamadas a API dentro del proceso objetivo que mapean y ejecutan el shellcode.
Es útil principalmente para investigadores de seguridad que estudian técnicas avanzadas de inyección que evaden la detección de EDR al evitar las API tradicionales de manipulación de memoria.
