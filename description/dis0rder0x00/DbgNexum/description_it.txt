Iniettore di shellcode proof-of-concept che usa la Windows Debugging API e la memoria condivisa (file mapping) per iniettare payload senza chiamare WriteProcessMemory, VirtualAllocEx o ReadProcessMemory.
Si collega come debugger, imposta hardware breakpoint e manipola i registri del contesto del thread per orchestrare una catena di chiamate API nel processo di destinazione che mappano ed eseguono lo shellcode.
È soprattutto utile per i ricercatori di sicurezza che studiano tecniche avanzate di injection che eludono il rilevamento EDR evitando le API tradizionali di manipolazione della memoria.
