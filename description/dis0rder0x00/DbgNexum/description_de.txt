Ein Proof-of-Concept-Shellcode-Injector, der die Windows Debugging API und Shared Memory (File Mapping) nutzt, um Payloads zu injizieren, ohne WriteProcessMemory, VirtualAllocEx oder ReadProcessMemory aufzurufen.
Er hängt sich als Debugger an, setzt Hardware-Breakpoints und manipuliert Thread-Kontextregister, um im Zielprozess eine Kette von API-Aufrufen zu orchestrieren, die den Shellcode mappen und ausführen.
Er ist vor allem für Sicherheitsforscher nützlich, die fortgeschrittene Injection-Techniken untersuchen, die die EDR-Erkennung umgehen, indem sie traditionelle Speicher-Manipulations-APIs vermeiden.
