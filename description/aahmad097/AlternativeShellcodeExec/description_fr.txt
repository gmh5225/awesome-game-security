Alternative Code Execution est une collection de projets de preuve de concept Windows pour exécuter du shellcode via des API de callback plutôt que CreateThread.
Le dépôt est implémenté en C++ et organisé en de nombreux exemples Visual Studio, chacun démontrant une primitive d'exécution basée sur les callbacks différente.
Il se concentre sur l'exécution de shellcode indépendante de la position et sur des schémas d'abus d'API qui peuvent réduire la dépendance aux chemins d'injection courants.
Le cas d'usage principal est la recherche en sécurité offensive et en sécurité des jeux sur les techniques d'évasion en mode utilisateur et de contournement de détection.
