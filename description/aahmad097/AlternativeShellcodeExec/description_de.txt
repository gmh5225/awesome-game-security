Alternative Code Execution ist eine Sammlung von Windows-Proof-of-Concept-Projekten zum Ausführen von Shellcode über Callback-APIs statt über CreateThread.
Das Repository ist in C++ umgesetzt und als zahlreiche Visual-Studio-Beispiele organisiert; jedes demonstriert eine andere callback-basierte Ausführungsprimitive.
Es konzentriert sich auf positionsunabhängige Shellcode-Ausführung und API-Missbrauchsmuster, die die Abhängigkeit von gängigen Injektionspfaden verringern können.
Der Hauptanwendungsfall ist Offensive-Security- und Game-Security-Forschung zu User-Mode-Evasion und Detection-Bypass-Techniken.
