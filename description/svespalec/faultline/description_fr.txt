Faultline est une preuve de concept d'anti-cheat Windows qui détecte le code exécuté depuis une mémoire mappée manuellement ou autrement non enregistrée, en surveillant les défauts de page du working set et en inspectant les threads suspects. Écrit en C++23 moderne et construit avec CMake, il utilise les API de surveillance du working set (InitializeProcessForWsWatch, GetWsChangesEx et QueryWorkingSet) pour trouver les FaultingPc ou les pages exécutables résidentes situées hors des modules listés dans le PEB. Lors d'une détection, il suspend le thread fautif et parcourt sa pile d'appels pour capturer la chaîne d'exécution. Le projet comprend un processus hôte, un injecteur de mapping manuel (thread distant ou détournement via SetThreadContext) et une charge utile de test pour démontrer la détection d'injections de type shellcode et de mapping manuel. Il s'adresse aux chercheurs en sécurité des jeux et anti-cheat qui étudient la détection en mode utilisateur des triches basées sur la mémoire et des techniques d'injection.
