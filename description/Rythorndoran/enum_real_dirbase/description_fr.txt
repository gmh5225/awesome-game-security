Ce projet est une preuve de concept de pilote en mode noyau Windows pour énumérer les véritables valeurs de base de répertoire de processus (CR3) à partir des structures de mémoire physique.
Il est écrit en C++ pour le Windows Driver Kit et implémente des helpers de pagination bas niveau, un balayage de motifs noyau et une traversée de la base de données PFN.
Le code initialise des bases de tables de pages auto-référencées, résout MmPfnDatabase à l'exécution et parcourt des plages physiques pour récupérer les données de contexte de processus.
Il est surtout utile pour la recherche anti-triche noyau, les expériences de forensique mémoire et l'étude du suivi des espaces d'adressage masqués ou protégés.
