Questo progetto è una dimostrazione di hooking del kernel Windows che modifica i mapping delle page table invece di patchare direttamente il codice eseguibile. Il suo driver C++ costruisce e modifica le strutture di paging per reindirizzare l'esecuzione ai gestori di hook, preservando l'accesso alle routine originali. L'esempio include l'intercettazione intorno a percorsi di sistema come NtCreateFile ed è impostato per evitare gli approcci tipici che attivano PatchGuard. È destinato alla ricerca avanzata sulla sicurezza del kernel e allo studio delle tecniche di bypass degli anti-cheat.
