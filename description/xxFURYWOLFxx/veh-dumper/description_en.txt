veh-dumper is an injectable x64 DLL that enumerates every registered VectoredExceptionHandler and VectoredContinueHandler in a target process and dumps each one as a standalone synthetic PE that IDA can open directly. For each handler it surgically extracts the handler function bytes (via RtlLookupFunctionEntry unwind info or a heuristic scan), RIP-relative data references, direct callees through a depth-capped BFS, and indirect-call IAT slots with their resolved code. System-module callees under Windows are not dumped; instead their export names are emitted so IDA shows readable imports. The VEH list is walked by registering a foothold handler and dynamically discovering the encoded handler pointer offset with RtlDecodePointer, targeting reverse engineers analyzing VEH-based protection or anti-cheat logic on Windows.
