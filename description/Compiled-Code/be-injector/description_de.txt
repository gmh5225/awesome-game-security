Dieses Projekt ist ein Windows-Injection-Proof-of-Concept, das signierten Modulcode über physisches Memory-Mapping patcht, um Copy-on-Write-Artefakte zu vermeiden.
Es ist in C++ implementiert und demonstriert eine Technik, die gängige Integrity-Annahmen von Anti-Cheat-Systemen zu geladenen Modulen umgehen soll.
Das Design behandelt das Umgehen von Prüfungen wie Thread-Monitoring, API-Call-Scrutiny und signaturbasierten Scans, indem Seiten geändert werden, bevor das normale Modul-Mapping-Verhalten divergiert.
Der Anwendungsfall ist Low-Level-Anti-Cheat-Bypass-Forschung und Experimente zur Erkennungsresistenz.
