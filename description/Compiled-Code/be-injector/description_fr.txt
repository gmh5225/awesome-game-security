Ce projet est une preuve de concept d'injection Windows qui corrige le code de modules signés via le mappage de mémoire physique afin d'éviter les artefacts de copy-on-write.
Il est implémenté en C++ et démontre une technique visant à contourner les hypothèses courantes d'intégrité des anti-cheats sur les modules chargés.
La conception aborde le contournement de contrôles tels que la surveillance des threads, l'examen des appels d'API et les scans basés sur des signatures, en modifiant les pages avant que le comportement normal de mappage des modules ne diverge.
Son cas d'usage est la recherche de contournement anti-cheat bas niveau et l'expérimentation de résilience à la détection.
