Questo progetto è una proof of concept di injection su Windows che patcha il codice di moduli firmati tramite mappatura della memoria fisica per evitare artefatti di copy-on-write.
È implementato in C++ e dimostra una tecnica pensata per eludere le comuni ipotesi di integrità degli anti-cheat sui moduli caricati.
Il design discute come aggirare controlli come il monitoraggio dei thread, l'esame delle chiamate API e le scansioni basate su signature, modificando le pagine prima che il normale comportamento di mapping dei moduli diverga.
Il caso d'uso è la ricerca di bypass anti-cheat a basso livello e la sperimentazione sulla resilienza al rilevamento.
