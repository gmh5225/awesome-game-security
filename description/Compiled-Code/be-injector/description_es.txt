Este proyecto es una prueba de concepto de inyección en Windows que parchea el código de módulos firmados mediante el mapeo de memoria física para evitar artefactos de copy-on-write.
Está implementado en C++ y demuestra una técnica pensada para evadir las suposiciones habituales de integridad de los anti-cheat sobre los módulos cargados.
El diseño analiza cómo eludir comprobaciones como la monitorización de hilos, el escrutinio de llamadas a API y los escaneos basados en firmas, modificando páginas antes de que el comportamiento normal de mapeo de módulos diverja.
Su caso de uso es la investigación de bypass anti-cheat a bajo nivel y la experimentación de resiliencia frente a la detección.
