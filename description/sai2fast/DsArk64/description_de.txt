Dieses Projekt konzentriert sich auf bYOVD: DsArk64.sys (Qihoo 360) — WHQL-signiert, Prozessbeendigung aus Ring 0 sowie Kernel-R/W. Laden Sie einen beliebigen 360-Installer von 360.cn herunter (kein Konto, kein CAPTCHA), starten Sie ihn angehalten (CREATE_SUSPENDED) und injizieren Sie Shellcode per CreateRemoteThread; der Shellcode ruft CreateFileW("\\.\DsArk") auf. Der Treiber prüft den Imagenamen von IoGetCurrentProcess(), erkennt das Qihoo-Zertifikat und erlaubt das Öffnen. DuplicateHandle gibt das Handle an den Angreiferprozess zurück, TerminateProcess beendet den Spenderprozess. Vor allem nützlich für Game-Security-Forscher und Reverse Engineers, die offensive Techniken im Cheat-/Vulnerable-Driver-Bereich untersuchen.
