Este proyecto es una prueba de concepto de un bypass de BattlEye en modo de usuario que demuestra una ruta de carga anteriormente vulnerable. Incluye un implantador en C++ y un flujo de trabajo DLL de ejemplo que engancha CreateFileW y manipula las comprobaciones de archivos para hacer pasar una carga útil por un módulo de confianza. El código está organizado como proyectos de Visual Studio con ejemplos de uso para inyectar en un proceso de juego objetivo y gestionar callbacks de hooks exportados. Su uso principal es la investigación de vulnerabilidades de anti-trampas y el estudio histórico de debilidades en la validación de confianza en modo de usuario.
