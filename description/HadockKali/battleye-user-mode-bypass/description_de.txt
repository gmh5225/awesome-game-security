Dieses Projekt ist ein Proof-of-Concept für einen User-Mode-BattlEye-Bypass und demonstriert einen zuvor anfälligen Ladepfad. Es enthält einen C++-Implanter und einen Beispiel-DLL-Workflow, der CreateFileW hookt und Dateiprüfungen manipuliert, um eine Payload als vertrauenswürdiges Modul auszugeben. Der Code ist als Visual-Studio-Projekte strukturiert und enthält Beispiele zur Injektion in einen Zielspielprozess sowie zur Behandlung exportierter Hook-Callbacks. Der Hauptzweck ist die Forschung zu Anti-Cheat-Schwachstellen und die historische Analyse von Schwächen der User-Mode-Vertrauensvalidierung.
