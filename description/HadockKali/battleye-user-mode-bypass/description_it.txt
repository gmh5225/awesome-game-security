Questo progetto è una proof-of-concept di bypass BattlEye in modalità utente che dimostra un percorso di caricamento precedentemente vulnerabile. Include un implanter in C++ e un flusso di lavoro DLL di esempio che esegue l’hook di CreateFileW e manipola i controlli sui file per far passare un payload come modulo attendibile. Il codice è strutturato come progetti Visual Studio con esempi di utilizzo per l’iniezione in un processo di gioco di destinazione e la gestione dei callback degli hook esportati. L’uso principale è la ricerca di vulnerabilità anti-cheat e lo studio storico delle debolezze nella convalida della fiducia in modalità utente.
