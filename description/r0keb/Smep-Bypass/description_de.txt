Dieses Projekt ist ein Proof of Concept zum Umgehen von SMEP (Supervisor Mode Execution Prevention) unter Windows. SMEP verhindert, dass der Kernel Code in User-Mode-Seiten ausführt. Dieser Exploit demonstriert Techniken, um SMEP durch Manipulation des CR4-Registers über ROP-Chains oder angreifbare Treiber-Primitives zu deaktivieren und so die Ausführung von User-Space-Shellcode im Kernel-Mode zu ermöglichen. Er richtet sich an Kernel-Exploitation-Forscher, die Bypass-Techniken für Hardware-Sicherheitsfunktionen untersuchen.
