Ce projet est une preuve de concept pour contourner le SMEP (Supervisor Mode Execution Prevention) sous Windows. Le SMEP empêche le noyau d'exécuter du code situé dans des pages en mode utilisateur. Cet exploit présente des techniques pour désactiver le SMEP en manipulant le registre CR4 via des chaînes ROP ou des primitives de pilotes vulnérables, permettant ainsi l'exécution en mode noyau de shellcode en espace utilisateur. Il s'adresse aux chercheurs en exploitation noyau qui étudient les contournements des mécanismes de sécurité matériels.
