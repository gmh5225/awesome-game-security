Este proyecto es una prueba de concepto para eludir SMEP (Supervisor Mode Execution Prevention) en Windows. SMEP impide que el kernel ejecute código en páginas de modo usuario. Este exploit demuestra técnicas para desactivar SMEP manipulando el registro CR4 mediante cadenas ROP o primitivas de controladores vulnerables, lo que permite la ejecución en modo kernel de shellcode del espacio de usuario. Está dirigido a investigadores de explotación del kernel que estudian técnicas de elusión de funciones de seguridad por hardware.
