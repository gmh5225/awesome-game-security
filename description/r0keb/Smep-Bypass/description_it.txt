Questo progetto è una proof of concept per aggirare SMEP (Supervisor Mode Execution Prevention) su Windows. SMEP impedisce al kernel di eseguire codice nelle pagine in modalità utente. Questo exploit dimostra tecniche per disabilitare SMEP manipolando il registro CR4 tramite catene ROP o primitive di driver vulnerabili, consentendo l'esecuzione in modalità kernel di shellcode nello spazio utente. È rivolto ai ricercatori di kernel exploitation che studiano le tecniche di bypass delle funzionalità di sicurezza hardware.
