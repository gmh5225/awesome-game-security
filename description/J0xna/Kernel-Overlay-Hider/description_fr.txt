Ce projet est une preuve de concept qui masque le handle d'une fenêtre overlay lors de l'énumération des fenêtres en manipulant les structures de fenêtres côté noyau.
Il utilise un pilote noyau Windows ainsi que des programmes de test en mode utilisateur pour localiser et modifier des pointeurs liés à win32k et des données associées à TAGWND.
Le code se concentre sur des techniques de type DKOM et inclut des exemples pour déclencher ce comportement depuis un contexte d'overlay DirectX.
Il est destiné à la recherche bas niveau sur les internals Windows autour de la visibilité des overlays et des mécanismes d'évasion anti-cheat.
