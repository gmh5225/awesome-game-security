Questo progetto è un driver del kernel Windows per intercettare e dumpare il traffico IOCTL di altri driver di dispositivo. Registra metadati delle richieste come codice IOCTL, tipo di percorso di trasporto e dimensioni dei buffer, e può memorizzare il contenuto dei buffer di input deduplicando le combinazioni ripetute. Il repository include componenti lato driver e lato client per selezionare i dispositivi di destinazione e raccogliere i log in un layout di output definito. È usato principalmente per il reverse engineering di driver proprietari, comprese le interfacce anti-cheat e altri percorsi di comunicazione del kernel sensibili alla sicurezza.
