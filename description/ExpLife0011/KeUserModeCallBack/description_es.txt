Este proyecto es una demo del kernel de Windows que invoca rutinas en modo usuario mediante KeUserModeCallback. El controlador crea una interfaz de dispositivo, recibe solicitudes IOCTL y prepara argumentos de callback y stubs de shellcode para rutas de 32 y 64 bits. Recorre estructuras de proceso como el PEB y las tablas de exportación de módulos para resolver user32 y MessageBoxA antes de despachar el callback. Está pensado sobre todo para investigar la transición de kernel a usuario y comprender primitivas de ejecución de código basadas en callbacks.
