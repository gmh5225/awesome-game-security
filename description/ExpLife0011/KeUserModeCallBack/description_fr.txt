Ce projet est une démonstration noyau Windows qui invoque des routines en mode utilisateur via KeUserModeCallback. Le pilote crée une interface de périphérique, reçoit des requêtes IOCTL et prépare les arguments de callback ainsi que des stubs de shellcode pour les chemins 32 bits et 64 bits. Il parcourt des structures de processus telles que le PEB et les tables d'export de modules pour résoudre user32 et MessageBoxA avant de dispatcher le callback. Il sert principalement à la recherche sur les transitions noyau-utilisateur et à la compréhension des primitives d'exécution de code basées sur les callbacks.
