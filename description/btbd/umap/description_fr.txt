Ce projet est umap, un mappeur minimaliste de pilotes noyau Windows écrit en C, qui mappe manuellement un pilote non signé en mémoire noyau. Il s'appuie sur un pilote signé vulnérable pour accéder à la mémoire physique afin d'allouer de l'espace dans le pool noyau, copier les sections du pilote, traiter les relocalisations, résoudre les imports et appeler le point d'entrée du pilote mappé entièrement depuis le mode utilisateur. Le mappeur évite de laisser des traces dans le registre et ne passe pas par les chemins de chargement de pilotes standards. Il s'adresse aux chercheurs noyau qui étudient les techniques de mapping furtif de pilotes et leurs vecteurs de détection.
