이 프로젝트는 Windows WPP(Windows Software Trace Preprocessor) 추적 인프라를 하이재킹하여 드라이버 DeviceControl 호출을 가로채는 개념 증명입니다.
disk.sys, mountmgr.sys 같은 드라이버의 WPP 추적 함수 포인터와 제어 플래그를 수정한 뒤, 반환 주소 검사와 스택 워킹 또는 레지스터 캡처를 통한 IRP 포인터 추출로 DeviceControl 호출을 식별합니다.
이 커널 드라이버는 시스템 드라이버 .data 섹션의 WPP 포인터를 DeviceControl 가로채기에 전용할 수 있음을 보여줍니다.
주로 비전통적인 드라이버 후킹 기법과 디스크 시리얼 가로채기를 통한 HWID 스푸핑을 연구하는 커널 보안 연구자에게 유용합니다.
