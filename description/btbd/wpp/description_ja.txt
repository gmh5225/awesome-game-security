本プロジェクトは、Windows WPP（Windows Software Trace Preprocessor）のトレーシング基盤をハイジャックすることで、ドライバの DeviceControl 呼び出しを傍受する概念実証です。
disk.sys や mountmgr.sys などのドライバ内の WPP トレース関数ポインタと制御フラグを改変し、戻りアドレスの検査と、スタックウォーキングまたはレジスタ取得による IRP ポインタ抽出によって DeviceControl 呼び出しを識別します。
このカーネルドライバは、システムドライバの .data セクションにある WPP ポインタを DeviceControl 傍受に転用できることを示します。
主に、非定型のドライバフッキング手法や、ディスクシリアル傍受による HWID スプーフィングを研究するカーネルセキュリティ研究者向けです。
