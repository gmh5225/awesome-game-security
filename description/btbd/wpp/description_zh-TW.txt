此專案是一個概念驗證，透過劫持 Windows WPP（Windows Software Trace Preprocessor）追蹤基礎架構來攔截驅動程式的 DeviceControl 呼叫。
它會修改 disk.sys、mountmgr.sys 等驅動程式中的 WPP 追蹤函式指標與控制旗標，再透過返回位址檢查以及基於堆疊回溯或暫存器擷取的 IRP 指標提取，識別 DeviceControl 呼叫。
該核心驅動程式展示如何將系統驅動程式 .data 區段中的 WPP 指標重新用於 DeviceControl 攔截。
它主要適用於研究非常規驅動程式掛鉤技術，以及透過磁碟序號攔截進行 HWID 偽裝的核心安全性研究人員。
