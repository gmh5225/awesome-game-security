이 프로젝트는 Windows에서 디스크 장치(HBA 컨트롤러)를 사용해 DMA 작업을 수행함으로써 하이퍼바이저 수준 메모리 보호를 우회하는 개념 증명입니다.
가상화되지 않은 장치 접근을 허용하는 Hyper-V와 같은 네이티브 하이퍼바이저가 ATA 디스크 장치 DMA를 통해 SLAT(Second Level Address Translation)를 우회할 수 있음을 보여줍니다.
커널 드라이버는 베어 메탈에서 런타임에 Hyper-V를 수정하는 시연이 이루어졌으나, 현재는 ATA만 지원하며 HBA의 64비트 주소 지정 능력에 의해 제한될 수 있습니다.
디스크 기반 DMA 공격과 SLAT 우회 기법을 연구하는 커널 및 하이퍼바이저 보안 연구자에게 주로 유용합니다.
