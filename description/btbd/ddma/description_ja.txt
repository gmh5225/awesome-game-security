本プロジェクトは、Windows 上でディスクデバイス（HBA コントローラ）を用いて DMA を実行し、ハイパーバイザレベルのメモリ保護を回避する概念実証です。
デバイスへの非仮想化アクセスを許す Hyper-V のようなネイティブハイパーバイザが、ATA ディスクデバイスの DMA を通じて SLAT（Second Level Address Translation）を迂回できることを示します。
カーネルドライバはベアメタル上で実行時に Hyper-V を改変するデモが行われましたが、現状は ATA のみ対応で、HBA の 64 ビットアドレッシング能力によって制限される場合があります。
ディスクベースの DMA 攻撃や SLAT バイパス手法を研究するカーネル／ハイパーバイザセキュリティ研究者向けです。
