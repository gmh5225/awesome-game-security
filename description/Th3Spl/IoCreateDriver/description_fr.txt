Ce projet partage une implémentation personnalisée du comportement de IoCreateDriver pour l'expérimentation en mode noyau Windows.
Il met en avant des techniques destinées à éviter les chemins de visibilité classiques du chargement de pilotes, y compris le contournement des points de journalisation courants.
Le code est écrit en C/C++ pour les workflows Visual Studio et WDK, avec des notes sur les configurations de manual mapping et les ajustements du point d'entrée.
Il s'adresse principalement à la recherche sur les internals Windows bas niveau et le contournement d'anti-cheat.
