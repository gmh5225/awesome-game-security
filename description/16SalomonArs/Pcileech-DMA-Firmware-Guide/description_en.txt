This project is a hands-on guide to building custom PCILeech DMA FPGA firmware that properly emulates a real PCIe donor device, rather than only changing VID and DID. It walks through Windows-first donor capture, PCIe core and BAR setup, shadow configuration space, write-mask protection, capability structures, and TLP response work using Vivado, SystemVerilog, Python COE generation, and tools such as Arbor and TeleScan PE. The guide covers common Artix-7 DMA boards (Squirrel, CaptainDMA, LeetDMA, Enigma, ZDMA, and related platforms), board-specific flashing, cold-boot validation, and recovery when firmware fails to enumerate. It also includes advanced options such as Zero4K BAR handling, Vivado ILA debugging, and optional Linux capture paths. The primary audience is researchers and hardware learners doing controlled-lab DMA and PCIe firmware work relevant to game security and reverse engineering tooling.
