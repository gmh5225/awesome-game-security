DroidShield è un SDK Android open source che rileva minacce runtime comuni come root, debugger, framework di hooking, emulatori e manomissione o ripacchettizzazione di APK. È realizzato principalmente in Kotlin, con un layer nativo C++ su JNI per controlli più difficili da aggirare, come l’anti-debug ptrace e la scansione di firme Frida o Xposed nelle mappe di processo. La libreria usa un’architettura modulare pulita con un contratto di dominio Kotlin puro, controlli sui dati Android, un motore di rilevamento delle minacce e una facade SDK pubblica cablata con Dagger 2. Un plugin Gradle può iniettare e ordinare in modo polimorfico varianti di controllo per build, così che i bypass su una build non si trasferiscano facilmente a un’altra. Mira a casi d’uso di sicurezza per app e giochi Android in cui gli integratori vogliono segnali anti-cheat e anti-tamper lato client, senza un backend ospitato.
