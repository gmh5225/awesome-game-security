DroidShield est un SDK Android open source qui détecte les menaces d’exécution courantes telles que le root, les débogueurs, les frameworks de hooking, les émulateurs, ainsi que la falsification ou le repackaging d’APK. Il est principalement écrit en Kotlin, avec une couche native C++ via JNI pour des contrôles plus difficiles à contourner, comme l’anti-debug ptrace et le scan de signatures Frida ou Xposed dans les maps de processus. La bibliothèque repose sur une architecture modulaire claire : un contrat de domaine Kotlin pur, des contrôles de données Android, un moteur de détection de menaces et une façade SDK publique câblée avec Dagger 2. Un plugin Gradle peut injecter et ordonner de façon polymorphe des variantes de contrôles par build, afin qu’un contournement contre un build ne se reporte pas facilement à un autre. Il vise les cas d’usage de sécurité des applications et jeux Android où les intégrateurs veulent des signaux anti-triche et anti-falsification côté client, sans backend hébergé.
