CMDT is a compact Windows utility that launches processes under a TrustedInstaller service token so they can access resources protected by TrustedInstaller ACLs without taking ownership. It is written entirely in hand-coded native x86 and x64 assembly as dual-mode GUI and CLI binaries with no C runtime or third-party dependencies, kept under roughly 40 KiB and 30 KiB respectively. Key capabilities include duplicating the TrustedInstaller process token, enabling a large set of token privileges, UAC self-elevation, Explorer context-menu installation, CLI I/O redirection relay, optional MRU history, and related system-maintenance helpers. It is aimed at administrators, forensics and reverse-engineering workflows, and deep Windows repair or debugging where SYSTEM alone is not enough to modify protected OS components.
