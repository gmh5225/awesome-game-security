本專案是一個 UEFI EFI 應用程式，在 ExitBootServices 時於記憶體中修補 Windows SYSTEM 登錄檔 hive，搶在 NT 核心接管之前完成。
示範透過設定 SetupType 與 CmdLine，使下次啟動時以 NT AUTHORITY\SYSTEM 身分啟動 cmd.exe，且不寫入磁碟、不載入核心驅動程式。
文件涵蓋 ExitBootServices 掛鉤、regf hive cell 修補、反分析字串混淆，以及面向 Windows 11 研究的 BitLocker 或 PCR 邊界影響。
主要面向遊戲安全研究人員與逆向工程師，用於學習作弊 / EFI 驅動程式領域相關的攻擊性技術。
