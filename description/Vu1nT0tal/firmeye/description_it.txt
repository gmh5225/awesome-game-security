Questo progetto è un plugin di IDA per la ricerca di vulnerabilità nel firmware IoT, basato sul tracciamento degli argomenti verso funzioni sensibili. È scritto principalmente in Python e combina controlli statici con analisi dinamica assistita dal debugger per ridurre lo sforzo di triage manuale. Le regole documentate mirano a problemi come buffer overflow, rischi di esecuzione di comandi e problemi di format string, con supporto all'automazione tramite workflow batch da riga di comando. Il caso d'uso principale è l'audit di sicurezza del firmware e la ricerca di reverse engineering su target embedded.
