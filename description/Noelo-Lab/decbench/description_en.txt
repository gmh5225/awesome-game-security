DecBench is a Python benchmarking suite for evaluating how well decompilers recover source from compiled C binaries. It runs a three-stage pipeline—compile projects at multiple optimization levels, decompile with pluggable backends, then score the output—supporting traditional tools such as angr, Ghidra, IDA, Binary Ninja, r2dec, and dewolf, plus optional LLM or coding-agent backends. Evaluation uses three core metrics: CFG graph-edit distance for structural correctness, type matching against DWARF ground truth, and recompilation byte-match for assembly similarity. Benchmark corpora include Debian-style packages, cross-compiled embedded and RTOS firmware, and carefully sandboxed malware samples. It is aimed at reverse engineers, binary-analysis researchers, and anyone comparing or improving decompiler quality.
