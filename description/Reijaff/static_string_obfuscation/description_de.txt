Dieses Projekt demonstriert kompilierzeitliche statische String-Obfuskation mit Zig.
Es erzeugt während des Builds zufällige Schlüssel und wendet eine XOR-basierte Transformationslogik an, sodass Klartext-Strings nicht direkt in Binärdateien gespeichert werden.
Die Build-Pipeline zielt auf gestrippte x86_64-Windows-Executables ab und legt Wert auf leichte Laufzeit-Entschlüsselung.
Es eignet sich für Experimente zur Reverse-Engineering-Resistenz und zur Anti-Analyse-Härtung in sicherheitsorientierter Software.
