Dieses Projekt ist PCM (Process Context Monitor), ein Werkzeug oder eine Bibliothek zur Low-Level-Überwachung von Prozess-Kontextwechseln, Thread-Scheduling und CPU-Auslastung. Es erfasst Scheduling-Ereignisse pro Kern sowie Kontextwechseldaten und liefert damit eine feingranulare Sicht auf Ausführungs­muster von Prozessen und Threads im System. Es richtet sich an Performance-Analysten und Sicherheitsforscher, die Scheduling-Verhalten von Prozessen und den CPU-Ressourcenverbrauch untersuchen.
