Драйвер ядра Windows, демонстрирующий, как немаскируемые прерывания (NMI) можно использовать для обхода стека: структура MACHINE_FRAME находится при возврате ISR через iretq, что позволяет определить указатель прерванной инструкции.
Он регистрирует обработчик NMI-колбэка, который захватывает прерванный RIP на всех процессорах, что даёт возможность обнаруживать код, выполняющийся из подозрительных областей памяти.
В первую очередь полезен разработчикам античитов и исследователям безопасности ядра, изучающим обход стека потоков на основе NMI как метод обнаружения читов.
