Ce projet est une preuve de concept Windows compacte qui illustre les cas limites des vérifications d'alignement avec les callbacks d'instrumentation de processus. Il combine C et assembleur x64 pour définir des drapeaux liés à AC, déclencher un accès mémoire non aligné et observer les exceptions STATUS_DATATYPE_MISALIGNMENT qui en résultent. L'exemple utilise NtSetInformationProcess avec ProcessInstrumentationCallback et une gestion d'exceptions personnalisée pour mettre en évidence les points où le code de callback peut échouer. Il est surtout utile pour la recherche sur l'instrumentation noyau et runtime de bas niveau.
