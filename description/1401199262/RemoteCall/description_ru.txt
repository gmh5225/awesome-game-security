Этот проект — техника ядра Windows, выполняющая код в пользовательском режиме в произвольном целевом процессе.
Она связывает APC ядра с KeUserModeCallback, переключает выполнение через I/O-процедуру драйвера и возвращается с контролируемым восстановлением контекста.
Реализация на C++ избегает выделения RWX-памяти для шеллкода в целевом процессе, сохраняя при этом возможность вызываемого выполнения в пользовательском режиме.
Основной сценарий использования — исследование продвинутой инъекции процессов и оценка компромиссов обнаружения в средах игровой безопасности.
