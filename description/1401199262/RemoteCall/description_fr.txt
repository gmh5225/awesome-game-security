Ce projet est une technique du noyau Windows qui exécute du code en mode utilisateur dans un processus cible arbitraire.
Il enchaîne une APC noyau avec KeUserModeCallback, bascule l'exécution via une routine d'E/S du pilote, puis revient avec une restauration contrôlée du contexte.
L'implémentation C++ évite d'allouer de la mémoire shellcode RWX dans le processus cible tout en permettant une exécution appelable en mode utilisateur.
Son cas d'usage principal est la recherche avancée d'injection de processus et l'évaluation des compromis de détection dans les environnements de sécurité des jeux.
