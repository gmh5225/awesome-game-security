Dieses Projekt ist ein Windows-Kernel-Proof-of-Concept zum Hooken von Ausführungspfaden im Zusammenhang mit Kontextwechseln.
Es nutzt einen ETW/CKCL-basierten Hook-Pfad und benutzerdefinierte Stack-Frame-Prüfungen, um während ausgewählter Scheduling-Abläufe einen Handler aufzurufen.
Der Code ist in C++ geschrieben und enthält Low-Level-Module für Trace-Steuerung, syscall-seitige Hooks und Kernel-Utility-Routinen.
Der Hauptanwendungsfall ist das Experimentieren mit Techniken zum Abfangen der Thread-Planung für die Kernel-Sicherheitsforschung.
