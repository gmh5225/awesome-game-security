Questo progetto è una proof of concept per il kernel Windows volta a effettuare l'hooking di percorsi di esecuzione legati al context switch.
Usa un percorso di hook basato su ETW/CKCL e controlli personalizzati degli stack frame per invocare un handler durante un flusso di scheduling selezionato.
Il codice è scritto in C++ e include moduli a basso livello per il controllo del tracing, hook orientati alle syscall e routine di utilità del kernel.
Il caso d'uso principale è sperimentare tecniche di intercettazione dello scheduling dei thread per la ricerca sulla sicurezza del kernel.
