Ce projet est un injecteur de DLL en mode noyau qui utilise kdmapper avec un pilote Intel vulnérable pour charger un pilote noyau personnalisé, puis effectue un mappage manuel de DLL dans les processus cibles depuis l’espace noyau. La base de code C++ couvre le mappage des sections PE, la résolution des imports, la gestion des callbacks TLS, ainsi que la communication utilisateur-noyau via des opérations IOCTL. Il est surtout utile aux chercheurs en sécurité des jeux qui étudient l’injection de DLL assistée par le noyau, les techniques de mappage manuel et l’exploitation de pilotes vulnérables pour la manipulation de processus.
