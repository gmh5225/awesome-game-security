Dieses Projekt ist eine Sammlung von Proof-of-Concept-Programmen zum Auslösen bekannter Android-Kernel-Schwachstellen. Es ist überwiegend in C geschrieben und enthält viele CVE-spezifische Beispiele, darunter Fälle zu Hersteller-Treibern und Wi-Fi-Komponenten. Der Code zeigt Low-Level-Techniken wie präparierte ioctl-Aufrufe, Race Conditions und speicherbezogene Manipulationen gegenüber Treibern. Es richtet sich vor allem an Kernel-Sicherheitsforscher, die Bugs reproduzieren, Mitigations prüfen und Android-Exploit-Primitive studieren möchten.
