Este proyecto es una colección de programas de prueba de concepto para activar vulnerabilidades conocidas del kernel de Android. Está escrito principalmente en C e incluye numerosos ejemplos específicos de CVE, también casos ligados a drivers de fabricantes y componentes Wi-Fi. El código muestra técnicas de bajo nivel como llamadas ioctl manipuladas, condiciones de carrera y manipulación de memoria orientada a drivers. Resulta útil sobre todo para investigadores de seguridad del kernel que quieran reproducir fallos, validar mitigaciones y estudiar primitivas de explotación en Android.
