Questo progetto è un driver kernel non firmato affiancato a un esempio in user-mode che espone un ampio insieme di funzioni di manipolazione della memoria e di simulazione degli input tramite un hook di comunicazione personalizzato.
Il README archiviato elenca ricerca dei processi, scoperta della base dei moduli, allocazione e modifiche di protezione della memoria virtuale, primitive di lettura e scrittura, simulazione degli input, pattern scanning e persino mapping manuale di DLL, mentre il wrapper di esempio invia richieste `KALDERETA_MEMORY` attraverso un percorso di chiamata Win32 agganciato invece di un'interfaccia dispositivo pubblica convenzionale.
La sua struttura rende il repository utile sia come scheletro di cheat-driver ricco di funzionalità sia come riferimento per abbinare operazioni di memoria kernel a una libreria helper user-mode di livello superiore.
È principalmente utile per ricercatori di kernel Windows e reverse engineering che studiano hook di comunicazione dei driver, primitive di tooling della memoria e flussi di mapping manuale integrati user/kernel.
