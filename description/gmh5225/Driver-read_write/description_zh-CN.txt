本项目是一个手动映射的内核读写驱动，通过劫持 `Beep.sys` 的设备控制处理来暴露内存拷贝与模块基址查询能力。
其控制路径会替换 `\Driver\Beep` 上的 `IRP_MJ_DEVICE_CONTROL`，校验用户请求，附加到目标进程以进行直接内存访问，并复用被挂钩的驱动对象，使映射载荷能够伪装成更合法的内核模块。
仓库还包含针对 `PiDDBCacheTable` 与 `MmUnloadedDrivers` 的清理逻辑，因此既可作为通信示例，也可作为基于漏洞驱动映射工作流中减少痕迹的参考。
它主要适用于研究 IRP 劫持、基本进程内存原语，以及未签名驱动加载器中映射后伪迹清理的 Windows 内核研究者。
