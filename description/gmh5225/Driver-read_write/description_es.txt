Este proyecto es un controlador de lectura/escritura en modo kernel mapeado manualmente que secuestra el manejo de control de dispositivo de `Beep.sys` para exponer copia de memoria y consultas de base de módulo.
Su ruta de control reemplaza `IRP_MJ_DEVICE_CONTROL` en `\Driver\Beep`, valida las solicitudes del usuario, se adjunta a procesos objetivo para acceso directo a memoria y reutiliza el objeto del controlador enganchado para que la carga mapeada pueda hacerse pasar por un módulo del kernel más legítimo.
El repositorio también incluye lógica de limpieza para `PiDDBCacheTable` y `MmUnloadedDrivers`, por lo que sirve tanto como ejemplo de comunicación como referencia de reducción de rastros en flujos de mapeo basados en controladores vulnerables.
Resulta especialmente útil para investigadores del kernel de Windows que estudian el secuestro de IRP, primitivas básicas de memoria de proceso y la limpieza de artefactos tras el mapeo en cargadores de controladores sin firmar.
