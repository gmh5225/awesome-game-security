Ce projet est un pilote noyau de lecture/écriture mappé manuellement qui détourne la gestion du contrôle de périphérique de `Beep.sys` pour exposer la copie mémoire et les requêtes de base de module.
Son chemin de contrôle remplace `IRP_MJ_DEVICE_CONTROL` sur `\Driver\Beep`, valide les requêtes utilisateur, s’attache aux processus cibles pour un accès mémoire direct, et réutilise l’objet pilote accroché afin que la charge mappée puisse se faire passer pour un module noyau plus légitime.
Le dépôt inclut aussi une logique de nettoyage pour `PiDDBCacheTable` et `MmUnloadedDrivers`, ce qui en fait à la fois un exemple de communication et une référence de réduction de traces pour les flux de mapping basés sur des pilotes vulnérables.
Il est surtout utile aux chercheurs du noyau Windows qui étudient le détournement d’IRP, les primitives de mémoire de processus de base, et le nettoyage des artefacts après mapping dans les chargeurs de pilotes non signés.
