이 프로젝트는 수동으로 매핑된 커널 읽기/쓰기 드라이버로, `Beep.sys`의 장치 제어 처리를 하이재킹하여 메모리 복사와 모듈 베이스 조회를 노출합니다.
제어 경로는 `\Driver\Beep`의 `IRP_MJ_DEVICE_CONTROL`을 교체하고, 사용자 요청을 검증하며, 대상 프로세스에 부착해 직접 메모리 접근을 수행하고, 훅된 드라이버 객체를 재사용해 매핑된 페이로드가 더 정상적인 커널 모듈처럼 보이게 합니다.
저장소에는 `PiDDBCacheTable`과 `MmUnloadedDrivers` 정리 로직도 포함되어 있어, 통신 예제이자 취약한 드라이버 기반 매핑 워크플로에서 흔적을 줄이는 참고 자료로도 쓰입니다.
주로 IRP 하이재킹, 기본 프로세스 메모리 프리미티브, 그리고 서명되지 않은 드라이버 로더에서 매핑 후 아티팩트 정리를 연구하는 Windows 커널 연구자에게 유용합니다.
