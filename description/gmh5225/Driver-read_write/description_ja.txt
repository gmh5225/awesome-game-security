本プロジェクトは、手動マップされたカーネル読み書きドライバであり、`Beep.sys` のデバイス制御処理をハイジャックして、メモリコピーとモジュールベースアドレスの照会を公開します。
制御パスは `\Driver\Beep` 上の `IRP_MJ_DEVICE_CONTROL` を差し替え、ユーザー要求を検証し、ターゲットプロセスにアタッチして直接メモリアクセスを行い、フックされたドライバオブジェクトを再利用することで、マップされたペイロードをより正当なカーネルモジュールのように見せかけます。
リポジトリには `PiDDBCacheTable` と `MmUnloadedDrivers` のクリーンアップロジックも含まれており、通信の実装例であると同時に、脆弱なドライバを用いたマッピングワークフローにおける痕跡低減の参考にもなります。
主に、IRP ハイジャック、基本的なプロセスメモリプリミティブ、および未署名ドライバローダにおけるマップ後アーティファクトのクリーンアップを研究する Windows カーネル研究者向けです。
