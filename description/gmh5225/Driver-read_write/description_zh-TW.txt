本專案是一個手動對應的核心讀寫驅動程式，透過劫持 `Beep.sys` 的裝置控制處理來暴露記憶體複製與模組基底位址查詢能力。
其控制路徑會替換 `\Driver\Beep` 上的 `IRP_MJ_DEVICE_CONTROL`，驗證使用者請求，附加到目標行程以進行直接記憶體存取，並重用被掛鉤的驅動物件，使對應載荷能夠偽裝成更合法的核心模組。
儲存庫還包含針對 `PiDDBCacheTable` 與 `MmUnloadedDrivers` 的清理邏輯，因此既可作為通訊範例，也可作為基於漏洞驅動對應工作流程中減少痕跡的參考。
它主要適用於研究 IRP 劫持、基本行程記憶體原語，以及未簽署驅動載入器中對應後痕跡清理的 Windows 核心研究者。
