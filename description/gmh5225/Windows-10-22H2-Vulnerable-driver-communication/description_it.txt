Questo progetto è un wrapper in user-mode intorno a asromgdrv.sys che mostra come usare un driver vulnerabile ancora caricabile su Windows 10 22H2 e Windows 11 per servizi lato kernel. Il README archiviato evidenzia l’allocazione di memoria kernel contigua e la lettura-scrittura dei registri di controllo come principali primitive esposte, e il livello di comunicazione apre \\.\AsrOmgDrv e guida direttamente gli IOCTL del vendor tramite DeviceIoControl. Il file communication.cpp implementa helper per allocare e liberare memoria contigua e per leggere e scrivere i registri di controllo, quindi il repository è essenzialmente un’interfaccia reverse concisa per un driver firmato vulnerabile. È soprattutto utile per i ricercatori del kernel Windows che studiano la comunicazione BYOVD, l’abuso dei registri di controllo e come piccoli wrapper IOCTL reverse possano diventare la base di tooling kernel più avanzato.
