本プロジェクトは asromgdrv.sys をラップするユーザーモード層であり、Windows 10 22H2 および Windows 11 上でまだ読み込み可能な脆弱なドライバを使い、カーネル側サービスを提供する手法を示します。アーカイブされた README では、主な公開プリミティブとして連続カーネルメモリの割り当てと制御レジスタの読み書きが挙げられており、通信層は \\.\AsrOmgDrv を開き、DeviceIoControl 経由でベンダー IOCTL を直接駆動します。communication.cpp には連続メモリの確保・解放、および制御レジスタの読み書き用ヘルパーが実装されており、このリポジトリは実質的に脆弱な署名済みドライバ向けの簡潔なリバースドインターフェースです。BYOVD 通信、制御レジスタの悪用、そして小さなリバースド IOCTL ラッパーが高度なカーネルツーリングの基盤になり得るかを学ぶ Windows カーネル研究者向けに特に有用です。
