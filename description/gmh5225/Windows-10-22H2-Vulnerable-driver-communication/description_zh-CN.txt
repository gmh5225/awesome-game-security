本项目是面向 asromgdrv.sys 的用户态封装，演示如何在 Windows 10 22H2 与 Windows 11 上利用仍可加载的漏洞驱动提供内核侧服务。归档 README 指出，主要暴露的原语是连续内核内存分配以及控制寄存器读写；通信层打开 \\.\AsrOmgDrv，并通过 DeviceIoControl 直接驱动厂商 IOCTL。其 communication.cpp 实现了分配与释放连续内存、以及读写控制寄存器的辅助函数，因此该仓库本质上是对一个已签名漏洞驱动的简洁逆向接口。它主要供 Windows 内核研究者学习 BYOVD 通信、控制寄存器滥用，以及小型逆向 IOCTL 封装如何成为更高级内核工具的基础。
