이 프로젝트는 asromgdrv.sys를 감싸는 사용자 모드 래퍼로, Windows 10 22H2와 Windows 11에서 여전히 로드 가능한 취약한 드라이버를 사용해 커널 측 서비스를 제공하는 방법을 보여 줍니다. 보관된 README는 연속 커널 메모리 할당과 제어 레지스터 읽기/쓰기를 주요 노출 프리미티브로 강조하며, 통신 계층은 \\.\AsrOmgDrv를 열고 DeviceIoControl을 통해 벤더 IOCTL을 직접 구동합니다. communication.cpp는 연속 메모리 할당·해제와 제어 레지스터 읽기/쓰기 헬퍼를 구현하므로, 이 저장소는 본질적으로 취약한 서명된 드라이버에 대한 간결한 리버스 인터페이스입니다. BYOVD 통신, 제어 레지스터 남용, 그리고 작은 리버스 IOCTL 래퍼가 더 고급 커널 도구의 기반이 될 수 있는지를 연구하는 Windows 커널 연구자에게 특히 유용합니다.
