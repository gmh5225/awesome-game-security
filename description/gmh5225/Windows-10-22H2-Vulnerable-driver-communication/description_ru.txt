Этот проект — user-mode обёртка вокруг asromgdrv.sys, демонстрирующая использование по-прежнему загружаемого уязвимого драйвера в Windows 10 22H2 и Windows 11 для сервисов на стороне ядра. В архивном README основными открытыми примитивами названы выделение непрерывной памяти ядра и чтение-запись управляющих регистров; слой связи открывает \\.\AsrOmgDrv и напрямую вызывает vendor IOCTL через DeviceIoControl. В communication.cpp реализованы вспомогательные функции для выделения и освобождения непрерывной памяти, а также чтения и записи управляющих регистров, поэтому репозиторий по сути представляет собой краткий reverse-интерфейс к уязвимому подписанному драйверу. Он в первую очередь полезен исследователям ядра Windows, изучающим BYOVD-коммуникацию, злоупотребление управляющими регистрами и то, как небольшие reverse-обёртки IOCTL могут стать основой более продвинутых kernel-инструментов.
