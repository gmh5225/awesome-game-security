Este proyecto es un wrapper en modo usuario alrededor de asromgdrv.sys que demuestra cómo usar un controlador vulnerable aún cargable en Windows 10 22H2 y Windows 11 para servicios del lado del kernel. El README archivado destaca la asignación de memoria contigua en el kernel y la lectura-escritura de registros de control como las principales primitivas expuestas, y la capa de comunicación abre \\.\AsrOmgDrv y opera directamente los IOCTL del proveedor mediante DeviceIoControl. Su communication.cpp implementa helpers para asignar y liberar memoria contigua, además de leer y escribir registros de control, de modo que el repositorio es esencialmente una interfaz invertida concisa para un controlador firmado vulnerable. Resulta especialmente útil para investigadores del kernel de Windows que estudian la comunicación BYOVD, el abuso de registros de control y cómo pequeños wrappers IOCTL invertidos pueden convertirse en la base de herramientas de kernel más avanzadas.
