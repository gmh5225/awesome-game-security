Dieses Projekt ist ein Usermode-Wrapper um asromgdrv.sys und zeigt, wie ein weiterhin ladbarer verwundbarer Treiber unter Windows 10 22H2 und Windows 11 für Dienste auf der Kernel-Seite genutzt werden kann. Die archivierte README hebt die Allokation zusammenhängenden Kernel-Speichers sowie das Lesen und Schreiben von Steuerregistern als zentrale exponierte Primitive hervor; die Kommunikationsschicht öffnet \\.\AsrOmgDrv und steuert die Hersteller-IOCTLs direkt über DeviceIoControl. In communication.cpp sind Hilfsfunktionen zum Allokieren und Freigeben zusammenhängenden Speichers sowie zum Lesen und Schreiben von Steuerregistern umgesetzt — das Repository ist im Wesentlichen eine knappe reverse-engineerte Schnittstelle für einen signierten verwundbaren Treiber. Es ist vor allem für Windows-Kernel-Forscher nützlich, die BYOVD-Kommunikation, Missbrauch von Steuerregistern und den Aufbau fortgeschrittener Kernel-Werkzeuge auf kleinen reverse-engineerten IOCTL-Wrappern untersuchen.
