Dieses Projekt demonstriert Code-Injection über Windows-NLS-Registrierungsschlüssel (National Language Support). Es verändert NLS-bezogene Registrierungseinträge, um das Laden von NLS-Codepage-Übersetzungs-DLLs umzuleiten, sodass benutzerdefinierte DLLs in Prozesse geladen werden, die das NLS-Subsystem initialisieren. Diese Persistenztechnik lädt Code früh in der Prozessinitialisierung. Sie richtet sich an Red-Team-Forscher, die NLS-basierte Code-Injection- und Persistenztechniken untersuchen.
