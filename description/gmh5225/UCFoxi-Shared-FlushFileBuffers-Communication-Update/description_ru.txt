Этот проект — пример связи ядра и пользовательского режима, который переиспользует IRP_MJ_FLUSH_BUFFERS на \Driver\PEAUTH как триггер, а реальные данные запроса хранит в общем пользовательском буфере.
Его точка входа драйвера разрешает целевой объект драйвера, подменяет MajorFunction[IRP_MJ_FLUSH_BUFFERS] через InterlockedExchangePointer и заново загружает указатель общего буфера и PID клиента из значений реестра по пути \Registry\Machine\SOFTWARE\ucflash.
Хук копирует блок REQUEST_DATA с помощью MmCopyVirtualMemory, ротирует magic-значение для синхронизации сессии и диспетчеризует операции read, write, protect, alloc, free, module и main-base через вспомогательные колбэки на основе кода доступа к физической памяти.
Он в основном полезен исследователям ядра Windows, сравнивающим альтернативные каналы связи драйверов на основе перехваченных путей IRP, общих буферов, инициализируемых из реестра, и лёгких сервисов работы с памятью в ядре.
