本プロジェクトは、\Driver\PEAUTH 上の IRP_MJ_FLUSH_BUFFERS をトリガーとして再利用し、実際のリクエストデータは共有ユーザーバッファに置く、カーネルとユーザーモード間の通信サンプルです。
ドライバーエントリは対象ドライバーオブジェクトを解決し、InterlockedExchangePointer で MajorFunction[IRP_MJ_FLUSH_BUFFERS] を差し替え、\Registry\Machine\SOFTWARE\ucflash 配下のレジストリ値から共有バッファポインタとクライアント PID を再読み込みします。
フックは MmCopyVirtualMemory で REQUEST_DATA ブロックをコピーし、セッション同期のためマジック値をローテートし、物理メモリアクセスコードを支えるヘルパーコールバック経由で read、write、protect、alloc、free、module、main-base 操作をディスパッチします。
ハイジャックされた IRP パス、レジストリで初期化される共有バッファ、軽量なカーネルメモリサービスに基づく代替ドライバー通信チャネルを比較する Windows カーネル研究者に主に役立ちます。
