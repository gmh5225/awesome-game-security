Ce projet est un exemple de communication noyau/utilisateur qui réutilise IRP_MJ_FLUSH_BUFFERS sur \Driver\PEAUTH comme déclencheur, tandis que les données de requête réelles résident dans un tampon utilisateur partagé.
Son entry point de pilote résout l'objet pilote cible, remplace MajorFunction[IRP_MJ_FLUSH_BUFFERS] avec InterlockedExchangePointer, et recharge le pointeur du tampon partagé ainsi que le PID client depuis les valeurs de registre sous \Registry\Machine\SOFTWARE\ucflash.
Le hook copie un bloc REQUEST_DATA avec MmCopyVirtualMemory, fait tourner une valeur magic pour maintenir la session synchronisée, et dispatche les opérations read, write, protect, alloc, free, module et main-base via des callbacks d'aide s'appuyant sur du code d'accès à la mémoire physique.
Il est surtout utile aux chercheurs du noyau Windows qui comparent des canaux de communication de pilotes alternatifs basés sur des chemins IRP détournés, des tampons partagés initialisés via le registre, et des services mémoire noyau légers.
