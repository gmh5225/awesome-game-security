本项目是一个教学示例，演示如何通过向目标进程中可运行的线程排队特殊内核 APC，从内核读取另一进程的内存。
代码保存 KeInitializeApc 与 KeInsertQueueApc，查找仍允许 APC 投递的线程，分配非分页暂存缓冲区，由 APC 回调将目标地址内容 memcpy 到内核内存，再把结果复制回用户空间。
归档的测试程序将该 APC 路径与普通 ReadProcessMemory 进行对比，README 明确将此次内存读取定位为特殊 APC 插入机制的演示，而非仓库的唯一目的。
它主要适合希望获得特殊 APC 插入、线程选择约束以及内核介导内存采集具体参考的 Windows 内核研究者。
