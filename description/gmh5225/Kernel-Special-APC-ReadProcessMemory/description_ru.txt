Этот проект — учебный пример чтения памяти другого процесса из ядра путём постановки специального kernel APC в очередь исполняемого потока целевого процесса.
Код сохраняет KeInitializeApc и KeInsertQueueApc, ищет поток, у которого доставка APC ещё включена, выделяет невыгружаемый промежуточный буфер и даёт callback APC выполнить memcpy целевого адреса в память ядра, после чего копирует результат обратно в пользовательское пространство.
Архивная тестовая программа сравнивает этот путь APC с обычным ReadProcessMemory, а README явно представляет чтение памяти как демонстрацию механики вставки special APC, а не как единственную цель репозитория.
Он в основном полезен исследователям ядра Windows, которым нужна конкретная справка по вставке special APC, ограничениям выбора потока и сбору памяти через ядро.
