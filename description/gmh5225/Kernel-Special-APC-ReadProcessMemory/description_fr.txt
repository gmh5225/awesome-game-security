Ce projet est un exemple pédagogique pour lire un autre processus depuis le noyau en mettant en file d'attente un APC noyau spécial dans un thread exécutable du processus cible.
Le code conserve KeInitializeApc et KeInsertQueueApc, recherche un thread dont la livraison d'APC est encore activée, alloue un tampon de staging non paginé, et laisse le rappel APC effectuer un memcpy de l'adresse cible vers la mémoire noyau avant de recopier le résultat vers l'espace utilisateur.
Le programme de test archivé compare ce chemin APC à ReadProcessMemory ordinaire, et le README présente explicitement cette lecture mémoire comme une démonstration des mécanismes d'insertion d'APC spéciaux plutôt que comme le seul objet du dépôt.
Il est surtout utile aux chercheurs du noyau Windows qui veulent une référence concrète pour l'insertion d'APC spéciaux, les contraintes de sélection de thread et la collecte mémoire médiée par le noyau.
