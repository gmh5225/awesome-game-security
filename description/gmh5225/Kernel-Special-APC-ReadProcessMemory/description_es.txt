Este proyecto es un ejemplo didáctico para leer otro proceso desde el kernel encolando un APC especial de kernel en un hilo ejecutable del proceso objetivo.
El código guarda KeInitializeApc y KeInsertQueueApc, busca un hilo cuya entrega de APC siga habilitada, asigna un búfer de staging no paginado y deja que el callback del APC haga memcpy de la dirección objetivo a memoria de kernel antes de copiar el resultado de vuelta al espacio de usuario.
El programa de prueba archivado compara esta ruta APC con ReadProcessMemory ordinario, y el README enmarca explícitamente la lectura de memoria como una demostración de la mecánica de inserción de APC especiales, no como el único propósito del repositorio.
Resulta especialmente útil para investigadores del kernel de Windows que buscan una referencia concreta sobre inserción de APC especiales, restricciones de selección de hilos y recolección de memoria mediada por el kernel.
