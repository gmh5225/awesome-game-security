本專案是一個教學範例，示範如何透過向目標行程中可執行的執行緒佇列特殊核心 APC，從核心讀取另一行程的記憶體。
程式碼保存 KeInitializeApc 與 KeInsertQueueApc，搜尋仍允許 APC 傳遞的執行緒，配置非分頁暫存緩衝區，由 APC 回呼將目標位址內容 memcpy 到核心記憶體，再把結果複製回使用者空間。
封存的測試程式將此 APC 路徑與一般 ReadProcessMemory 進行比較，README 明確將此次記憶體讀取定位為特殊 APC 插入機制的示範，而非儲存庫的唯一目的。
它主要適合希望取得特殊 APC 插入、執行緒選擇限制以及核心仲介記憶體收集具體參考的 Windows 核心研究者。
