이 프로젝트는 대상 프로세스의 실행 가능한 스레드에 특수 커널 APC를 큐잉하여 커널에서 다른 프로세스의 메모리를 읽는 방법을 보여주는 교육용 예제입니다.
코드는 KeInitializeApc와 KeInsertQueueApc를 저장하고, APC 전달이 아직 활성화된 스레드를 찾은 뒤 비페이지드 스테이징 버퍼를 할당하며, APC 콜백이 대상 주소를 커널 메모리로 memcpy한 다음 결과를 사용자 공간으로 다시 복사합니다.
보관된 테스트 프로그램은 이 APC 경로를 일반 ReadProcessMemory와 비교하며, README는 메모리 읽기를 저장소의 유일한 목적이 아니라 특수 APC 삽입 메커니즘을 보여 주기 위한 데모로 명확히 설명합니다.
특수 APC 삽입, 스레드 선택 제약, 커널 매개 메모리 수집에 대한 구체적인 참고 자료를 원하는 Windows 커널 연구자에게 주로 유용합니다.
