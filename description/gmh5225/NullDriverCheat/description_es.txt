Este proyecto es una adaptación a Windows 11 del patrón de driver-cheat de Null que engancha NtOpenCompositionSurfaceSectionInfo dentro de dxgkrnl para crear una vía encubierta de comunicación con el kernel.
Su instalador de hooks escribe un pequeño stub de salto en el export, y el manejador acepta solicitudes estructuradas para búsqueda de base de módulo, operaciones de lectura y escritura de memoria de proceso, y ayudas opcionales de dibujo GDI tomadas de exports de win32k.
El README archivado lo presenta explícitamente como una reimplementación modificada, con un pool tag distinto y la elección de syscall pensada para reducir la detección inmediata frente a enfoques más simples basados en dispositivos.
Resulta principalmente útil para ingenieros inversos que estudian hooks de exports de dxgkrnl, mensajería estructurada usuario-kernel y ayudas de memoria u overlays asistidos por GDI en drivers de cheat.
