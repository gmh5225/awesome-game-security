本项目是 Null 驱动作弊模式在 Windows 11 上的适配版本，通过挂钩 dxgkrnl 中的 NtOpenCompositionSurfaceSectionInfo 建立隐蔽的内核通信通道。
其挂钩安装程序会向该导出写入一小段跳转桩，处理程序则接受结构化请求，用于模块基址查找、进程内存读写，以及可选的、源自 win32k 导出的 GDI 绘图辅助功能。
归档的 README 明确将其定位为经过修改的再实现：使用不同的池标签，并有意选择该系统调用，以期比更简单的基于设备的方法更不易被立即检测。
它主要适用于逆向工程师研究 dxgkrnl 导出挂钩、结构化用户态–内核态消息传递，以及作弊驱动中依赖 GDI 的覆盖层或内存辅助技术。
