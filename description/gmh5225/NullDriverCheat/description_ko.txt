이 프로젝트는 Null의 드라이버 치트 패턴을 Windows 11에 맞게 적응시킨 것으로, dxgkrnl 내부의 NtOpenCompositionSurfaceSectionInfo를 후킹하여 은밀한 커널 통신 경로를 만듭니다.
후크 설치기는 해당 익스포트에 작은 점프 스텁을 쓰고, 핸들러는 모듈 베이스 조회, 프로세스 메모리 읽기/쓰기, 그리고 win32k 익스포트에서 가져온 선택적 GDI 그리기 도우미를 위한 구조화된 요청을 받습니다.
보관된 README는 이를 다른 풀 태그를 쓰고, 단순한 디바이스 기반 방식보다 즉시 탐지되기 어렵게 하려는 의으로 해당 시스템 콜을 선택한 수정된 재구현으로 분명히 설명합니다.
주로 dxgkrnl 익스포트 후킹, 구조화된 사용자-커널 메시징, 그리고 치트 드라이버에서의 GDI 보조 오버레이나 메모리 헬퍼를 연구하는 리버스 엔지니어에게 유용합니다.
