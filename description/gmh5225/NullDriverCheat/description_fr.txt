Ce projet est une adaptation pour Windows 11 du schéma de driver-cheat de Null qui accroche NtOpenCompositionSurfaceSectionInfo dans dxgkrnl afin de créer un canal de communication noyau discret.
Son installateur de hook écrit un petit stub de saut dans l'export, et le gestionnaire accepte des requêtes structurées pour la recherche de base de module, les opérations de lecture et d'écriture en mémoire de processus, ainsi que des aides de dessin GDI optionnelles issues des exports win32k.
Le README archivé le présente explicitement comme une réimplémentation modifiée, avec un pool tag différent et un choix de syscall visant à réduire la détection immédiate par rapport aux approches plus simples basées sur un périphérique.
Il est surtout utile aux reverse engineers qui étudient les hooks d'exports dxgkrnl, la messagerie structurée user-kernel, et les aides mémoire ou overlays assistés par GDI dans les drivers de cheat.
