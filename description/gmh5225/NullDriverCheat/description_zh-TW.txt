本專案是 Null 驅動作弊模式在 Windows 11 上的適配版本，透過掛鉤 dxgkrnl 中的 NtOpenCompositionSurfaceSectionInfo 建立隱蔽的核心通訊通道。
其掛鉤安裝程式會向該匯出寫入一小段跳轉樁，處理程式則接受結構化請求，用於模組基底位址查詢、處理序記憶體讀寫，以及可選的、源自 win32k 匯出的 GDI 繪圖輔助功能。
歸檔的 README 明確將其定位為經過修改的再實作：使用不同的集區標籤，並有意選擇該系統呼叫，以期比更簡單的基於裝置的方法更不易被立即偵測。
它主要適用於逆向工程師研究 dxgkrnl 匯出掛鉤、結構化使用者態–核心態訊息傳遞，以及作弊驅動中依賴 GDI 的覆蓋層或記憶體輔助技術。
