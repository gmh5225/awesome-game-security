이 프로젝트는 일반 독립 드라이버 이미지 경로를 노출하지 않도록 다른 드라이버를 통해 사용자 지정 shellcode를 실행하는 데 초점을 둔 커널 드라이버 개념 증명입니다.
보관된 README에 따르면 서명 관련 제약을 우회하기 위해 다른 드라이버에 shellcode를 주입하도록 설계되었으며, 소스 트리에는 일반적인 장치 인터페이스만이 아니라 전용 shellcode 파일, 인라인 hook 헬퍼, 패치 모듈이 포함되어 있습니다.
구현은 NTFS 드라이버 핸들러와 같은 기존 디스패치 루틴을 패치하고, 쓰기 보호를 일시적으로 비활성화하여 인라인 점프를 설치하며, 원래 경로를 계속 호출하거나 언로드 시 복원할 수 있도록 trampoline을 보존합니다.
주로 Windows 커널 연구자를 대상으로 하며, 드라이버 디스패치 하이재킹, 기존 드라이버 내부의 shellcode 기반 실행, 사용자 지정 커널 페이로드 가시성을 줄이는 기법을 연구하는 데 유용합니다.
