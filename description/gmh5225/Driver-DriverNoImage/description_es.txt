Este proyecto es una prueba de concepto de controlador de kernel centrada en ejecutar shellcode personalizado a través de otros controladores para evitar exponer una ruta de imagen de controlador independiente habitual.
El README archivado indica que se diseñó para inyectar shellcode en otro controlador y eludir restricciones relacionadas con la firma; el árbol de código incluye archivos de shellcode dedicados, helpers de hook inline y módulos de parche, en lugar de solo una interfaz de dispositivo convencional.
Su implementación parchea rutinas de dispatch existentes, como los handlers del controlador NTFS, desactiva temporalmente la protección de escritura para instalar saltos inline y conserva trampolines para que la ruta original pueda seguir llamándose o restaurarse al descargar.
Resulta especialmente útil para investigadores del kernel de Windows que estudian el secuestro del dispatch de controladores, la ejecución basada en shellcode dentro de controladores existentes y técnicas para reducir la visibilidad de cargas útiles de kernel personalizadas.
