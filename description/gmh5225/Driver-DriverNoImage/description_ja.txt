本プロジェクトは、通常のスタンドアロンなドライバイメージパスを露出させないよう、他のドライバ経由でカスタム shellcode を実行することを中心にしたカーネルドライバの概念実証です。
アーカイブされた README によれば、署名関連の制約を回避するために別ドライバへ shellcode を注入する設計であり、ソースツリーには従来のデバイスインターフェイスだけでなく、専用の shellcode ファイル、インライン hook ヘルパー、パッチモジュールが含まれています。
実装では NTFS ドライバのハンドラなど既存のディスパッチルーチンをパッチし、書き込み保護を一時的に無効化してインラインジャンプを仕込み、元の経路を呼び出し可能、またはアンロード時に復元できるよう trampoline を保持します。
主に Windows カーネル研究者向けで、ドライバディスパッチのハイジャック、既存ドライバ内での shellcode 実行、カスタムカーネルペイロードの可視性を下げる手法の研究に役立ちます。
