Ce projet est une preuve de concept de pilote noyau centrée sur l’exécution de shellcode personnalisé via d’autres pilotes, afin d’éviter d’exposer un chemin d’image de pilote autonome classique.
Le README archivé indique qu’il a été conçu pour injecter du shellcode dans un autre pilote afin de contourner des contraintes liées à la signature ; l’arborescence source comprend des fichiers shellcode dédiés, des helpers de hook inline et des modules de patch, plutôt qu’une simple interface périphérique conventionnelle.
Son implémentation patch les routines de dispatch existantes, comme les handlers du pilote NTFS, désactive temporairement la protection en écriture pour installer des sauts inline, et conserve des trampolines afin que le chemin d’origine puisse encore être appelé ou restauré au déchargement.
Il est surtout utile aux chercheurs du noyau Windows qui étudient le détournement du dispatch de pilotes, l’exécution basée sur du shellcode dans des pilotes existants, et les techniques visant à réduire la visibilité des charges utiles noyau personnalisées.
