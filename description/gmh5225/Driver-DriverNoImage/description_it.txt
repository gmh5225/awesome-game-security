Questo progetto è una proof of concept di driver kernel costruita intorno all’esecuzione di shellcode personalizzato tramite altri driver, così da evitare di esporre un normale percorso di immagine di driver autonomo.
Il README archiviato afferma che è stato progettato per iniettare shellcode in un altro driver e aggirare vincoli legati alla firma; l’albero dei sorgenti include file shellcode dedicati, helper per hook inline e moduli di patch, piuttosto che solo un’interfaccia dispositivo convenzionale.
L’implementazione applica patch a routine di dispatch esistenti, come gli handler del driver NTFS, disabilita temporaneamente la protezione in scrittura per installare jump inline e conserva trampoline in modo che il percorso originale possa ancora essere chiamato o ripristinato allo scaricamento.
È soprattutto utile per i ricercatori del kernel Windows che studiano l’hijacking del dispatch dei driver, l’esecuzione basata su shellcode all’interno di driver esistenti e le tecniche per ridurre la visibilità di payload kernel personalizzati.
