Этот проект — proof of concept драйвера ядра, построенный вокруг выполнения пользовательского shellcode через другие драйверы, чтобы не раскрывать обычный путь к образу самостоятельного драйвера.
В архивированном README указано, что он был рассчитан на внедрение shellcode в другой драйвер для обхода ограничений, связанных с подписью; в дереве исходников есть отдельные файлы shellcode, вспомогательные средства inline-hook и модули патчей, а не только обычный интерфейс устройства.
Реализация патчит существующие dispatch-процедуры, например обработчики драйвера NTFS, временно отключает защиту записи для установки inline-переходов и сохраняет trampoline, чтобы исходный путь можно было по-прежнему вызывать или восстановить при выгрузке.
Проект в основном полезен исследователям ядра Windows, изучающим перехват dispatch драйверов, выполнение на основе shellcode внутри существующих драйверов и техники снижения видимости пользовательских полезных нагрузок ядра.
