本プロジェクトは、通常のカーネルモジュール読み込み経路ではなく、ドライバーイメージをセッションメモリへ手動で読み込むセッション空間ドライバーマッパーです。
アーカイブされた README ではセッション空間へのドライバーマッピングとして説明されており、カーネル側コンポーネントと、生のイメージデータを渡してインポート修正、セクションコピー、再配置、エントリポイント実行を行うクライアントが組み合わされています。
主なフックは内部の `ntoskrnl` コールバック経路を置き換え、対象イメージ用にセッション空間メモリを割り当て、インポートと再配置を解決し、アンロード時にはローダーメタデータの一部を消去してマッピング済みドライバーの可視性を下げます。
主に、セッション空間への手動マッピング、代替的なドライバー読み込みフロー、通常のローダー構造からマッピング済みモジュールを隠す手法を研究する Windows カーネル研究者向けです。
