Ce projet est un mappeur de pilotes en espace de session qui charge manuellement une image de pilote en mémoire de session, au lieu d'utiliser le chemin normal de chargement des modules noyau.
Le README archivé le décrit comme un mappage de pilotes dans l'espace de session ; la solution associe un composant noyau à un client qui transmet les données brutes de l'image pour la réparation des imports, la copie des sections, la relocation et l'exécution du point d'entrée.
Son hook principal remplace un chemin de callback interne de `ntoskrnl`, alloue de la mémoire d'espace de session pour l'image cible, résout les imports et relocations, puis efface une partie des métadonnées du chargeur au déchargement afin de réduire la visibilité du pilote mappé.
Il est surtout utile aux chercheurs du noyau Windows qui étudient le mappage manuel en espace de session, les flux alternatifs de chargement de pilotes, et les techniques pour masquer les modules mappés des structures ordinaires du chargeur.
