Dieses Projekt ist ein Kommandozeilen-Proof-of-Concept zum Missbrauch des Genshin-Impact-Treibers mhyprot2.sys, um speicherbezogene Operationen mit Kernel-Privilegien aus dem User Mode heraus durchzuführen.
Die archivierte README beschreibt den Fehler als verwundbare IOCTL-Pfade, die MmCopyVirtualMemory und memcpy-ähnliches Verhalten freigeben und so beliebiges Lesen und Schreiben von Kernel- und User-Speicher ermöglichen, sobald der Dienst läuft.
Die CLI findet einen Zielprozess, initialisiert den verwundbaren Dienst und das Gerät in mhyprot::init und dispatcht anschließend Tests oder Hilfsoperationen über eine schlanke Treiberabstraktion — damit ist sie ein nutzbares Frontend zum Experimentieren mit dem Issue, nicht nur ein Write-up.
Sie ist vor allem für Windows-Sicherheitsforscher nützlich, die verwundbare Anti-Cheat-Treiber, Ring3-zu-Ring0-Speicherprimitive und dienstbasierte BYOVD-Workflows untersuchen.
