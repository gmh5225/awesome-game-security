本项目是一个命令行概念验证，用于滥用《原神》的 mhyprot2.sys 驱动，以便从用户模式执行具有内核权限的内存操作。
归档 README 将缺陷描述为存在漏洞的 IOCTL 路径，这些路径暴露了 MmCopyVirtualMemory 以及类似 memcpy 的行为，一旦服务运行，即可对内核与用户内存进行任意读写。
该 CLI 会查找目标进程，在 mhyprot::init 中初始化有漏洞的服务与设备，再通过精简的驱动抽象层分发测试或实用操作，从而成为可实际使用的问题复现前端，而不只是一篇分析文章。
它主要面向研究有漏洞的反作弊驱动、ring3 到 ring0 内存原语，以及基于服务的 BYOVD 工作流的 Windows 安全研究人员。
