本專案是一個命令列概念驗證，用於濫用《原神》的 mhyprot2.sys 驅動程式，以便從使用者模式執行具有核心權限的記憶體操作。
歸檔 README 將缺陷描述為存在漏洞的 IOCTL 路徑，這些路徑暴露了 MmCopyVirtualMemory 以及類似 memcpy 的行為，一旦服務執行，即可對核心與使用者記憶體進行任意讀寫。
該 CLI 會尋找目標行程，在 mhyprot::init 中初始化有漏洞的服務與裝置，再透過精簡的驅動抽象層分派測試或實用操作，從而成為可實際使用的問題實驗前端，而不只是一篇分析文章。
它主要面向研究有漏洞的反作弊驅動、ring3 到 ring0 記憶體原語，以及基於服務的 BYOVD 工作流程的 Windows 安全性研究人員。
