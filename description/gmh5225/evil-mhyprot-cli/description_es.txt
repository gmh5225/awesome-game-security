Este proyecto es una prueba de concepto por línea de comandos para abusar del controlador mhyprot2.sys de Genshin Impact y realizar operaciones de memoria con privilegios de kernel desde modo usuario.
El README archivado describe el fallo como rutas IOCTL vulnerables que exponen MmCopyVirtualMemory y un comportamiento similar a memcpy, lo que permite lectura y escritura arbitrarias de memoria de kernel y de usuario una vez que el servicio está en ejecución.
La CLI localiza un proceso objetivo, inicializa el servicio y el dispositivo vulnerables en mhyprot::init y luego despacha pruebas u operaciones de utilidad mediante una pequeña abstracción del controlador, convirtiéndola en un frontend usable para experimentar con el problema y no solo en un write-up.
Resulta especialmente útil para investigadores de seguridad de Windows que estudian controladores anti-trampas vulnerables, primitivas de memoria de ring3 a ring0 y flujos BYOVD basados en servicios.
