Ce projet est une preuve de concept en ligne de commande visant à abuser du pilote mhyprot2.sys de Genshin Impact pour effectuer des opérations mémoire avec privilèges noyau depuis le mode utilisateur.
Le README archivé décrit la faille comme des chemins IOCTL vulnérables qui exposent MmCopyVirtualMemory et un comportement de type memcpy, permettant une lecture et une écriture arbitraires de la mémoire noyau et utilisateur une fois le service en cours d’exécution.
Le CLI trouve un processus cible, initialise le service et le périphérique vulnérables dans mhyprot::init, puis dispatch des tests ou des opérations utilitaires via une petite abstraction de pilote, ce qui en fait une interface utilisable pour expérimenter le problème plutôt qu’un simple write-up.
Il est surtout utile aux chercheurs en sécurité Windows qui étudient les pilotes anti-triche vulnérables, les primitives mémoire ring3-vers-ring0 et les workflows BYOVD basés sur des services.
