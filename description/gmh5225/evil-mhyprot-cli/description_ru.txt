Этот проект — командная proof of concept для злоупотребления драйвером mhyprot2.sys из Genshin Impact с целью выполнения операций с памятью с привилегиями ядра из пользовательского режима.
В архивном README уязвимость описана как уязвимые пути IOCTL, которые раскрывают MmCopyVirtualMemory и поведение, похожее на memcpy, что позволяет произвольно читать и писать память ядра и пользовательскую память после запуска службы.
CLI находит целевой процесс, инициализирует уязвимую службу и устройство в mhyprot::init, а затем диспетчеризует тесты или служебные операции через небольшое абстрагирование драйвера, превращаясь в удобный фронтенд для экспериментов с проблемой, а не только в write-up.
Он в первую очередь полезен исследователям безопасности Windows, изучающим уязвимые античит-драйверы, примитивы памяти ring3-to-ring0 и сервисные сценарии BYOVD.
