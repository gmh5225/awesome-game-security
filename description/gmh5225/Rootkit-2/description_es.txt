Este proyecto es una prueba de concepto en el kernel para detectar procesos ocultos recorriendo las estructuras de datos mantenidas por csrss, en lugar de confiar en las rutas habituales de enumeración de procesos. El controlador se adjunta a cada instancia de csrss.exe, localiza CsrExecServerThread dentro de csrsrv.dll, la analiza por patrones para recuperar la cabecera de la lista CSR_PROCESS y luego recorre esa lista enlazada para resolver cada PID de cliente de vuelta a un EPROCESS. Al imprimir los identificadores de proceso y los nombres de imagen recuperados de la lista CSRSS, el repositorio demuestra una ruta de enumeración secundaria que aún puede exponer procesos ocultos a vistas más obvias. Resulta especialmente útil para investigadores de anti-cheat y del kernel que estudian la detección de procesos ocultos, los internos de CSRSS y estrategias alternativas de enumeración para el triage de rootkits.
