Ce projet est une preuve de concept pour contourner Windows HVCI (Hypervisor-Protected Code Integrity) afin d'exécuter du code noyau non signé. HVCI empêche normalement le chargement de pilotes non signés en s'appuyant sur Hyper-V pour appliquer l'intégrité du code au niveau de l'hyperviseur. Cet outil présente des techniques pour contourner la protection HVCI, telles que l'exploitation de cas limites dans la politique HVCI ou l'utilisation de pilotes signés vulnérables. Il s'adresse aux chercheurs en sécurité du noyau qui étudient les méthodes de contournement d'HVCI et les limites de la sécurité basée sur la virtualisation de Windows.
