Questo progetto è una proof of concept per aggirare Windows HVCI (Hypervisor-Protected Code Integrity) ed eseguire codice kernel non firmato. HVCI normalmente impedisce il caricamento di driver non firmati usando Hyper-V per applicare l'integrità del codice a livello di hypervisor. Questo strumento dimostra tecniche per eludere la protezione HVCI, ad esempio sfruttando casi limite nella policy HVCI o facendo leva su driver firmati vulnerabili. È rivolto ai ricercatori di sicurezza del kernel che studiano i metodi di bypass di HVCI e i limiti della sicurezza basata sulla virtualizzazione di Windows.
