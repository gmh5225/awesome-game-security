Dieses Projekt ist ein Proof of Concept zum Umgehen von Windows HVCI (Hypervisor-Protected Code Integrity), um unsignierten Kernelcode auszuführen. HVCI verhindert normalerweise das Laden unsignierter Treiber, indem Hyper-V die Codeintegrität auf Hypervisor-Ebene durchsetzt. Dieses Tool zeigt Techniken zur Umgehung des HVCI-Schutzes, etwa durch Ausnutzen von Grenzfällen in der HVCI-Richtlinie oder durch Nutzung anfälliger signierter Treiber. Es richtet sich an Kernel-Sicherheitsforscher, die HVCI-Bypass-Methoden und die Grenzen der virtualisierungsbasierten Sicherheit von Windows untersuchen.
