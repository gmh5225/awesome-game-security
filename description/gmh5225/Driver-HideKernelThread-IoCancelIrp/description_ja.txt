このリポジトリは、カスタムカーネルスレッドの可視な開始ルーチンを `IoCancelIrp` のように見せることで当該スレッドを偽装する、小さな概念実証です。
アーカイブされた README では手法が直接説明されています。IRP を作成し、カスタムキャンセルルーチンを設定したうえで、システムスレッドを `IoCancelIrp` から開始し、実際のペイロードはスレッドの名目上のエントリポイントとしてではなく、後から IRP キャンセル経路経由で実行されるようにします。
単一のソースファイルが、IRP 割り当て、キャンセルルーチンの配線、`UserBuffer` と `MdlAddress` によるコンテキスト引き渡し、さらに `IoCancelIrp` 起点のスレッド開始の検査や APC によるシステムスレッドスタックの走査といった検出戦略の注記まで、一連の流れを示します。
隠蔽されたスレッド起動手法と、なおそれらを露呈し得る防御ヒューリスティックを研究する Windows カーネル研究者向けに主に有用です。
