Questo repository è una piccola proof of concept per mascherare un thread kernel personalizzato facendo sì che la sua routine di avvio visibile appaia come `IoCancelIrp`.
Il README archiviato spiega direttamente il trucco: creare un IRP, impostare una routine di cancel personalizzata, quindi avviare il thread di sistema su `IoCancelIrp`, così che il payload reale venga eseguito in seguito attraverso il percorso di cancellazione dell’IRP invece di apparire come punto di ingresso nominale del thread.
Il suo unico file sorgente dimostra l’intero flusso, inclusa l’allocazione dell’IRP, il collegamento della cancel routine, il passaggio di contesto tramite `UserBuffer` e `MdlAddress`, e note sulle strategie di rilevamento come controllare avvii di thread su `IoCancelIrp` o attraversare gli stack dei thread di sistema con APC.
È soprattutto utile per i ricercatori del kernel Windows che studiano tecniche di avvio thread occultate e le euristiche difensive che possono ancora esporle.
