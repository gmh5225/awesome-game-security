Ce dépôt est une petite preuve de concept visant à déguiser un thread noyau personnalisé en faisant apparaître sa routine de démarrage visible comme `IoCancelIrp`.
Le README archivé explique directement l’astuce : créer un IRP, définir une routine d’annulation personnalisée, puis démarrer le thread système sur `IoCancelIrp`, de sorte que la charge utile réelle s’exécute plus tard via le chemin d’annulation d’IRP plutôt que d’apparaître comme le point d’entrée nominal du thread.
Son unique fichier source démontre le flux complet, notamment l’allocation d’IRP, le branchement de la routine d’annulation, le transfert de contexte via `UserBuffer` et `MdlAddress`, ainsi que des notes sur les stratégies de détection telles que la recherche de démarrages de threads sur `IoCancelIrp` ou le parcours des piles des threads système avec des APC.
Il est surtout utile aux chercheurs du noyau Windows qui étudient les techniques de démarrage de threads dissimulées et les heuristiques défensives susceptibles de les exposer encore.
