Этот репозиторий — небольшой proof of concept для маскировки пользовательского потока ядра: видимая стартовая процедура выглядит как `IoCancelIrp`.
Архивный README прямо объясняет приём: создать IRP, задать пользовательскую cancel-процедуру, затем запустить системный поток с адреса `IoCancelIrp`, чтобы реальная полезная нагрузка выполнилась позже через путь отмены IRP, а не выглядела как номинальная точка входа потока.
Единственный исходный файл демонстрирует полный сценарий: выделение IRP, подключение cancel-процедуры, передачу контекста через `UserBuffer` и `MdlAddress`, а также заметки о стратегиях обнаружения — например, поиск стартов потоков с `IoCancelIrp` или обход стеков системных потоков с помощью APC.
Он в основном полезен исследователям ядра Windows, изучающим скрытые техники запуска потоков и защитные эвристики, которые всё ещё могут их выявить.
