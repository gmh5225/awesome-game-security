本仓库是一个小型概念验证，通过让自定义内核线程的可见启动例程看起来像 `IoCancelIrp` 来伪装该线程。
存档 README 直接说明了这一技巧：创建 IRP、设置自定义取消例程，再将系统线程启动地址设为 `IoCancelIrp`，使真实载荷稍后经由 IRP 取消路径执行，而不是表现为线程的名义入口点。
其单一源文件演示了完整流程，包括 IRP 分配、取消例程接线、通过 `UserBuffer` 与 `MdlAddress` 传递上下文，以及关于检测方法的说明，例如检查以 `IoCancelIrp` 启动的线程，或用 APC 遍历系统线程栈。
它主要面向研究隐蔽线程启动技术、以及仍可暴露此类手法的防御启发式的 Windows 内核研究人员。
