Questo progetto è una proof-of-concept che abbina un driver EFI runtime a un client Windows per operazioni di lettura e scrittura diretta della memoria dei processi dopo l'avvio del sistema operativo.
Il README archiviato lo descrive come un esempio minimale di process dumper basato su EFI, e l'albero del codice sorgente include sia il componente runtime `driver` sia un'applicazione `client` separata che comunica con esso.
Il flusso di build e utilizzo ruota intorno alla compilazione di `driver.efi` con `gnu-efi`, al caricamento da una shell EDK2 su un dispositivo USB FAT32 e all'uso del lato client dopo l'avvio di Windows; l'autore sottolinea esplicitamente la mancanza di hardening e di controlli di memory safety.
È soprattutto utile per ricercatori di basso livello su Windows e firmware che studiano come i driver EFI runtime possano sopravvivere nell'OS ed esporre primitive di accesso alla memoria a uno strumento companion in user mode.
