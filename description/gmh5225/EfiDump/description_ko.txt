이 프로젝트는 운영 체제 부팅 이후 프로세스 메모리를 직접 읽고 쓰기 위한 개념 증명용 EFI 런타임 드라이버와 Windows 클라이언트 조합입니다.
아카이브된 README는 이를 EFI 기반 프로세스 덤퍼의 최소 예제로 설명하며, 소스 트리에는 `driver` 런타임 구성 요소와 이와 통신하는 별도의 `client` 애플리케이션이 모두 포함되어 있습니다.
빌드 및 사용 흐름은 `gnu-efi`로 `driver.efi`를 컴파일하고 FAT32 USB 장치의 EDK2 셸에서 로드한 뒤 Windows 시작 후 클라이언트 측을 사용하는 방식이며, 저자는 강화와 메모리 안전성 검사가 없음을 명시적으로 밝히고 있습니다.
런타임 EFI 드라이버가 OS 안으로 살아남아 동반 사용자 모드 도구에 메모리 접근 프리미티브를 노출하는 방식을 연구하는 저수준 Windows 및 펌웨어 연구자에게 주로 유용합니다.
