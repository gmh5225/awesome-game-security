Этот проект — proof-of-concept EFI runtime-драйвера в паре с Windows-клиентом для прямого чтения и записи памяти процессов после загрузки операционной системы.
В архивном README он описан как минимальный пример EFI-based process dumper; в дереве исходников есть runtime-компонент `driver` и отдельное приложение `client`, взаимодействующее с ним.
Сборка и использование сводятся к компиляции `driver.efi` с помощью `gnu-efi`, загрузке из оболочки EDK2 с USB-устройства FAT32 и последующей работе с клиентом после старта Windows; автор явно отмечает отсутствие hardening и проверок memory safety.
Проект в первую очередь полезен исследователям низкоуровневых аспектов Windows и прошивок, изучающим, как runtime EFI-драйверы могут сохраняться после перехода в ОС и предоставлять примитивы доступа к памяти сопутствующему user-mode инструменту.
