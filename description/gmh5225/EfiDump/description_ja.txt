本プロジェクトは、オペレーティングシステム起動後にプロセスメモリを直接読み書きするための、概念実証用 EFI ランタイムドライバと Windows クライアントの組み合わせです。
アーカイブされた README では、EFI ベースのプロセスダンパーの最小限の例として説明されており、ソースツリーには `driver` ランタイムコンポーネントと、それと通信する独立した `client` アプリケーションの両方が含まれています。
ビルドと利用の流れは、`gnu-efi` で `driver.efi` をコンパイルし、FAT32 USB デバイス上の EDK2 シェルから読み込み、Windows 起動後にクライアント側を使うという構成で、作者はハードニングやメモリ安全性チェックの欠如を明示しています。
ランタイム EFI ドライバが OS 上で存続し、連携するユーザーモードツールにメモリアクセスプリミティブを提供する仕組みを研究する、低レベルな Windows およびファームウェア研究者向けに主に有用です。
