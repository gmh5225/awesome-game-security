Ce projet est une preuve de concept associant un pilote EFI en mode runtime et un client Windows pour des opérations de lecture et d'écriture directe de la mémoire de processus après le démarrage du système d'exploitation.
Le README archivé le présente comme un exemple minimal de dumper de processus basé sur EFI, et l'arborescence source comprend à la fois le composant runtime `driver` et une application `client` distincte qui communique avec lui.
Son flux de construction et d'utilisation consiste à compiler `driver.efi` avec `gnu-efi`, à le charger depuis un shell EDK2 sur un périphérique USB FAT32, puis à utiliser le côté client une fois Windows démarré, l'auteur soulignant explicitement l'absence de durcissement et de contrôles de sécurité mémoire.
Il est surtout utile aux chercheurs bas niveau sur Windows et le firmware qui étudient comment les pilotes EFI runtime peuvent survivre dans l'OS et exposer des primitives d'accès mémoire à un outil compagnon en mode utilisateur.
