Este proyecto es una prueba de concepto que combina un controlador EFI en tiempo de ejecución con un cliente de Windows para operaciones de lectura y escritura directa de memoria de procesos tras el arranque del sistema operativo.
El README archivado lo describe como un ejemplo mínimo de un dumper de procesos basado en EFI, y el árbol de código fuente incluye tanto el componente de tiempo de ejecución `driver` como una aplicación `client` independiente que se comunica con él.
Su flujo de compilación y uso se centra en compilar `driver.efi` con `gnu-efi`, cargarlo desde un shell EDK2 en un dispositivo USB FAT32 y, a continuación, usar el lado cliente tras iniciar Windows; el autor señala explícitamente la falta de endurecimiento y de comprobaciones de seguridad de memoria.
Resulta especialmente útil para investigadores de bajo nivel de Windows y firmware que estudian cómo los controladores EFI en tiempo de ejecución pueden persistir en el SO y exponer primitivas de acceso a memoria a una herramienta compañera en modo usuario.
