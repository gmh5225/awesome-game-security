Este proyecto es una pequeña demostración en modo usuario de secuestro de handles: enumera los handles del sistema y duplica un handle objetivo desde otro proceso. El ejemplo llama a NtQuerySystemInformation con SystemHandleInformation, busca en la tabla devuelta un puntero de objeto elegido, abre el proceso propietario con PROCESS_DUP_HANDLE y usa DuplicateHandle para copiar ese handle al proceso actual. Su README dedica la mayor parte del espacio a explicar el modelo de ataque y los parámetros de DuplicateHandle, así que el código se entiende mejor como una prueba de concepto mínima de reutilización de un handle robado, no como un framework de propósito general. Resulta útil sobre todo para investigadores de seguridad en Windows que quieran un ejemplo compacto de enumeración de la tabla de handles, duplicación entre procesos y la mecánica práctica del handle hijacking.
