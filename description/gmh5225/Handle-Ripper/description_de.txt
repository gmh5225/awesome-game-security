Dieses Projekt ist eine kleine User-Mode-Demonstration von Handle-Hijacking: System-Handles werden aufgezählt und ein Ziel-Handle aus einem anderen Prozess dupliziert. Das Beispiel ruft NtQuerySystemInformation mit SystemHandleInformation auf, durchsucht die zurückgegebene Tabelle nach einem gewählten Objektzeiger, öffnet den besitzenden Prozess mit PROCESS_DUP_HANDLE und kopiert das Handle mit DuplicateHandle in den aktuellen Prozess. Die README erklärt vor allem das Angriffsmodell und die DuplicateHandle-Parameter; der Code ist daher am besten als minimaler Proof of Concept für die Wiederverwendung gestohlener Handles zu lesen, nicht als allgemeines Framework. Er ist vor allem für Windows-Sicherheitsforscher nützlich, die ein kompaktes Beispiel für Handle-Tabellen-Enumeration, prozessübergreifende Handle-Duplizierung und die praktischen Mechanismen hinter Handle-Hijacking suchen.
