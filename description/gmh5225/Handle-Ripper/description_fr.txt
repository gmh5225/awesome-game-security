Ce projet est une petite démonstration en mode utilisateur du détournement de handles, en énumérant les handles système et en dupliquant un handle cible depuis un autre processus. L’exemple appelle NtQuerySystemInformation avec SystemHandleInformation, parcourt la table renvoyée pour un pointeur d’objet choisi, ouvre le processus propriétaire avec PROCESS_DUP_HANDLE, puis utilise DuplicateHandle pour copier ce handle dans le processus courant. Son README consacre l’essentiel de l’espace au modèle d’attaque et aux paramètres de DuplicateHandle ; le code se lit donc surtout comme une preuve de concept minimale de réutilisation d’un handle volé, plutôt que comme un framework généraliste. Il est surtout utile aux chercheurs en sécurité Windows qui veulent un exemple compact d’énumération de la table des handles, de duplication inter-processus et des mécanismes concrets du handle hijacking.
