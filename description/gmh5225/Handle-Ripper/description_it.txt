Questo progetto è una piccola dimostrazione in user mode di handle hijacking: enumera gli handle di sistema e duplica un handle target da un altro processo. L’esempio chiama NtQuerySystemInformation con SystemHandleInformation, scorre la tabella restituita alla ricerca di un puntatore a oggetto scelto, apre il processo proprietario con PROCESS_DUP_HANDLE e usa DuplicateHandle per copiare quell’handle nel processo corrente. Il README dedica gran parte dello spazio al modello di attacco e ai parametri di DuplicateHandle, quindi il codice va letto soprattutto come proof of concept minimale sul riuso di handle rubati, non come un framework generico. È utile soprattutto ai ricercatori di sicurezza Windows che vogliono un esempio compatto di enumerazione della tabella degli handle, duplicazione cross-process e della meccanica pratica dell’handle hijacking.
