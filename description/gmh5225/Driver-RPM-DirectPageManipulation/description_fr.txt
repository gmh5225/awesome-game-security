Ce projet est un exemple minimal de noyau Windows montrant comment lire et écrire la mémoire d'un autre processus en manipulant directement les structures de pagination.
Son README présente le code comme une démonstration minimale, et l'implémentation fait exactement cela en allouant une page contiguë, en localisant son propre PTE, en écrasant le numéro de cadre de page pour remapper des pages physiques arbitraires, puis en utilisant une traduction manuelle virtuel-vers-physique pour implémenter des routines de copie mémoire entre processus.
Le point d'entrée du pilote exemple applique la technique à un processus cible pour lire des données de module sans s'appuyer sur les aides de copie documentées standard, tout en laissant volontairement de côté la gestion de la mémoire paginée hors RAM et les questions de multithreading pour garder la logique simple.
Il est surtout utile aux chercheurs du noyau Windows qui souhaitent une référence compacte sur la réécriture de PTE, la traduction d'adresses manuelle et l'accès à la mémoire des processus via les tables de pages.
