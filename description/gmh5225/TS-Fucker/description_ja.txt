本プロジェクトは、Dell の脆弱なドライバ dbutil_2_3.sys を悪用してカーネルの読み書き権限を得ることにより、実行時に Windows の TestSigning モードを切り替える概念実証です。
アーカイブされた README では、再起動なしでマシンのテスト署名状態を変更すること、必要なフィールドを特定するために現行 Windows ビルド向けのシンボルファイルをダウンロードすること、および当該脆弱ドライバが既にロードされていることを前提とすることが強調されています。
そのため、このリポジトリは汎用的な BYOVD ツールキットというより、カーネルメモリプリミティブを用いてセキュリティ関連のシステム状態をライブパッチする方法を示す、焦点を絞ったユーティリティです。
主に、実行時のテスト署名操作、シンボル支援によるオフセット発見、および dbutil_2_3.sys の実用的な影響を研究する Windows カーネル研究者向けです。
