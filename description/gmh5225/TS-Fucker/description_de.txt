Dieses Projekt ist ein Proof of Concept zum Umschalten des Windows-TestSigning-Modus zur Laufzeit, indem der anfällige Dell-Treiber dbutil_2_3.sys für Kernel-Lese-/Schreibzugriff missbraucht wird.
Die archivierte README betont, dass der Testsignatur-Zustand der Maschine ohne Neustart geändert wird, Symboldateien für den aktuellen Windows-Build heruntergeladen werden, um die benötigten Felder zu finden, und der anfällige Treiber bereits geladen sein muss.
Daher ist das Repository weniger ein generisches BYOVD-Toolkit als vielmehr ein fokussiertes Hilfsprogramm, das zeigt, wie eine Kernel-Speicherprimitiv sicherheitsrelevante Systemzustände live patchen kann.
Es ist vor allem für Windows-Kernel-Forscher nützlich, die die Laufzeit-Manipulation von Test-Signing, symbolgestützte Offset-Ermittlung und die praktischen Auswirkungen von dbutil_2_3.sys untersuchen.
