Este proyecto es una prueba de concepto para activar o desactivar el modo TestSigning de Windows en tiempo de ejecución abusando del controlador vulnerable de Dell dbutil_2_3.sys para obtener acceso de lectura-escritura al kernel.
El README archivado enfatiza que cambia el estado de firma de prueba de la máquina sin reiniciar, descarga archivos de símbolos de la compilación actual de Windows para localizar los campos necesarios y espera que el controlador vulnerable ya esté cargado.
Por tanto, el repositorio no es tanto un kit genérico BYOVD como una utilidad enfocada que muestra cómo una primitiva de memoria del kernel puede parchear en vivo un estado del sistema relevante para la seguridad.
Resulta especialmente útil para investigadores del kernel de Windows que estudian la manipulación del test-signing en tiempo de ejecución, el descubrimiento de offsets asistido por símbolos y el impacto práctico de dbutil_2_3.sys.
