Ce projet est une preuve de concept permettant d’activer ou désactiver le mode TestSigning de Windows à l’exécution en abusant du pilote Dell vulnérable dbutil_2_3.sys pour obtenir un accès en lecture-écriture au noyau.
Le README archivé souligne qu’il modifie l’état de signature de test de la machine sans redémarrage, télécharge les fichiers de symboles de la version Windows courante pour localiser les champs requis, et suppose que le pilote vulnérable est déjà chargé.
Par conséquent, le dépôt n’est pas une boîte à outils BYOVD générique, mais plutôt un utilitaire ciblé montrant comment une primitive de mémoire noyau peut modifier à chaud un état système lié à la sécurité.
Il s’adresse surtout aux chercheurs du noyau Windows qui étudient la manipulation du test-signing à l’exécution, la découverte d’offsets assistée par symboles, et l’impact pratique de dbutil_2_3.sys.
