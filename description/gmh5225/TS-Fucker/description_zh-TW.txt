本專案是一個概念驗證，透過濫用戴爾易受攻擊的驅動程式 dbutil_2_3.sys 取得核心讀寫能力，從而在執行階段切換 Windows TestSigning（測試簽章）模式。
歸檔的 README 強調：它可在無需重新開機的情況下變更機器的測試簽章狀態，會為目前 Windows 版本下載符號檔以定位所需欄位，並假定該易受攻擊驅動程式已經載入。
因此，此儲存庫並非通用的 BYOVD 工具組，而是一個聚焦型公用程式，展示如何利用核心記憶體原語即時修補與安全相關的系統狀態。
它主要面向 Windows 核心研究人員，用於研究執行階段測試簽章操控、借助符號的偏移探索，以及 dbutil_2_3.sys 的實際影響。
