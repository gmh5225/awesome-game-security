本项目是一个概念验证，通过滥用戴尔易受攻击的驱动 dbutil_2_3.sys 获得内核读写能力，从而在运行时切换 Windows TestSigning（测试签名）模式。
归档的 README 强调：它可在无需重启的情况下更改机器的测试签名状态，会为当前 Windows 版本下载符号文件以定位所需字段，并假定该易受攻击驱动已经加载。
因此，该仓库并非通用的 BYOVD 工具集，而是一个聚焦型实用程序，展示如何利用内核内存原语实时修补与安全相关的系统状态。
它主要面向 Windows 内核研究人员，用于研究运行时测试签名操控、借助符号的偏移发现，以及 dbutil_2_3.sys 的实际影响。
