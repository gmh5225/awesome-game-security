Questo progetto è una proof of concept per attivare o disattivare la modalità TestSigning di Windows a runtime abusando del driver vulnerabile Dell dbutil_2_3.sys per ottenere accesso in lettura-scrittura al kernel.
Il README archiviato sottolinea che modifica lo stato di test-signing della macchina senza riavvio, scarica i file di simboli della build Windows corrente per individuare i campi necessari e presuppone che il driver vulnerabile sia già caricato.
Di conseguenza, il repository non è tanto un toolkit BYOVD generico quanto un’utilità mirata che mostra come una primitiva di memoria del kernel possa patchare dal vivo uno stato di sistema rilevante per la sicurezza.
È soprattutto utile per i ricercatori del kernel Windows che studiano la manipolazione del test-signing a runtime, la scoperta di offset assistita dai simboli e l’impatto pratico di dbutil_2_3.sys.
