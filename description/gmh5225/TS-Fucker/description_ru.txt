Этот проект — proof of concept для переключения режима Windows TestSigning во время выполнения за счёт злоупотребления уязвимым драйвером Dell dbutil_2_3.sys с целью получения доступа на чтение и запись в ядре.
В архивированном README подчёркивается, что состояние тестовой подписи машины меняется без перезагрузки, для текущей сборки Windows загружаются файлы символов, чтобы найти нужные поля, и предполагается, что уязвимый драйвер уже загружен.
Таким образом, репозиторий — не универсальный BYOVD-набор, а скорее узкоспециализированная утилита, показывающая, как примитив доступа к памяти ядра может «на лету» изменять связанное с безопасностью состояние системы.
Он в первую очередь полезен исследователям ядра Windows, изучающим манипуляцию test-signing во время выполнения, поиск смещений с помощью символов и практическое влияние dbutil_2_3.sys.
