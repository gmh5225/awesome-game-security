이 프로젝트는 Dell의 취약한 드라이버 dbutil_2_3.sys를 악용해 커널 읽기·쓰기 권한을 확보함으로써, 런타임에 Windows TestSigning 모드를 전환하는 개념 증명입니다.
보관된 README는 재부팅 없이 시스템의 테스트 서명 상태를 변경한다는 점, 필요한 필드를 찾기 위해 현재 Windows 빌드용 심볼 파일을 다운로드한다는 점, 그리고 해당 취약 드라이버가 이미 로드되어 있어야 한다는 점을 강조합니다.
따라서 이 저장소는 일반적인 BYOVD 툴킷이라기보다, 커널 메모리 프리미티브를 이용해 보안과 관련된 시스템 상태를 라이브 패치하는 방법을 보여주는 초점형 유틸리티에 가깝습니다.
주로 런타임 테스트 서명 조작, 심볼 기반 오프셋 탐색, dbutil_2_3.sys의 실질적 영향을 연구하는 Windows 커널 연구자에게 유용합니다.
