本專案是一個核心驅動程式手動對應器，濫用 nvaudio.sys 作為脆弱傳輸通道，並包含使用者模式載入器與記憶體中對應邏輯。
載入器會啟用 SeLoadDriverPrivilege，安裝並啟動該脆弱驅動程式，開啟其裝置，將 nvaudio.sys 載入使用者空間以重用內部 EncodePayLoad 常式，然後移除驅動程式的執行時期清單項目，使輔助元件在使用後較不易被察覺。
其對應器程式碼還包含明確的重定位、匯入解析與映像載入常式，因此此封存並非對現有對應器的薄包裝，而是一套適配 nvaudio 路徑、相當完整的手動對應實作。
它主要適用於研究基於 BYOVD 的驅動程式對應、廠商輔助常式重用，以及圍繞執行時期清單的載入後清理的 Windows 核心研究者。
