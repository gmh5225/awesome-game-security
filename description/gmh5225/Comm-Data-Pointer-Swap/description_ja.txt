このプロジェクトは、通常のデバイスインターフェイスを公開する代わりに、win32k パス内の内部データポインタを入れ替えることで実現する、概念実証レベルのカーネル通信手法です。アーカイブされたドライバは win32kbase.sys 内のパターンを探し、対象の qword ポインタを解決し、explorer.exe にアタッチしたうえで、InterlockedExchangePointer を使ってそのポインタを独自ハンドラに置き換えます。ユーザーモード側は win32u.dll から NtDCompositionSetChildRootVisual を解決して当該パスを起動するため、本リポジトリは本格的なフレームワークではなく、GUI システムコール経由のポインタスワップ通信を示すコンパクトな例です。主に、隠蔽されたドライバ通信、GUI サブシステム内のポインタリダイレクト、および明示的なワンオフフック配置のトレードオフを研究する Windows カーネル研究者向けです。
