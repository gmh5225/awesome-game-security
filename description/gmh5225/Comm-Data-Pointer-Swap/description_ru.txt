Этот проект — доказательство концепции метода ядровой коммуникации, построенного на подмене внутреннего указателя данных в пути win32k вместо обычного интерфейса устройства. Архивированный драйвер ищет сигнатуру в win32kbase.sys, разрешает целевой qword-указатель, присоединяется к explorer.exe и с помощью InterlockedExchangePointer заменяет этот указатель своим обработчиком. Пользовательская сторона разрешает NtDCompositionSetChildRootVisual из win32u.dll, чтобы вызвать этот путь, поэтому репозиторий — скорее компактный пример pointer-swap-коммуникации через GUI-сисколл, а не полноценный фреймворк. Он в первую очередь полезен исследователям ядра Windows, изучающим скрытую коммуникацию драйверов, перенаправление указателей в GUI-подсистемах и компромиссы явной одноразовой установки хука.
