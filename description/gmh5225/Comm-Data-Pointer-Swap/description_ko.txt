이 프로젝트는 일반적인 디바이스 인터페이스를 노출하는 대신 win32k 경로의 내부 데이터 포인터를 교체하는 방식으로 구현된 개념 증명용 커널 통신 기법입니다. 보관된 드라이버는 win32kbase.sys에서 패턴을 찾고, 대상 qword 포인터를 해석한 뒤 explorer.exe에 연결하며, InterlockedExchangePointer로 해당 포인터를 자체 핸들러로 바꿉니다. 사용자 모드 측은 win32u.dll에서 NtDCompositionSetChildRootVisual을 해석해 해당 경로를 트리거하므로, 이 저장소는 완전한 프레임워크가 아니라 GUI 시스템 호출을 통한 포인터 스왑 통신의 간결한 예제입니다. 은닉 드라이버 통신, GUI 서브시스템 내부 포인터 리다이렉션, 그리고 명백한 일회성 훅 배치의 장단점을 연구하는 Windows 커널 연구자에게 주로 유용합니다.
