Dieses Projekt ist eine Proof-of-Concept-Methode zur Kernel-Kommunikation, die statt einer normalen Geräteschnittstelle auf dem Austauschen eines internen Datenzeigers in einem win32k-Pfad basiert. Der archivierte Treiber sucht ein Muster in win32kbase.sys, löst den Ziel-qword-Zeiger auf, hängt sich an explorer.exe und ersetzt diesen Zeiger mit InterlockedExchangePointer durch den eigenen Handler. Die User-Mode-Seite löst NtDCompositionSetChildRootVisual aus win32u.dll auf, um den Pfad auszulösen; das Repository ist daher eher ein kompaktes Beispiel für Pointer-Swap-Kommunikation über einen GUI-Syscall als ein vollständiges Framework. Es eignet sich vor allem für Windows-Kernel-Forscher, die verdeckte Treiberkommunikation, Zeigerumleitung in GUI-Subsystemen und die Kompromisse einer offensichtlichen einmaligen Hook-Platzierung untersuchen.
