Este proyecto es un método de comunicación en modo kernel de tipo prueba de concepto, basado en intercambiar un puntero de datos interno en una ruta de win32k en lugar de exponer una interfaz de dispositivo normal. El controlador archivado busca un patrón en win32kbase.sys, resuelve el puntero qword objetivo, se adjunta a explorer.exe y usa InterlockedExchangePointer para reemplazar ese puntero por su propio manejador. En modo usuario se resuelve NtDCompositionSetChildRootVisual desde win32u.dll para activar la ruta, por lo que el repositorio es en realidad un ejemplo compacto de comunicación por intercambio de punteros mediante una syscall de GUI, no un framework completo. Resulta sobre todo útil para investigadores del kernel de Windows que estudian la comunicación encubierta de controladores, la redirección de punteros en subsistemas GUI y las compensaciones de colocar un hook puntual demasiado obvio.
