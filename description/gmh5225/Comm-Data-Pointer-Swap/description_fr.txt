Ce projet est une méthode de communication noyau de type preuve de concept, fondée sur le remplacement d’un pointeur de données interne dans un chemin win32k plutôt que sur l’exposition d’une interface de périphérique classique. Le pilote archivé recherche un motif dans win32kbase.sys, résout le pointeur qword cible, s’attache à explorer.exe, puis utilise InterlockedExchangePointer pour remplacer ce pointeur par son propre gestionnaire. Côté mode utilisateur, NtDCompositionSetChildRootVisual est résolu depuis win32u.dll pour déclencher ce chemin ; le dépôt est donc surtout un exemple compact de communication par échange de pointeur via un appel système GUI, et non un framework complet. Il s’adresse surtout aux chercheurs du noyau Windows qui étudient la communication furtive de pilotes, la redirection de pointeurs dans les sous-systèmes GUI, et les compromis d’un crochet ponctuel trop visible.
