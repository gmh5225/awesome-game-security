Questo progetto è un metodo di comunicazione kernel a livello di proof-of-concept, basato sullo scambio di un puntatore di dati interno in un percorso win32k invece di esporre una normale interfaccia di dispositivo. Il driver archiviato individua un pattern in win32kbase.sys, risolve il puntatore qword di destinazione, si attacca a explorer.exe e usa InterlockedExchangePointer per sostituire quel puntatore con il proprio handler. Il lato user-mode risolve NtDCompositionSetChildRootVisual da win32u.dll per attivare il percorso, quindi il repository è in realtà un esempio compatto di comunicazione tramite pointer-swap attraverso una syscall GUI, non un framework completo. È soprattutto utile per i ricercatori del kernel Windows che studiano la comunicazione nascosta dei driver, la redirezione di puntatori nei sottosistemi GUI e i compromessi di un hook una tantum troppo evidente.
