Ce projet est un framework d'overlay Escape From Tarkov apparié en mode utilisateur et mode noyau, qui communique via un hook sur NtDxgkGetTrackedWorkloadStatistics de dxgkrnl. Le composant noyau patch l'export avec un petit stub de saut, traite les requêtes de recherche de base de module et de lecture ou d'écriture de la mémoire de processus, et résout aussi les exports de dessin win32k afin de pouvoir rendre des boîtes et du texte depuis l'espace noyau. Côté mode utilisateur, le même syscall est encapsulé depuis win32u.dll, des structures de requête sont envoyées dans le chemin accroché, et une overlay basée sur ImGui ainsi que des visuels spécifiques au jeu s'ajoutent sur ce canal de communication. Il est surtout utile aux reverse engineers qui étudient la communication de pilotes basée sur dxgkrnl, les chemins de dessin assistés par le noyau, et les conceptions d'overlay mixtes utilisateur/noyau pour des outils de type Tarkov.
