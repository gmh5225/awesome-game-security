Dieses Projekt ist eine modifizierte Version des BTBD-Hardware-Identifier-Spoofers, angepasst für Manual Mapping auf Systemen der Windows-1909-Ära.
Der Code hookt Festplatten- und Partitionssteuerpfade, schreibt serienbezogene Daten um, die von Storage-IOCTLs zurückgegeben werden, löscht GPT-Kennungen, aktualisiert Festplatteneigenschaften, deaktiviert die SMART-Ausfallvorhersage und durchläuft außerdem storahci-RAID-Einheiten, um Serienstrings tiefer im Storage-Stack zu überschreiben.
Das Ergebnis ist nicht nur ein einzelner Serien-Patch, sondern eine breitere Pipeline zum Spoofen der Speicheridentität, die nach dem Laden des Treibers mehrere festplattenbezogene Abfrageflächen konsistent halten soll.
Es ist vor allem für Windows-Kernel-Forscher nützlich, die HWID-Spoofing-Techniken im Storage-Stack analysieren, insbesondere solche, die von manuell gemappten Treibern im BTBD-Stil abgeleitet sind.
