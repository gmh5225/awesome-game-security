Ce projet est une version modifiée du spoofer d’identifiant matériel BTBD, adaptée au manual mapping sur les systèmes de l’ère Windows 1909.
Son code accroche les chemins de contrôle des disques et partitions, réécrit les données liées aux numéros de série renvoyées par les IOCTL de stockage, efface les identifiants GPT, met à jour les propriétés du disque, désactive la prédiction de panne SMART, et parcourt aussi les unités RAID storahci pour écraser les chaînes de série plus profondément dans la pile de stockage.
Le résultat n’est pas un simple correctif de numéro de série, mais un pipeline plus large de spoofing d’identité de stockage qui vise à garder cohérentes plusieurs surfaces d’interrogation liées aux disques après le chargement du pilote.
Il est surtout utile aux chercheurs du noyau Windows qui analysent les techniques de spoofing HWID de la pile de stockage, en particulier celles dérivées des pilotes mappés manuellement de style BTBD.
