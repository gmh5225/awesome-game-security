이 프로젝트는 Windows 1909 세대 시스템에서 수동 매핑(manual mapping)에 맞게 조정한 BTBD 하드웨어 식별자 스푸퍼의 수정 버전입니다.
코드는 디스크 및 파티션 제어 경로를 후킹하고, 스토리지 IOCTL이 반환하는 시리얼 관련 데이터를 다시 쓰며, GPT 식별자를 지우고, 디스크 속성을 갱신하고, SMART 장애 예측을 비활성화하며, storahci RAID 유닛을 순회해 스토리지 스택 더 깊은 계층의 시리얼 문자열도 덮어씁니다.
결과는 단일 시리얼 패치가 아니라, 드라이버 로드 이후 여러 디스크 관련 조회 표면을 일관되게 유지하려는 더 넓은 스토리지 신원 스푸핑 파이프라인입니다.
주로 BTBD 스타일 수동 매핑 드라이버에서 파생된 기법을 포함해, 스토리지 스택 HWID 스푸핑 기법을 분석하는 Windows 커널 연구자에게 유용합니다.
