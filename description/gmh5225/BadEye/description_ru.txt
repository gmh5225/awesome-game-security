Этот проект работает по двум причинам: во-первых, BattlEye предполагает, что у дескриптора уже есть нужный доступ; во-вторых, BattlEye использует дескриптор только для получения EPROCESS, чтобы вызвать MmCopyVirtualMemory. Он в основном написан на C++ и сосредоточен на анализе памяти. В первую очередь полезен исследователям игровой безопасности и специалистам по реверс-инжинирингу, изучающим атакующие техники в области читов и исследования античит-систем.
