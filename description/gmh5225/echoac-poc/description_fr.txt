Ce projet est une preuve de concept appuyée par un writeup sur des vulnérabilités dans echo_driver.sys d'echo.ac, l'exemple fourni se concentrant sur l'élévation de privilèges vers NT AUTHORITY\SYSTEM.
Le PoC utilise l'IOCTL de lecture mémoire du pilote pour fuiter PsInitialSystemProcess, parcourir ActiveProcessLinks, récupérer le jeton SYSTEM depuis EPROCESS, puis écraser le jeton d'un processus cmd.exe nouvellement lancé.
Son archive conserve aussi le contexte selon lequel echo.ac est un outil commercial de partage d'écran utilisé dans les communautés compétitives, si bien que le dépôt sert aussi d'étude de cas sur la façon dont un pilote adjacent à l'anti-cheat peut devenir une surface d'exploitation.
Il est surtout utile aux chercheurs du noyau Windows qui étudient le vol de jeton via un pilote vulnérable, les primitives de lecture noyau, et les chaînes d'élévation de privilèges réelles basées sur des logiciels anti-cheat tiers.
