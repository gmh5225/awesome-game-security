Este proyecto es una prueba de concepto respaldada por un writeup sobre vulnerabilidades en echo_driver.sys de echo.ac; el ejemplo incluido se centra en la elevación de privilegios a NT AUTHORITY\SYSTEM.
El PoC usa el IOCTL de lectura de memoria del controlador para filtrar PsInitialSystemProcess, recorrer ActiveProcessLinks, recuperar el token SYSTEM de EPROCESS y luego sobrescribir el token de un proceso cmd.exe recién iniciado.
Su archivo también conserva el contexto de que echo.ac es una herramienta comercial de compartir pantalla usada en comunidades competitivas, por lo que el repositorio sirve además como estudio de caso de cómo un controlador cercano al anti-cheat puede convertirse en superficie de explotación.
Resulta especialmente útil para investigadores del kernel de Windows que estudian el robo de tokens mediante controladores vulnerables, primitivas de lectura en kernel y cadenas reales de elevación de privilegios basadas en software anti-cheat de terceros.
