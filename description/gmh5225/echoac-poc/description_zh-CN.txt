本项目是一份有 writeup 支撑的概念验证，针对 echo.ac 的 echo_driver.sys 中的漏洞，附带示例聚焦于提升到 NT AUTHORITY\SYSTEM 权限。
该 PoC 利用驱动的读内存 IOCTL 泄露 PsInitialSystemProcess，遍历 ActiveProcessLinks，从 EPROCESS 中取回 SYSTEM token，再覆盖新启动的 cmd.exe 进程的 token。
其归档还保留了背景信息：echo.ac 是竞技社区常用的商业屏幕共享工具，因此本仓库同时可作为反作弊相关驱动如何成为利用面的案例研究。
它主要面向研究易受攻击驱动 token 窃取、内核读原语，以及基于第三方反作弊软件的真实提权链的 Windows 内核研究者。
