本專案是一份有 writeup 支撐的概念驗證，針對 echo.ac 的 echo_driver.sys 中的漏洞，附帶範例聚焦於提升到 NT AUTHORITY\SYSTEM 權限。
該 PoC 利用驅動的讀記憶體 IOCTL 洩漏 PsInitialSystemProcess，遍歷 ActiveProcessLinks，從 EPROCESS 中取回 SYSTEM token，再覆寫新啟動的 cmd.exe 行程的 token。
其封存亦保留了背景脈絡：echo.ac 是競技社群常用的商業螢幕分享工具，因此本儲存庫同時可作為反作弊相關驅動如何成為利用面的案例研究。
它主要面向研究易受攻擊驅動 token 竊取、核心讀取原語，以及基於第三方反作弊軟體的真實提權鏈的 Windows 核心研究者。
