Este proyecto es un exploit para el controlador vulnerable del kernel de Windows ampa.sys. Demuestra cómo la interfaz IOCTL insegura del controlador puede abusarse para obtener lectura/escritura arbitraria en el kernel o ejecución de código, lo que permite cargar controladores sin firma, elevar privilegios de procesos o eludir anti-cheat. El exploit en C/C++ automatiza el disparo de la vulnerabilidad. Está dirigido a investigadores BYOVD que estudian la explotación de controladores vulnerables para acceso a nivel de kernel.
