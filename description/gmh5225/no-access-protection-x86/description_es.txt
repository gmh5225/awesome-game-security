Este proyecto demuestra una técnica de protección de memoria que usa protectores de página PAGE_NOACCESS en Windows x86. Marca las páginas de código como sin acceso y emplea un manejador VEH para restaurar temporalmente el acceso bajo demanda, creando una forma de descifrado al acceder que dificulta el análisis estático y el volcado de memoria. Está dirigido a investigadores de protección de software que estudian el control de acceso a nivel de página para anti-manipulación y anti-dump.
