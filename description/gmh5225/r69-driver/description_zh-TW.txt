本專案是一個精簡的核心通訊函式庫，透過 HalPrivateDispatchTable 掛鉤路由讀、寫與行程查詢請求，而不依賴傳統的裝置物件。
驅動程式將 HalTimerQueryAuxiliaryCounterFrequency 替換為自訂處理常式，從呼叫端 trap frame 取出 c_packet，解碼 read_process_memory、write_process_memory 與 query_process_data 請求，並透過目標行程的 CR3 將虛擬位址轉換為實體頁後完成服務。
第二個掛鉤位於 HalClearLastBranchRecordStack，以目前 CR3 重新整理 DirectoryTableBase；使用者模式封裝則透過 c_r69 類別暴露此路徑，該類別包裝了 NtQueryAuxiliaryCounterFrequency，並可依行程名稱附加到目標行程。
它主要適合 Windows 核心研究者，用於研究接近系統呼叫的通訊方式、HalPrivateDispatchTable 濫用，以及不依賴標準 IOCTL 介面、以實體記憶體為基礎的行程存取。
