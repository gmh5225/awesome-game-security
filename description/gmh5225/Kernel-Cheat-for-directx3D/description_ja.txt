本プロジェクトは、dxgkrnl 内の NtDxgkGetTrackedWorkloadStatistics をハイジャックし、同じチャネルで簡単なメモリ操作と描画を行う、カーネル／ユーザーモード通信のサンプルです。
ドライバは dxgkrnl のエクスポートを小さな絶対ジャンプスタブで上書きし、MmCopyVirtualMemory と KeStackAttachProcess に基づく読み書きヘルパーを公開し、NtUserGetDC、NtGdiPatBlt、NtGdiCreateSolidBrush などの win32k GDI ルーチンを解決して、カーネルコンテキストからボックスを描画します。
ユーザーモードクライアントは win32u.dll 経由で NtDxgkGetTrackedWorkloadStatistics を呼び出し、モジュールベースの取得や矩形描画を要求できる NULL_MEMORY 構造体を渡し、フックしたグラフィックスシステムコールをコマンドディスパッチャとして扱います。
主に、グラフィックス周辺の通信フック、アドホックな読み書きサービス、win32k エクスポート経由のカーネル支援オーバーレイ描画を研究する Windows カーネル研究者向けです。
