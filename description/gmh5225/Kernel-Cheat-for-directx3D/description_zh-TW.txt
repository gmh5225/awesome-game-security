本專案是一個核心與使用者模式通訊範例：它劫持 dxgkrnl 中的 NtDxgkGetTrackedWorkloadStatistics，再透過同一通道完成簡單的記憶體讀寫與繪製操作。
驅動程式以一小段絕對跳躍 stub 覆寫該 dxgkrnl 匯出，基於 MmCopyVirtualMemory 與 KeStackAttachProcess 提供讀寫輔助，並解析 win32k 的 GDI 常式（如 NtUserGetDC、NtGdiPatBlt、NtGdiCreateSolidBrush），以便在核心內容中繪製方框。
使用者模式用戶端透過 win32u.dll 呼叫 NtDxgkGetTrackedWorkloadStatistics，傳入可請求模組基底或繪製矩形的 NULL_MEMORY 結構，並將該被掛鉤的圖形系統呼叫視為命令分派器。
它主要適合研究圖形相關通訊掛鉤、臨時讀寫服務，以及透過 win32k 匯出實作核心輔助覆蓋層繪製的 Windows 核心研究者。
