Ce projet combine un client utilisateur et un pilote noyau mappé manuellement qui réutilisent NtQueryIntervalProfile comme canal de contrôle pour basculer l'état d'Easy Anti-Cheat.
Le pilote remplace des entrées de HalDispatchTable afin que les appels acheminés via NtQueryIntervalProfile atteignent un gestionnaire personnalisé, puis localise les threads d'EasyAntiCheat.sys, les suspend ou les reprend, et désactive temporairement les object callbacks à l'altitude suivie avant de les restaurer plus tard.
Le code côté utilisateur construit un petit stub shellcode, résout NtQueryIntervalProfile depuis ntdll et envoie des requêtes CODE_DISABLE ou CODE_RESTORE, tout en gérant le workflow de chargement de DLL autour du processus cible.
Il est surtout utile aux reverse engineers qui étudient la communication pilote via syscall, l'abus de HalDispatchTable, ainsi que la manipulation de callbacks ou de threads contre Easy Anti-Cheat.
