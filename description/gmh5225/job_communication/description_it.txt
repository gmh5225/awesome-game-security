Questo progetto è una piccola proof of concept per una comunicazione non documentata da ring0 a ring3 tramite NtQueryInformationJobObject.
Le note lato kernel illustrano l'idea di controllare il campo Job del processo corrente, risolvere il server silo del job con PsGetJobServerSilo e copiare i dati da ServerSiloGlobals o UserSharedData nel buffer di output restituito in user mode.
Il campione in user mode chiama poi NtQueryInformationJobObject con JobObjectReserved17Information e interpreta i campi SILO_USER_SHARED_DATA restituiti come risultato della comunicazione nascosta.
È soprattutto utile per i ricercatori di Windows internals che studiano percorsi di comunicazione poco noti basati su job object e silo che aggirano le interfacce IOCTL di dispositivo standard.
