Ce projet est une petite preuve de concept pour une communication non documentée de ring0 vers ring3 via NtQueryInformationJobObject.
Les notes côté noyau présentent l'idée de vérifier le champ Job du processus courant, de résoudre le server silo du job avec PsGetJobServerSilo, puis de copier des données depuis ServerSiloGlobals ou UserSharedData vers le tampon de sortie renvoyé en mode utilisateur.
L'exemple en mode utilisateur appelle ensuite NtQueryInformationJobObject avec JobObjectReserved17Information et interprète les champs SILO_USER_SHARED_DATA renvoyés comme le résultat de cette communication cachée.
Il est surtout utile aux chercheurs en internals Windows qui étudient des chemins de communication obscurs basés sur les job objects et les silos, contournant les interfaces IOCTL de périphérique standard.
