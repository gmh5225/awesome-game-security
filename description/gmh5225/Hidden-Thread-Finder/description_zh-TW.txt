本專案是一個概念驗證偵測器，用於發現被竄改或隱藏的系統執行緒，核心作法是比較在關鍵 KTHREAD 欄位被竄改後，APC 與 NMI 回呼仍能還原出哪些資訊。
示範會建立一個系統執行緒，在 Windows 20H2 特定的 KTHREAD 配置上故意清除 SystemThread、ApcQueueable、StackBase、InitialStack 等欄位，再分別透過排隊的 APC 回呼與已註冊的 NMI 回呼檢查該執行緒。
其記錄程式碼會顯示每種機制是否仍能觀察到堆疊中繼資料或系統執行緒狀態，因此此封存更像是在實驗哪些隱藏執行緒手法會破壞簡單檢查，而不是一個完善的掃描器。
它主要適合反作弊與核心研究人員，用於研究隱藏執行緒偵測、KTHREAD 欄位偽造，以及以 APC 與以 NMI 為基礎的驗證各自的限制。
