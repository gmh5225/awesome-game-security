Ce projet est un détecteur proof-of-concept pour les threads système manipulés ou masqués, centré sur la comparaison de ce que les callbacks APC et NMI peuvent encore récupérer après altération de champs KTHREAD clés.
La démo crée un thread système, efface délibérément des champs tels que SystemThread, ApcQueueable, StackBase et InitialStack sur une disposition KTHREAD spécifique à Windows 20H2, puis inspecte le thread via un callback APC mis en file et un callback NMI enregistré.
Son code de journalisation indique si chaque mécanisme peut encore observer les métadonnées de pile ou l’état de thread système, si bien que l’archive s’apparente davantage à une expérience sur les techniques de masquage de threads qui cassent une inspection simple qu’à un scanner abouti.
Il est surtout utile aux chercheurs en anti-cheat et en noyau qui étudient la détection de threads cachés, le spoofing des champs KTHREAD, et les limites de la validation fondée sur APC versus NMI.
