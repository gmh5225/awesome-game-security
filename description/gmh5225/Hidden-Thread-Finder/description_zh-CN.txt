本项目是一个概念验证检测器，用于发现被篡改或隐藏的系统线程，核心思路是对比在关键 KTHREAD 字段被篡改后，APC 与 NMI 回调仍能恢复出哪些信息。
演示会创建一个系统线程，在 Windows 20H2 特定的 KTHREAD 布局上故意清除 SystemThread、ApcQueueable、StackBase、InitialStack 等字段，再分别通过排队的 APC 回调和已注册的 NMI 回调检查该线程。
其日志代码会显示每种机制是否仍能观察到栈元数据或系统线程状态，因此该归档更像是在实验哪些藏线程手段会破坏简单检查，而不是一个完善的扫描器。
它主要面向反作弊与内核研究人员，用于研究隐藏线程检测、KTHREAD 字段伪造，以及基于 APC 与基于 NMI 的校验各自的边界。
