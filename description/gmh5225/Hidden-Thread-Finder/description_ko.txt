이 프로젝트는 조작되거나 숨겨진 시스템 스레드를 탐지하기 위한 개념 증명(PoC) 탐지기로, 핵심 KTHREAD 필드가 변조된 뒤에도 APC와 NMI 콜백이 어떤 정보를 여전히 복구할 수 있는지를 비교하는 구조로 만들어졌습니다.
데모는 시스템 스레드를 생성한 뒤 Windows 20H2 전용 KTHREAD 레이아웃에서 SystemThread, ApcQueueable, StackBase, InitialStack 같은 필드를 의도적으로 지우고, 대기열에 넣은 APC 콜백과 등록된 NMI 콜백으로 해당 스레드를 검사합니다.
로깅 코드는 각 메커니즘이 스택 메타데이터나 시스템 스레드 상태를 여전히 관찰할 수 있는지를 보여 주므로, 완성된 스캐너라기보다 어떤 스레드 은닉 기법이 단순한 검사를 깨뜨리는지를 실험하는 성격이 강합니다.
주로 안티치트 및 커널 연구자가 숨겨진 스레드 탐지, KTHREAD 필드 스푸핑, APC 기반 검증과 NMI 기반 검증의 한계를 연구할 때 유용합니다.
