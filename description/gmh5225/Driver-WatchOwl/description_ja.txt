本プロジェクトは、ユーザーモードのイメージ読み込みとスレッド作成を監視し、スタックトレース検査によって不審なマッピング動作を識別する防御的な Windows ドライバです。
実装では `PsSetLoadImageNotifyRoutine` と `PsSetCreateThreadNotifyRoutine` コールバックを登録し、`csrss.exe` から `NtMapViewOfSection` や `RtlUserThreadStart` といった想定されるユーザーモードフレームを解決したうえで、イメージ読み込みコールバックが正当なモジュールのテキスト範囲から発生したかを検証します。
観測されたスタックが想定されるマッピング経路と一致しない場合、ドライバはそのイベントを記録し、署名レベルが低いイメージを強調します。攻撃的なフック集ではなく、コールバックベースのインジェクション検出の簡潔な事例として位置づけられます。
アンチチート技術者や、イメージマッピングのスタックベース検証および不審なユーザーモードコード注入経路を研究する防御的カーネル研究者にとって特に有用です。
