이 프로젝트는 사용자 모드 이미지 로드와 스레드 생성을 감시하고, 스택 트레이스 검사를 통해 의심스러운 매핑 활동을 식별하는 방어용 Windows 드라이버입니다.
구현은 `PsSetLoadImageNotifyRoutine` 및 `PsSetCreateThreadNotifyRoutine` 콜백을 등록하고, `csrss.exe`에서 `NtMapViewOfSection`과 `RtlUserThreadStart` 같은 예상 사용자 모드 프레임을 해석한 뒤, 이미지 로드 콜백이 정상적인 모듈 텍스트 범위에서 비롯되었는지 검증합니다.
관찰된 스택이 예상 매핑 경로와 일치하지 않으면 드라이버는 해당 이벤트를 기록하고 서명 수준이 낮은 이미지를 강조합니다. 공격용 훅 모음이 아니라 콜백 기반 인젝션 탐지의 간결한 예제입니다.
안티치트 엔지니어와, 이미지 매핑의 스택 기반 검증 및 의심스러운 사용자 모드 코드 인젝션 경로를 연구하는 방어적 커널 연구자에게 특히 유용합니다.
