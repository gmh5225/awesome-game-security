Ce dépôt combine un environnement EDK II adapté à Visual Studio et MSBuild avec un projet de pilote UEFI `rainbow` distinct, ainsi que des ressources de débogage telles que des images OVMF, une ISO de shell UEFI et des fichiers ROM pour les tests locaux.
Selon le README archivé, la charge `rainbow.efi` accroche `ExitBootServices`, parcourt les structures de démarrage Windows précoces comme `OslLoaderBlock`, puis accroche `IopLoadDriver` pour appliquer une logique de spoofing avant de retirer son propre hook.
La structure du projet incluse présente à la fois le code du pilote en phase de démarrage et la couche de bibliothèque UEFI associée nécessaire pour construire et déboguer le composant EFI, plutôt qu’un simple dépôt de binaire autonome.
Il s’adresse surtout aux chercheurs en firmware et au démarrage Windows qui souhaitent étudier comment les pilotes UEFI sont construits, chargés depuis un shell EFI, et utilisés pour modifier le comportement lors de la transition vers le noyau Windows.
