Este repositorio combina un entorno EDK II amigable con Visual Studio y MSBuild con un proyecto de controlador UEFI `rainbow` independiente y recursos de depuración como imágenes OVMF, una ISO del shell UEFI y archivos ROM para pruebas locales.
Según el README archivado, la carga `rainbow.efi` engancha `ExitBootServices`, recorre estructuras tempranas de arranque de Windows como `OslLoaderBlock` y luego engancha `IopLoadDriver` para aplicar lógica de spoofing antes de eliminar su propio hook.
La estructura del proyecto incluida muestra tanto el código del controlador en la fase de arranque como la capa de biblioteca UEFI de apoyo necesaria para compilar y depurar el componente EFI, no solo un binario autónomo.
Resulta especialmente útil para investigadores de firmware y del arranque de Windows que quieran estudiar cómo se construyen los controladores UEFI, cómo se cargan desde un shell EFI y cómo se usan para alterar el comportamiento durante la transición al kernel de Windows.
