Questo repository combina un ambiente EDK II adatto a Visual Studio e MSBuild con un progetto separato di driver UEFI `rainbow` e risorse di debug come immagini OVMF, un’ISO della shell UEFI e file ROM per i test locali.
Secondo il README archiviato, il payload `rainbow.efi` aggancia `ExitBootServices`, attraversa strutture di boot Windows precoci come `OslLoaderBlock`, quindi aggancia `IopLoadDriver` per applicare la logica di spoofing prima di rimuovere il proprio hook.
La struttura del progetto inclusa mostra sia il codice del driver in fase di boot sia lo strato di libreria UEFI di supporto necessario per compilare e debuggare il componente EFI, non solo un binario autonomo.
È utile soprattutto a ricercatori di firmware e del boot di Windows che vogliono studiare come i driver UEFI vengono costruiti, caricati da una shell EFI e usati per alterare il comportamento durante la transizione nel kernel di Windows.
