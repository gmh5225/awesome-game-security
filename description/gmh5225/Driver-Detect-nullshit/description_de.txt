Dieses Projekt ist ein kompakter Detektor für einen bestimmten Stil manuell gemappter Treiber-Hooks, der exportierte Kernel-Routinen durch einfache absolute Jump-Stubs ersetzt.
Die archivierte README bezeichnet ihn als einfachen Null-Treiber-Detektor; der Code durchläuft die Exporttabellen von Modulen wie `win32kfull.sys`, `win32kbase.sys`, `dxgkrnl.sys` und `ntoskrnl.exe` und sucht nach Patches im Stil `mov rax, imm64 ; jmp rax`, deren Ziele außerhalb des eigenen Images liegen.
Statt ein vollständiges Integritäts-Framework bereitzustellen, konzentriert er sich auf eine gängige Inline-Hook-Signatur, die oft genutzt wird, um syscall-bezogene Funktionen beim Treiber-Mapping oder beim Aufbau der Kommunikation umzuleiten.
Er ist vor allem für Anti-Cheat- und defensive Kernel-Forscher nützlich, die die leichtgewichtige Erkennung grober Export-Hook-Trampoline in zentralen Windows-Modulen untersuchen.
