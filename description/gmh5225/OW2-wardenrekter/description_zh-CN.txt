一个在 DLL_PROCESS_ATTACH 时通过修补多个检测向量来禁用《守望先锋 2》Warden 反作弊保护的 DLL：它用 RET（0xC3）覆盖 KiUserExceptionDispatcher 以破坏 Warden 基于 VEH 的 INT3 hook 监控，对 DbgBreakPoint/DbgUserBreakPoint 完整性检查进行 NOP，清零 PEB.BeingDebugged 与 NtGlobalFlag 以隐藏调试器存在，桩化 GetTickCount64 以绕过计时检测，并修补 NtQuerySystemInformation 以阻止系统信息查询。
它还会通过 SetThreadContext 设置硬件调试寄存器 DR0，以演示针对 Warden 硬件断点检测的调试寄存器操控。
该项目主要面向研究 Warden 反作弊绕过技术的游戏安全研究者，涵盖 VEH hook 移除、PEB 伪造以及《守望先锋 2》中的 API 修补等。
