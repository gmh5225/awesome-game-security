Questo repository è un elenco di riferimento compatto dei percorsi di chiamata da user-mode a kernel-mode, utile per studiare come i punti di ingresso grafici e UI di Windows fluiscono nei componenti kernel di livello inferiore.
Il README archiviato si concentra su esempi concreti, come le chiamate a `win32u.dll` che si risolvono tramite `win32k.sys` e `win32base.sys` prima di raggiungere target in `dxgkrnl.sys`, risultando più un taccuino di mappatura che un tradizionale progetto di codice.
Il valore del repo sta negli esempi di catene di chiamata stessi, che forniscono tracce rapide per seguire i percorsi di comunicazione da analizzare poi nel debugger o nel disassemblatore.
È soprattutto utile per ricercatori di kernel Windows e reverse engineering che cercano un punto di partenza per indagare percorsi di syscall e superfici di comunicazione dei driver.
