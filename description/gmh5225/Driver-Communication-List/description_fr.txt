Ce dépôt est une liste de référence compacte des chemins d’appel du mode utilisateur vers le mode noyau, utile pour étudier comment les points d’entrée graphiques et UI de Windows aboutissent aux composants noyau de bas niveau.
Son README archivé se concentre sur des exemples concrets, comme les appels à `win32u.dll` qui se résolvent via `win32k.sys` et `win32base.sys` avant d’atteindre des cibles dans `dxgkrnl.sys`, ce qui en fait davantage un carnet de cartographie qu’un projet de code classique.
La valeur du dépôt réside dans les exemples de chaînes d’appels eux-mêmes, qui fournissent des indices rapides pour tracer les chemins de communication, ensuite analysables dans un débogueur ou un désassembleur.
Il s’adresse surtout aux chercheurs en noyau Windows et en rétro-ingénierie qui cherchent un point de départ pour explorer les chemins de syscalls et les surfaces de communication des pilotes.
