Este proyecto es un paquete Microsoft EDK2 para implementar Offline Crash Dump, en el que el firmware escribe un volcado de memoria antes o en lugar del sistema operativo.
El repositorio incluye encabezados compartidos para GUID y estructuras relacionadas con dumps, utilidades `OfflineDumpLib` para localizar particiones de dump y leer variables UEFI definidas por Windows, y código `OfflineDumpWriterLib` para generar dumps con almacenamiento en búfer, cifrado y redacción.
También incluye aplicaciones UEFI redistribuibles y de ejemplo como `OfflineDumpWrite.efi`, componentes de benchmark y prueba, y un diseño completo de paquete pensado para ejecutarse en un entorno UEFI DXE, sirviendo a la vez de referencia para otras pilas de firmware.
El README archivado deja claro que el código está pensado para escenarios de bring-up, depuración y estabilización de dispositivos, no para implementaciones comerciales, por lo que resulta más útil para ingenieros de firmware y plataforma que estudian cómo se estructura la recolección de fallos pre-OS en sistemas compatibles con Windows.
