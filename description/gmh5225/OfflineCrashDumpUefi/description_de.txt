Dieses Projekt ist ein Microsoft-EDK2-Paket zur Implementierung von Offline Crash Dump, bei dem die Firmware einen Speicherdump vor oder anstelle des Betriebssystems schreibt.
Das Repository enthält gemeinsame Header für dumpbezogene GUIDs und Strukturen, `OfflineDumpLib`-Hilfen zum Auffinden von Dump-Partitionen und zum Lesen von Windows-definierten UEFI-Variablen sowie `OfflineDumpWriterLib`-Code zum Erzeugen von Dumps mit Pufferung, Verschlüsselung und Redaktion.
Außerdem liefert es redistribuierbare und Beispiel-UEFI-Anwendungen wie `OfflineDumpWrite.efi`, Benchmark- und Testkomponenten sowie ein vollständiges Paketlayout, das für eine UEFI-DXE-Umgebung gedacht ist und zugleich als Referenz für andere Firmware-Stacks dienen kann.
Die archivierte README macht deutlich, dass der Code für Bring-up, Debugging und Gerätestabilisierung gedacht ist und nicht für den Einzelhandel; er ist daher besonders nützlich für Firmware- und Plattformingenieure, die untersuchen, wie die Pre-OS-Crash-Erfassung auf Windows-fähigen Systemen aufgebaut ist.
