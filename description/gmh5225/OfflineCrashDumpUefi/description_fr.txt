Ce projet est un package Microsoft EDK2 pour la prise en charge d’Offline Crash Dump, dans lequel le firmware écrit une image mémoire avant ou à la place du système d’exploitation.
Le dépôt contient des en-têtes partagés pour les GUID et structures liés aux dumps, des helpers `OfflineDumpLib` pour localiser les partitions de dump et lire les variables UEFI définies par Windows, ainsi que le code `OfflineDumpWriterLib` pour générer des dumps avec mise en mémoire tampon, chiffrement et rédaction.
Il fournit aussi des applications UEFI redistribuables et d’exemple telles que `OfflineDumpWrite.efi`, des composants de benchmark et de test, et une disposition complète de package destinée à s’exécuter dans un environnement UEFI DXE tout en servant de référence pour d’autres piles firmware.
Le README archivé précise que le code vise le bring-up, le débogage et la stabilisation des appareils plutôt que les déploiements grand public, ce qui le rend surtout utile aux ingénieurs firmware et plateforme qui étudient la structure de la collecte de crash pré-OS sur les systèmes compatibles Windows.
