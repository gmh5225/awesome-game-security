Questo progetto è una proof of concept di dump di LSASS in user mode che combina il furto di handle con una copia non hookata di DbgHelp e un percorso di dump leggermente offuscato.
Il README archiviato descrive l’approccio come la duplicazione di un handle di processo LSASS esistente da un altro processo, seguita dalla chiamata a MiniDumpWriteDump tramite una nuova copia di DbgHelp.dll caricata da disco, invece di affidarsi a export eventualmente hookati.
Il codice enumera SystemHandleInformation, duplica gli handle di processo candidati, li filtra in base al percorso completo dell’immagine che contiene lsass.exe, scrive il dump in un file temporaneo e poi cifra il file risultante su disco.
È soprattutto utile per i ricercatori di sicurezza Windows che studiano l’acquisizione di dump basata su handle, l’evasione degli hook in user mode intorno a MiniDumpWriteDump e i compromessi degli handle privilegiati riciclati.
