Этот проект — proof of concept дампа LSASS в user-mode, сочетающий кражу handle с неперехваченной копией DbgHelp и слегка обфусцированным путём дампа.
В архивированном README подход описан так: дублируется существующий handle процесса LSASS из другого процесса, после чего MiniDumpWriteDump вызывается через свежую копию DbgHelp.dll, загруженную с диска, вместо опоры на, возможно, перехваченные экспорты.
Код перечисляет SystemHandleInformation, дублирует кандидатные process handle, отфильтровывает их по полному пути образа, содержащему lsass.exe, записывает дамп во временный файл и затем шифрует получившийся файл на диске.
В основном полезен исследователям безопасности Windows, изучающим получение дампов через handle, обход user-mode хуков вокруг MiniDumpWriteDump и компромиссы повторного использования привилегированных handle.
