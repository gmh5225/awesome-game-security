Ce projet est une preuve de concept de dump LSASS en mode utilisateur qui combine le vol de handles, une copie non hookée de DbgHelp et un chemin de dump légèrement obfuscqué.
Le README archivé décrit l’approche comme la duplication d’un handle de processus LSASS existant depuis un autre processus, puis l’appel à MiniDumpWriteDump via une nouvelle copie de DbgHelp.dll chargée depuis le disque, plutôt que de s’appuyer sur des exports éventuellement hookés.
Son code énumère SystemHandleInformation, duplique les handles de processus candidats, les filtre selon le chemin d’image complet contenant lsass.exe, écrit le dump dans un fichier temporaire, puis chiffre le fichier résultant sur le disque.
Il est surtout utile aux chercheurs en sécurité Windows qui étudient l’acquisition de dumps basée sur les handles, le contournement des hooks en mode utilisateur autour de MiniDumpWriteDump, et les compromis liés aux handles privilégiés recyclés.
