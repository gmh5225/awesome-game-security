Dieses Projekt ist ein User-Mode-LSASS-Dumping-Proof-of-Concept, das Handle-Diebstahl mit einer unhooked Kopie von DbgHelp und einem leicht verschleierten Dump-Pfad kombiniert.
Die archivierte README beschreibt den Ansatz so: ein vorhandenes LSASS-Prozesshandle aus einem anderen Prozess wird dupliziert und MiniDumpWriteDump anschließend über eine frische, von der Festplatte geladene DbgHelp.dll-Kopie aufgerufen, statt sich auf möglicherweise gehookte Exporte zu verlassen.
Der Code enumeriert SystemHandleInformation, dupliziert Kandidaten-Prozesshandles, filtert sie anhand des vollständigen Image-Pfads mit lsass.exe, schreibt den Dump in eine temporäre Datei und verschlüsselt anschließend die resultierende Datei auf dem Datenträger.
Es ist vor allem für Windows-Sicherheitsforscher nützlich, die handlebasierte Dump-Beschaffung, User-Mode-Hook-Umgehung rund um MiniDumpWriteDump und die Kompromisse wiederverwendeter privilegierter Handles untersuchen.
