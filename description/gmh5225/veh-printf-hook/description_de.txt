Dieses Projekt zeigt, wie Windows Vectored Exception Handling (VEH) genutzt werden kann, um printf und ähnliche Ausgabefunktionen zu hooken. Es setzt Page Guards auf den Speicher der Zielfunktionen, fängt die resultierenden Exceptions in einem VEH-Handler ab und leitet die Ausführung an benutzerdefinierten Logging- oder Filtercode um. Diese nicht-invasive Hooking-Technik vermeidet das Patchen von Funktionsbytes. Sie richtet sich an Sicherheitsforscher, die VEH-basierte Funktionsabfangtechniken untersuchen.
