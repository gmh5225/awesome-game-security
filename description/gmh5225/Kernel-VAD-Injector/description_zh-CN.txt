本项目是一个借助未签名驱动的 DLL 注入器，通过操纵 VAD 与 PTE 来隐藏其暂存区域，而不是仅依赖普通的用户态内存分配。
归档 README 称，该驱动通过 PatchGuard 安全的 xKdEnumerateDebuggingDevices 钩子进行通信，随后在内核中截获用户线程并运行命令循环，处理读、写、分配内存、伪造 PTE、分配 VAD 以及移除 VAD 等请求。
内部通过特征码在 ntoskrnl 中定位 MiAllocateVad、MiInsertVad 与 MiInsertVadCharges，在内存操作周围使用 KeStackAttachProcess，并在映射完成后再次移除 VAD 节点，使注入区域更不易被 NtQueryVirtualMemory 等检查发现。
它主要面向研究 Windows 内核的人员，用于学习结合 VAD 树滥用的手动映射、可执行页隐藏，以及基于驱动的注入后清理。
