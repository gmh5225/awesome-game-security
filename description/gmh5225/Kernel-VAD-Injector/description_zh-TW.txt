本專案是一個借助未簽署驅動程式的 DLL 注入器，透過操縱 VAD 與 PTE 來隱藏其暫存區域，而非僅依賴一般的使用者模式記憶體配置。
歸檔 README 指出，該驅動程式透過對 PatchGuard 安全的 xKdEnumerateDebuggingDevices 掛鉤進行通訊，接著在核心中截獲使用者執行緒並執行命令迴圈，處理讀取、寫入、配置記憶體、偽造 PTE、配置 VAD 以及移除 VAD 等請求。
內部透過特徵碼在 ntoskrnl 中定位 MiAllocateVad、MiInsertVad 與 MiInsertVadCharges，在記憶體操作周圍使用 KeStackAttachProcess，並在對應完成後再次移除 VAD 節點，使注入區域較不易被 NtQueryVirtualMemory 等檢查發現。
它主要適合研究 Windows 核心的人員，用於學習結合 VAD 樹濫用的手動對應、可執行頁隱藏，以及基於驅動程式的注入後清理。
