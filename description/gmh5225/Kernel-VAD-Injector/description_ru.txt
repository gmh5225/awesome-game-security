Этот проект — DLL-инжектор с помощью неподписанного драйвера, который пытается скрыть область подготовки, манипулируя VAD и PTE, а не полагаясь только на обычные выделения памяти в пользовательском режиме.
В архивном README сказано, что драйвер общается через безопасный для PatchGuard хук xKdEnumerateDebuggingDevices, затем перехватывает пользовательский поток в ядре и запускает цикл команд для запросов чтения, записи, выделения памяти, подмены PTE, выделения VAD и удаления VAD.
Внутри он по сигнатурам находит MiAllocateVad, MiInsertVad и MiInsertVadCharges в ntoskrnl, использует KeStackAttachProcess вокруг операций с памятью и снова удаляет узел VAD после отображения, чтобы внедрённая область была менее заметна для NtQueryVirtualMemory и подобных проверок.
Он в основном полезен исследователям ядра Windows, изучающим manual mapping со злоупотреблением деревом VAD, сокрытие исполняемых страниц и очистку после внедрения на стороне драйвера.
