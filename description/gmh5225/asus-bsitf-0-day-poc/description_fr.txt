Ce projet est une preuve de concept en Rust qui démontre une vulnérabilité de mapping mémoire noyau dans le pilote ASUS bsitf.sys / AsusBSItf.sys (CVE-2026-13585). Il ouvre le périphérique \\.\bsitf et utilise l’IOCTL 0x222808 pour allouer une mémoire de pool noyau physiquement contiguë de taille choisie par l’attaquant, la mapper dans le processus appelant avec un accès usermode en lecture/écriture complet, et renvoyer les adresses virtuelle et physique. Le PoC peut vérifier le mapping en écrivant un motif de test, puis le libérer via l’IOCTL 0x22280C, en s’appuyant sur l’API Windows via le crate windows. Il s’agit d’une élévation admin-vers-noyau utile dans les scénarios Bring Your Own Vulnerable Driver (BYOVD), avec des impacts tels que l’épuisement du NonPagedPool, la divulgation d’adresses physiques et, sur les builds plus anciennes, le staging de mémoire noyau exécutable. Il est destiné aux chercheurs en sécurité autorisés qui étudient les pilotes signés vulnérables, l’abus d’IOCTL noyau et les techniques d’élévation de privilèges Windows.
