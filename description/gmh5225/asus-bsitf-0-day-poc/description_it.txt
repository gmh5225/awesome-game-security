Questo progetto è una proof-of-concept in Rust che dimostra una vulnerabilità di mapping della memoria kernel nel driver ASUS bsitf.sys / AsusBSItf.sys (CVE-2026-13585). Apre il dispositivo \\.\bsitf e usa l’IOCTL 0x222808 per allocare memoria di pool kernel fisicamente contigua di dimensione scelta dall’attaccante, mapparla nel processo chiamante con pieno accesso usermode in lettura/scrittura e restituire sia l’indirizzo virtuale sia quello fisico. Il PoC può verificare il mapping scrivendo un pattern di test, poi liberarlo tramite l’IOCTL 0x22280C, e si basa sulle API Windows tramite il crate windows. Il problema è un’escalation da admin a kernel utile negli scenari Bring Your Own Vulnerable Driver (BYOVD), con impatti quali l’esaurimento del NonPagedPool, la divulgazione di indirizzi fisici e, su build più vecchie, lo staging di memoria kernel eseguibile. È destinato a ricercatori di sicurezza autorizzati che studiano driver firmati vulnerabili, abuso di IOCTL kernel e tecniche di privilege escalation su Windows.
