Este proyecto es una prueba de concepto en Rust que demuestra una vulnerabilidad de mapeo de memoria del kernel en el controlador ASUS bsitf.sys / AsusBSItf.sys (CVE-2026-13585). Abre el dispositivo \\.\bsitf y usa el IOCTL 0x222808 para asignar memoria de pool del kernel físicamente contigua del tamaño elegido por el atacante, mapearla en el proceso llamante con acceso completo de lectura/escritura en usermode y devolver las direcciones virtual y física. El PoC puede verificar el mapeo escribiendo un patrón de prueba, liberarlo después mediante el IOCTL 0x22280C y se apoya en la API de Windows a través del crate windows. El problema es una escalada de administrador a kernel útil en escenarios Bring Your Own Vulnerable Driver (BYOVD), con impactos como el agotamiento de NonPagedPool, la divulgación de direcciones físicas y, en builds antiguos, el staging de memoria de kernel ejecutable. Está pensado para investigadores de seguridad autorizados que estudian controladores firmados vulnerables, abuso de IOCTL del kernel y técnicas de elevación de privilegios en Windows.
