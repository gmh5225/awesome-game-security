本プロジェクトは、ASUS の bsitf.sys / AsusBSItf.sys ドライバにおけるカーネルメモリマッピング脆弱性（CVE-2026-13585）を示す Rust 製の概念実証（PoC）です。\\.\bsitf デバイスを開き、IOCTL 0x222808 を用いて攻撃者が指定したサイズの物理連続カーネルプールメモリを割り当て、呼び出し元プロセスへユーザモードの完全な読み書き権限でマップし、仮想アドレスと物理アドレスの両方を返します。PoC はテストパターンを書き込んでマップを検証し、IOCTL 0x22280C で解放でき、windows crate 経由で Windows API を利用します。この問題は admin からカーネルへの権限昇格であり、Bring Your Own Vulnerable Driver（BYOVD）シナリオで有用で、NonPagedPool の枯渇、物理アドレスの漏洩、古いビルドでは実行可能なカーネルメモリの用意といった影響があります。脆弱な署名済みドライバ、カーネル IOCTL の悪用、Windows の権限昇格手法を研究する認可されたセキュリティ研究者向けです。
