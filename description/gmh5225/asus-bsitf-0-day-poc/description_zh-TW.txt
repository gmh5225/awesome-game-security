本專案是一個 Rust 概念驗證（PoC），用於展示 ASUS bsitf.sys / AsusBSItf.sys 驅動程式中的核心記憶體對應漏洞（CVE-2026-13585）。它會開啟 \\.\bsitf 裝置，並透過 IOCTL 0x222808 配置攻擊者指定大小的實體連續核心集區記憶體，以完整使用者模式讀寫權限對應至呼叫行程，並回傳虛擬位址與實體位址。此 PoC 可藉由寫入測試樣式驗證對應，再經 IOCTL 0x22280C 釋放，並透過 windows crate 呼叫 Windows API。此問題屬於系統管理員到核心的權限提升，適用於自帶漏洞驅動（BYOVD）情境，影響包括 NonPagedPool 耗盡、實體位址洩漏，以及在較舊組建上可用於可執行核心記憶體暫存。專案供經授權的安全研究人員研究有漏洞的簽章驅動、核心 IOCTL 濫用與 Windows 權限提升技術。
