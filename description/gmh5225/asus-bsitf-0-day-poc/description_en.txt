This project is a Rust proof-of-concept that demonstrates a kernel memory mapping vulnerability in the ASUS bsitf.sys / AsusBSItf.sys driver (CVE-2026-13585). It opens the \\.\bsitf device and uses IOCTL 0x222808 to allocate attacker-sized physically contiguous kernel pool memory, map it into the calling process with full usermode read/write access, and return both the virtual and physical addresses. The PoC can verify the mapping by writing a test pattern, then free it via IOCTL 0x22280C, and relies on the Windows API through the windows crate. The issue is an admin-to-kernel escalation useful in Bring Your Own Vulnerable Driver (BYOVD) scenarios, with impacts such as NonPagedPool exhaustion, physical address disclosure, and on older builds executable kernel memory staging. It is intended for authorized security researchers studying vulnerable signed drivers, kernel IOCTL abuse, and Windows privilege-escalation techniques.
