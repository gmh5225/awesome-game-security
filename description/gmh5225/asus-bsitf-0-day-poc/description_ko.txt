이 프로젝트는 ASUS bsitf.sys / AsusBSItf.sys 드라이버의 커널 메모리 매핑 취약점(CVE-2026-13585)을 보여주는 Rust 개념 증명(PoC)입니다. \\.\bsitf 장치를 열고 IOCTL 0x222808을 사용해 공격자가 지정한 크기의 물리적으로 연속된 커널 풀 메모리를 할당하며, 호출 프로세스에 전체 사용자 모드 읽기/쓰기 권한으로 매핑하고 가상 주소와 물리 주소를 모두 반환합니다. PoC는 테스트 패턴을 기록해 매핑을 검증한 뒤 IOCTL 0x22280C로 해제할 수 있으며, windows crate를 통해 Windows API를 사용합니다. 이 문제는 관리자에서 커널로의 권한 상승으로, Bring Your Own Vulnerable Driver(BYOVD) 시나리오에 유용하며 NonPagedPool 고갈, 물리 주소 노출, 구형 빌드에서의 실행 가능 커널 메모리 스테이징 등의 영향을 가집니다. 취약한 서명 드라이버, 커널 IOCTL 남용, Windows 권한 상승 기법을 연구하는 인가된 보안 연구자를 위한 자료입니다.
