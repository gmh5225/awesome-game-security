Este proyecto es un ejemplo antiguo de controlador de trampas que usa un intercambio de puntero de datos en win32kbase para la comunicación, en lugar de exponer una interfaz de dispositivo convencional.
El controlador busca en win32kbase el puntero objetivo, se adjunta a explorer.exe, lo sustituye por un manejador personalizado de NtSetCompositionSurfaceAnalogExclusive mediante InterlockedExchangePointer y enruta las peticiones a través de estructuras enviadas desde el modo de usuario.
Además del primitivo de comunicación, el código también incluye utilidades de limpieza para rastros como MmUnloadedDrivers y residuos relacionados del cargador, lo que hace del repositorio parte demo de comunicaciones y parte ayuda anti-forense.
Resulta especialmente útil para investigadores del kernel de Windows que estudian la comunicación por intercambio de puntero de datos, la colocación de hooks basados en win32k y la lógica de limpieza usada junto a controladores de trampas mapeados manualmente.
