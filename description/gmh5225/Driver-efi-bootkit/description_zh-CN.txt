本项目是一个 UEFI 启动阶段植入框架，通过分阶段载荷链，将执行流从被感染的 EFI 应用转移到 Windows 内核。
其 C 代码挂钩 ExitBootServices 与 SetVirtualAddressMap，跟踪映像的运行时虚拟地址，并修补 OslArchTransferToKernel，以便在操作系统完全启动前修改目标驱动。
内核侧阶段会定位选定的驱动映像，将其 .rsrc 节重用作可执行空间，通过 MmMapIoSpace 映射额外载荷，更新驱动入口点，最后恢复原始入口路径以降低启动不稳定风险。
仓库还包含 Python 工具，用于提取扁平 shellcode 二进制块、计算哈希标识符，以及将 BOOTDOOR 载荷注入 EFI 二进制，并可选择修补 bootmgfw 完整性检查。
它主要面向需要研究 EFI 挂钩、加载器拦截与预操作系统驱动修补实际实现方式的底层 Windows 启动、固件与内核研究人员。
