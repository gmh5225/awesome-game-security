이 프로젝트는 감염된 EFI 애플리케이션에서 단계적 페이로드 체인을 통해 Windows 커널로 실행을 넘기는 UEFI 부트 단계 임플란트 프레임워크입니다.
C 코드는 ExitBootServices와 SetVirtualAddressMap을 후킹하고 이미지의 런타임 가상 주소를 추적한 뒤 OslArchTransferToKernel을 패치하여, 운영체제가 완전히 시작되기 전에 대상 드라이버를 수정할 수 있게 합니다.
커널 측 스테이지는 선택한 드라이버 이미지를 찾아 .rsrc 섹션을 실행 가능 공간으로 재사용하고, MmMapIoSpace로 추가 페이로드를 매핑한 뒤 드라이버 진입점을 갱신하며, 마지막으로 원래 진입 경로를 복원해 부트 불안정성을 줄입니다.
저장소에는 평면 shellcode 블롭 추출, 해시 식별자 계산, BOOTDOOR 페이로드를 EFI 바이너리에 주입하고 선택적으로 bootmgfw 무결성 검사를 패치하는 Python 도구도 포함됩니다.
EFI 후킹, 로더 가로채기, OS 이전 드라이버 패치가 실제로 어떻게 구현되는지 연구해야 하는 저수준 Windows 부트·펌웨어·커널 연구자에게 주로 유용합니다.
