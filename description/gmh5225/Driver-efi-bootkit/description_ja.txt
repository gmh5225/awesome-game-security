本プロジェクトは、感染した EFI アプリケーションから段階的ペイロード連鎖を通じて Windows カーネルへ実行を移す UEFI ブート段階インプラントフレームワークです。
C コードは ExitBootServices と SetVirtualAddressMap をフックし、イメージのランタイム仮想アドレスを追跡したうえで OslArchTransferToKernel をパッチし、OS が完全に起動する前に対象ドライバを改変できるようにします。
カーネル側ステージは選択したドライバイメージを特定し、その .rsrc セクションを実行可能領域として再利用し、MmMapIoSpace で追加ペイロードをマップしてドライバのエントリポイントを更新し、最後に元のエントリ経路を復元してブート不安定性を抑えます。
リポジトリには、フラットなシェルコード Blob の抽出、ハッシュ識別子の計算、BOOTDOOR ペイロードの EFI バイナリへの注入、および任意で bootmgfw の整合性チェックをパッチする Python ツールも含まれます。
主に EFI フック、ローダー傍受、OS 起動前のドライバパッチが実際にどう実装されるかを研究する、低レベルな Windows ブート・ファームウェア・カーネル研究者向けです。
