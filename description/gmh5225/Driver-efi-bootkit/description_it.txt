Questo progetto è un framework di implant nella fase di boot UEFI che trasferisce l'esecuzione da un'applicazione EFI infetta al kernel di Windows tramite una catena di payload a stadi.
Il suo codice C aggancia ExitBootServices e SetVirtualAddressMap, tiene traccia dell'indirizzo virtuale runtime dell'immagine e quindi applica un patch a OslArchTransferToKernel per modificare un driver target prima che il sistema operativo si avvii completamente.
Lo stadio lato kernel individua l'immagine del driver scelto, riutilizza la sua sezione .rsrc come spazio eseguibile, mappa un payload aggiuntivo con MmMapIoSpace, aggiorna il punto di ingresso del driver e infine ripristina il percorso di ingresso originale per ridurre l'instabilità all'avvio.
Il repository include anche strumenti Python per estrarre il blob di shellcode flat, calcolare identificatori hash e iniettare il payload BOOTDOOR in un binario EFI, con un patch opzionale dei controlli di integrità di bootmgfw.
È utile soprattutto a ricercatori di basso livello su boot Windows, firmware e kernel che devono studiare come hook EFI, intercettazione del loader e patching di driver pre-OS vengono implementati in pratica.
