Ce projet est un framework d'implant au stade de démarrage UEFI qui transfère l'exécution d'une application EFI infectée vers le noyau Windows via une chaîne de charges utiles par étapes.
Son code C accroche ExitBootServices et SetVirtualAddressMap, suit l'adresse virtuelle d'exécution de l'image, puis patch OslArchTransferToKernel afin de modifier un pilote cible avant que le système d'exploitation ne démarre complètement.
L'étape côté noyau localise l'image du pilote choisi, réutilise sa section .rsrc comme espace exécutable, mappe une charge utile supplémentaire avec MmMapIoSpace, met à jour le point d'entrée du pilote, puis restaure le chemin d'entrée d'origine pour limiter l'instabilité au démarrage.
Le dépôt inclut aussi des outils Python pour extraire le blob shellcode plat, calculer des identifiants hachés et injecter la charge utile BOOTDOOR dans un binaire EFI, avec un patch optionnel des contrôles d'intégrité de bootmgfw.
Il s'adresse surtout aux chercheurs bas niveau sur le démarrage Windows, le firmware et le noyau qui doivent étudier comment les hooks EFI, l'interception du chargeur et le patch de pilotes pré-OS sont mis en œuvre en pratique.
