Este proyecto es un framework de implante en la fase de arranque UEFI que transfiere la ejecución desde una aplicación EFI infectada al kernel de Windows mediante una cadena de cargas útiles por etapas.
Su código en C engancha ExitBootServices y SetVirtualAddressMap, rastrea la dirección virtual de tiempo de ejecución de la imagen y luego parchea OslArchTransferToKernel para poder modificar un controlador objetivo antes de que el sistema operativo arranque por completo.
La etapa del lado del kernel localiza la imagen del controlador elegido, reutiliza su sección .rsrc como espacio ejecutable, mapea una carga útil adicional con MmMapIoSpace, actualiza el punto de entrada del controlador y finalmente restaura la ruta de entrada original para reducir la inestabilidad del arranque.
El repositorio también incluye herramientas en Python para extraer el blob de shellcode plano, calcular identificadores con hash e inyectar la carga útil BOOTDOOR en un binario EFI, con un parche opcional de las comprobaciones de integridad de bootmgfw.
Resulta útil principalmente para investigadores de bajo nivel del arranque de Windows, firmware y kernel que necesitan estudiar cómo se implementan en la práctica los hooks EFI, la intercepción del cargador y el parcheo de controladores pre-OS.
