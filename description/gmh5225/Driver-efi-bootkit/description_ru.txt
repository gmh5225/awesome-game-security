Этот проект — фреймворк импланта на стадии загрузки UEFI, который передаёт выполнение из заражённого EFI-приложения в ядро Windows через цепочку поэтапных полезных нагрузок.
Его код на C перехватывает ExitBootServices и SetVirtualAddressMap, отслеживает runtime-виртуальный адрес образа, затем патчит OslArchTransferToKernel, чтобы изменить целевой драйвер до полного запуска операционной системы.
Ядерная стадия находит выбранный образ драйвера, повторно использует его секцию .rsrc как исполняемое пространство, отображает дополнительную нагрузку через MmMapIoSpace, обновляет точку входа драйвера и в итоге восстанавливает исходный путь входа, чтобы снизить нестабильность загрузки.
В репозитории также есть инструменты на Python для извлечения плоского blob shellcode, вычисления хешированных идентификаторов и внедрения нагрузки BOOTDOOR в EFI-бинарник с опциональным патчем проверок целостности bootmgfw.
Он в первую очередь полезен исследователям низкоуровневой загрузки Windows, прошивки и ядра, которым нужно изучать, как на практике реализуются EFI-хуки, перехват загрузчика и патчинг драйверов до старта ОС.
