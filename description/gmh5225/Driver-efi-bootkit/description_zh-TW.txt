本專案是一個 UEFI 啟動階段植入框架，透過分階段載荷鏈，將執行流程從受感染的 EFI 應用程式轉移到 Windows 核心。
其 C 程式碼會掛鉤 ExitBootServices 與 SetVirtualAddressMap，追蹤映像的執行時期虛擬位址，並修補 OslArchTransferToKernel，以便在作業系統完全啟動前修改目標驅動程式。
核心端階段會定位選定的驅動程式映像，將其 .rsrc 區段轉用作可執行空間，透過 MmMapIoSpace 對應額外載荷，更新驅動程式進入點，最後還原原始進入路徑以降低啟動不穩定風險。
儲存庫亦包含 Python 工具，用於擷取扁平 shellcode 二進位區塊、計算雜湊識別碼，以及將 BOOTDOOR 載荷注入 EFI 二進位檔，並可選擇修補 bootmgfw 完整性檢查。
它主要供需要研究 EFI 掛鉤、載入器攔截與預作業系統驅動程式修補實際實作方式的底層 Windows 啟動、韌體與核心研究人員使用。
