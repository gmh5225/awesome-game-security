Este proyecto es una prueba de concepto para enganchar `nt!HvcallCodeVa`, el puntero de página de hypercall usado por las transiciones de hypercall del kernel relacionadas con Hyper-V.
El controlador localiza la referencia a la página de hypercall cerca de `HvlInvokeHypercall`, la sustituye por un dispatcher personalizado y cambia el indicador `HvlEnlightenments` para que hypercalls relacionados con el cambio de contexto, como `HvlSwitchVirtualAddressSpace`, se enruten a través del hook.
Su implementación se divide entre una capa de entrada del controlador en C++ y un dispatcher en ensamblador que inspecciona el código de entrada del hypercall, reenvía las operaciones objetivo a callbacks personalizados y recurre a la página de hypercall original para todo lo demás.
Es útil principalmente para investigadores de bajo nivel de Windows y virtualización que quieren estudiar cómo se puede interceptar el dispatch de hypercalls de Hyper-V dentro del kernel para trazado o experimentación.
