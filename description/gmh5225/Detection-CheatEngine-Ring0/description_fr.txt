Ce projet est une petite preuve de concept noyau qui tente de détecter une activité liée à Cheat Engine ou DBVM via le chemin de callback d'impression de débogage du noyau.
La seule logique réelle enregistre un gestionnaire DbgSetDebugPrintCallback, copie les chaînes de débogage entrantes, recherche le littéral dbvm-mode, puis laisse une voie de rapport TODO.
Comme le README le présente comme un simple meme et que le code est minimal, il faut le traiter comme une expérience autour d'un signal noyau très étroit, et non comme un cadre de détection complet.
Il sert surtout aux chercheurs anti-cheat qui explorent si la sortie de débogage noyau peut exposer l'état de Cheat Engine ou DBVM en environnement de test.
