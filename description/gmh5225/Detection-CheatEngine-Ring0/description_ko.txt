이 프로젝트는 커널 디버그 출력 콜백 경로를 통해 Cheat Engine 또는 DBVM 관련 활동을 감지하려는 작은 커널 PoC입니다.
실제 로직은 DbgSetDebugPrintCallback 핸들러를 등록하고, 들어오는 디버그 문자열을 복사한 뒤, 리터럴 dbvm-mode를 검사한 다음 TODO 보고 경로로 넘어가는 것이 전부입니다.
README에서 meme으로 표기되어 있고 코드도 최소 수준이므로, 완전한 탐지 프레임워크가 아니라 좁은 커널 측 신호 하나를 둘러싼 실험으로 보아야 합니다.
주로 테스트 환경에서 커널 디버그 출력이 Cheat Engine 또는 DBVM 상태를 드러낼 수 있는지 탐색하는 안티치트 연구자에게 유용합니다.
