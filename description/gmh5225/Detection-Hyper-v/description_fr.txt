Ce projet est un pilote minimal en mode noyau de détection Hyper-V qui lit l'état de l'hyperviseur directement depuis les structures KPCR ou KPRCB.
L'implémentation cible les en-têtes de Windows 10 build 17763, appelle KeGetPcr, parcourt jusqu'à CurrentPrcb, et inspecte PowerState.Hypervisor ainsi que PowerState.HvTargetState pour déterminer si la machine s'exécute en tant qu'invité Hyper-V.
Le pilote rapporte ensuite le résultat via des impressions de débogage et se termine avec STATUS_VIRUS_INFECTED, ce qui en fait davantage une expérience noyau ciblée qu'un module anti-cheat réutilisable.
Il est surtout utile aux chercheurs défensifs qui étudient les vérifications de structures noyau spécifiques à une build pour détecter la présence d'Hyper-V, sans s'appuyer uniquement sur des sondes CPUID en mode utilisateur.
