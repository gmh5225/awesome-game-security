Este proyecto es un controlador mínimo en modo kernel para detectar Hyper-V que lee el estado del hipervisor directamente desde las estructuras KPCR o KPRCB.
La implementación apunta a los encabezados de Windows 10 build 17763, llama a KeGetPcr, recorre hasta CurrentPrcb e inspecciona PowerState.Hypervisor y PowerState.HvTargetState para decidir si la máquina se ejecuta como invitado de Hyper-V.
A continuación, el controlador informa el resultado con impresiones de depuración y finaliza con STATUS_VIRUS_INFECTED, lo que lo acerca más a un experimento de kernel enfocado que a un módulo anti-cheat reutilizable.
Resulta especialmente útil para investigadores defensivos que estudian comprobaciones de estructuras del kernel específicas de una build para detectar la presencia de Hyper-V, sin depender únicamente de sondas CPUID en modo usuario.
