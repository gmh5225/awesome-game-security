Win32kHooker est un pilote noyau Windows qui montre comment localiser et accrocher les chemins de répartition de fonctions dans win32k.sys sur les systèmes modernes. Le code utilise le C++ avec les API noyau, l’attachement au contexte de processus, le mapping des syscalls et le désassemblage à l’exécution pour résoudre les cibles de hook dans l’espace de session. Il tient compte des changements d’architecture où les pointeurs clés sont stockés dans des structures d’état de session opaques plutôt que dans de simples sections de données globales. Son cas d’usage principal est la recherche avancée sur le sous-système graphique du noyau pour l’anti-triche, l’ingénierie défensive et les expériences de rétro-ingénierie.
