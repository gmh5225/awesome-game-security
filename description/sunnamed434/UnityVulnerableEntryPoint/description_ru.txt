Этот проект демонстрирует уязвимые точки входа в играх на Unity, которые можно использовать для внедрения кода и модификации игры. Он выявляет слабые места в процессе инициализации runtime Unity, загрузке Assembly-CSharp и жизненном цикле MonoBehaviour, где злоумышленники могут внедрять свой код или изменять поведение игры. Примеры на C# показывают практические сценарии эксплуатации. Материал рассчитан на разработчиков игр на Unity, изучающих уязвимости безопасности своих проектов, и исследователей безопасности, анализирующих характерные для Unity поверхности атаки.
