Dieses Projekt zeigt anfällige Einstiegspunkte in Unity-Spielen, die für Code-Injection und Spielmodifikation ausgenutzt werden können. Es identifiziert Schwachstellen im Initialisierungsprozess der Unity-Runtime, beim Laden von Assembly-CSharp und im MonoBehaviour-Lebenszyklus, an denen Angreifer benutzerdefinierten Code einschleusen oder das Spielverhalten verändern können. Die C#-Beispiele demonstrieren praxisnahe Exploitation-Szenarien. Es richtet sich an Unity-Spieleentwickler, die Sicherheitslücken in ihren Spielen kennenlernen möchten, sowie an Sicherheitsforscher, die Unity-spezifische Angriffsflächen untersuchen.
