Ce projet présente des points d'entrée vulnérables dans les jeux Unity pouvant être exploités pour l'injection de code et la modification du jeu. Il identifie les faiblesses du processus d'initialisation du runtime Unity, du chargement d'Assembly-CSharp et du cycle de vie MonoBehaviour, où des attaquants peuvent injecter du code personnalisé ou altérer le comportement du jeu. Les exemples C# illustrent des scénarios d'exploitation concrets. Il s'adresse aux développeurs Unity qui souhaitent comprendre les vulnérabilités de sécurité de leurs jeux, ainsi qu'aux chercheurs en sécurité étudiant les surfaces d'attaque spécifiques à Unity.
