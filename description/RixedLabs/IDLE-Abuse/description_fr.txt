Ce projet est une preuve de concept pour injecter du code lorsqu’un processus Windows devient inactif. Il s’appuie sur le comportement non documenté de RegisterWaitForInputIdle pour déclencher un callback après qu’un processus lancé a atteint un état d’inactivité. Les exemples C++ illustrent des schémas d’exécution de charge utile tels que la livraison de shellcode et des flux de manipulation de processus. Il est surtout utile pour l’expérimentation en sécurité offensive et pour étudier les détections d’abus du cycle de vie des processus.
