Este proyecto es una prueba de concepto para inyectar código cuando un proceso de Windows pasa a estado inactivo. Aprovecha el comportamiento no documentado de RegisterWaitForInputIdle para disparar una devolución de llamada tras alcanzar el estado idle un proceso generado. Los ejemplos en C++ muestran patrones de ejecución de payloads, como la entrega de shellcode y flujos de manipulación de procesos. Resulta útil principalmente para la experimentación en seguridad ofensiva y para estudiar detecciones de abuso del ciclo de vida de procesos.
