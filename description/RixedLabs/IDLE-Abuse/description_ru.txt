Этот проект — proof of concept для внедрения кода, когда процесс Windows переходит в состояние idle. Он использует недокументированное поведение RegisterWaitForInputIdle, чтобы вызвать callback после того, как созданный процесс достигает idle-состояния. Примеры на C++ демонстрируют схемы выполнения полезной нагрузки, такие как доставка shellcode и сценарии манипуляции процессами. Проект в основном полезен для экспериментов в offensive security и для изучения детектирования злоупотреблений жизненным циклом процессов.
