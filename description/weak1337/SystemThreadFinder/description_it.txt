Uno strumento che rileva i thread di sistema nascosti e mappati manualmente enumerando tutti i thread tramite NtQuerySystemInformation, confrontando l'indirizzo di avvio di ogni thread del kernel con gli intervalli dei moduli driver caricati e segnalando i thread i cui indirizzi di avvio ricadono fuori da qualsiasi regione di immagine di un driver legittimo.
Creato tramite reverse engineering della logica di rilevamento dei thread di BattlEye, usa API NT non documentate per interrogare le informazioni sui thread e confronta gli indirizzi di avvio con l'elenco dei moduli di sistema per identificare i thread generati da codice kernel mappato manualmente o iniettato.
È soprattutto utile ai ricercatori anti-cheat che studiano le tecniche di rilevamento dei thread a livello kernel e individuano cheat furtivi basati su driver che creano thread di sistema da regioni di memoria non mappate.
