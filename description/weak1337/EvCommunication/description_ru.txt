Фреймворк связи ядра с пользовательским режимом, использующий именованные события Windows (ZwOpenEvent/ZwSetEvent/ZwWaitForSingleObject) для двунаправленной сигнализации между драйвером ядра и клиентом в user-mode; драйвер перехватывает NtTokenManager, чтобы поймать kernel callback как начальную точку запуска.
Драйвер организует общий цикл событий: клиент в user-mode сигнализирует запросы через именованный объект события, обработчик в ядре выполняет чтение/запись памяти через MmCopyVirtualMemory и сигнализирует о завершении через второе событие, избегая традиционной IOCTL-связи, которую отслеживают античиты.
В основном полезен исследователям игровой безопасности, изучающим скрытные каналы связи драйверов, обходящие мониторинг IOCTL античит-системами вроде BattlEye и EAC.
