Windows 명명된 이벤트(ZwOpenEvent/ZwSetEvent/ZwWaitForSingleObject)를 사용해 커널 드라이버와 사용자 모드 클라이언트 간 양방향 시그널링을 수행하는 커널-사용자모드 통신 프레임워크입니다. 드라이버는 NtTokenManager를 후킹하여 커널 콜백을 초기 트리거 지점으로 가로챕니다.
드라이버는 공유 이벤트 루프를 구성합니다. 사용자 모드 클라이언트가 명명된 이벤트 객체로 요청을 알리면, 커널 핸들러가 MmCopyVirtualMemory로 메모리 읽기/쓰기를 처리한 뒤 두 번째 이벤트로 완료를 알리며, 안티치트가 감시하는 전통적인 IOCTL 기반 통신을 피합니다.
BattlEye, EAC 같은 안티치트의 IOCTL 모니터링을 우회하는 은밀한 드라이버 통신 채널을 연구하는 게임 보안 연구자에게 주로 유용합니다.
