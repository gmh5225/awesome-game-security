Un framework de comunicación de kernel a modo de usuario que usa eventos con nombre de Windows (ZwOpenEvent/ZwSetEvent/ZwWaitForSingleObject) para señalización bidireccional entre un controlador de kernel y un cliente en modo de usuario, con el controlador enganchando NtTokenManager para interceptar un callback de kernel como punto de activación inicial.
El controlador establece un bucle de eventos compartido en el que el cliente en modo de usuario señala solicitudes mediante un objeto de evento con nombre, el manejador del kernel procesa operaciones de lectura/escritura de memoria con MmCopyVirtualMemory y señala la finalización a través de un segundo evento, evitando la comunicación tradicional basada en IOCTL que los anti-cheats monitorean.
Resulta especialmente útil para investigadores de seguridad en juegos que estudian canales de comunicación sigilosos de controladores que evaden la supervisión de IOCTL de sistemas anti-cheat como BattlEye y EAC.
