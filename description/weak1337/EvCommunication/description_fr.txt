Un framework de communication noyau-vers-mode utilisateur qui utilise les événements nommés Windows (ZwOpenEvent/ZwSetEvent/ZwWaitForSingleObject) pour une signalisation bidirectionnelle entre un pilote noyau et un client en mode utilisateur, le pilote accrochant NtTokenManager afin d’intercepter un callback noyau comme point de déclenchement initial.
Le pilote établit une boucle d’événements partagée où le client en mode utilisateur signale les requêtes via un objet événement nommé, le gestionnaire noyau traite les opérations de lecture/écriture mémoire via MmCopyVirtualMemory, puis signale l’achèvement par un second événement, évitant ainsi la communication traditionnelle basée sur les IOCTL que les anti-cheats surveillent.
Il est surtout utile aux chercheurs en sécurité des jeux qui étudient des canaux de communication furtifs de pilotes contournant la surveillance des IOCTL par des systèmes anti-cheat comme BattlEye et EAC.
