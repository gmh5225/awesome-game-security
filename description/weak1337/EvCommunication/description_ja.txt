Windows の名前付きイベント（ZwOpenEvent/ZwSetEvent/ZwWaitForSingleObject）を用いて、カーネルドライバーとユーザーモードクライアント間で双方向シグナリングを行うカーネル対ユーザーモード通信フレームワークです。ドライバーは NtTokenManager をフックし、初期トリガーとしてカーネルコールバックを傍受します。
ドライバーは共有イベントループを確立し、ユーザーモードクライアントが名前付きイベントオブジェクト経由で要求を通知、カーネル側ハンドラが MmCopyVirtualMemory でメモリの読み書きを処理し、別のイベントで完了を通知します。これにより、アンチチートが監視する従来の IOCTL ベース通信を回避します。
BattlEye や EAC のようなアンチチートによる IOCTL 監視を回避するステルスなドライバー通信チャネルを研究するゲームセキュリティ研究者向けです。
