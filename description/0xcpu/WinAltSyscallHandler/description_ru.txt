Этот проект — исследовательский proof of concept для ядра Windows, посвящённый альтернативным обработчикам системных вызовов. В нём разбираются ограничения регистрации обработчиков, условия диспетчеризации и влияние отладочных флагов потока на то, какой путь callback будет выполнен. Реализация включает код на C и эксперименты с использованием trap frame, а также вызовами сведений о процессе, необходимыми для включения мониторинга. Материал рассчитан на исследователей низкоуровневой безопасности, изучающих перехват syscall и связанные ограничения стабильности.
