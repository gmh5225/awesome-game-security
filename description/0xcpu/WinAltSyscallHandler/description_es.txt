Este proyecto es una prueba de concepto de investigación del kernel de Windows que explora los mecanismos de manejadores alternativos de llamadas al sistema. Explica los límites de registro de manejadores, las condiciones de despacho y cómo los indicadores de depuración de hilos influyen en qué ruta de callback se ejecuta. La implementación incluye código en C y experimentos sobre el uso del trap frame y las llamadas de información de proceso necesarias para habilitar la supervisión. Está dirigido a investigadores de seguridad de bajo nivel que estudian el comportamiento de interceptación de syscalls y las restricciones de estabilidad relacionadas.
