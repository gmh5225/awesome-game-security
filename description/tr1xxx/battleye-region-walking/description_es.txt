Este proyecto demuestra la técnica de recorrido de regiones de memoria (memory region walking) que BattlEye usa para detectar código inyectado en el espacio de direcciones virtual de un proceso.
Implementa una enumeración de regiones basada en VirtualQuery con la lógica de filtrado de BattlEye, que identifica regiones de memoria ejecutables sospechosas según flags de protección, tamaños de región, tipos de memoria y patrones de direcciones.
El código en C++ clasifica las regiones como válidas o inválidas según comprobaciones de tipo MEM_PRIVATE/MEM_MAPPED y heurísticas específicas de tamaño y dirección que BattlEye usa para marcar shellcode y módulos mapeados manualmente.
Resulta especialmente útil para investigadores de anti-trampas e ingenieros de ingeniería inversa que estudian la metodología de detección por escaneo de memoria de BattlEye.
