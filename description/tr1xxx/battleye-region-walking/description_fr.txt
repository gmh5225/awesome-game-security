Ce projet présente la technique de parcours des régions mémoire (memory region walking) utilisée par BattlEye pour détecter du code injecté dans l'espace d'adressage virtuel d'un processus.
Il implémente une énumération des régions basée sur VirtualQuery avec la logique de filtrage de BattlEye, qui identifie les régions mémoire exécutables suspectes selon les indicateurs de protection, les tailles de région, les types de mémoire et les motifs d'adresses.
Le code C++ classifie les régions comme valides ou invalides d'après les contrôles de type MEM_PRIVATE/MEM_MAPPED et les heuristiques de taille et d'adresse spécifiques que BattlEye utilise pour signaler le shellcode et les modules mappés manuellement.
Il est principalement utile aux chercheurs anti-triche et aux ingénieurs en rétro-ingénierie qui étudient la méthodologie de détection par scan mémoire de BattlEye.
