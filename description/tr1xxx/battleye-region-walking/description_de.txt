Dieses Projekt demonstriert die von BattlEye genutzte Technik des Memory Region Walking zur Erkennung von injiziertem Code im virtuellen Adressraum eines Prozesses.
Es implementiert eine VirtualQuery-basierte Regionsaufzählung mit BattlEyes Filterlogik, die verdächtige ausführbare Speicherregionen anhand von Schutzflags, Regionsgrößen, Speichertypen und Adressmustern identifiziert.
Die C++-Codebasis klassifiziert Regionen als gültig oder ungültig anhand von MEM_PRIVATE/MEM_MAPPED-Typprüfungen sowie spezifischer Größen- und Adressheuristiken, mit denen BattlEye Shellcode und manuell gemappte Module kennzeichnet.
Es eignet sich vor allem für Anti-Cheat-Forscher und Reverse Engineers, die BattlEyes Methodik zur Speicher-Scan-Erkennung untersuchen.
