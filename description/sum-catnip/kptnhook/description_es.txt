Este proyecto se centra en un controlador de Windows en modo kernel para inyectar una DLL en cada proceso y realizar function hooking a nivel de sistema.
Se usa un controlador de kernel para garantizar el acceso a todos los procesos, incluidos los primeros que Windows inicia incluso antes de la pantalla de inicio de sesión.
Resulta especialmente útil para investigadores de seguridad en juegos e ingenieros inversos que estudian técnicas ofensivas en el ámbito cheat / injection en Windows.
