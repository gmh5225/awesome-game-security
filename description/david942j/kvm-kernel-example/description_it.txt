Un'implementazione minimale di hypervisor e kernel guest basati su KVM che dimostra come usare le API Linux KVM per creare una macchina virtuale con hypercall personalizzati, gestione della memoria, gestione delle syscall e caricamento ELF.
Il componente hypervisor agisce come un VMM simile a QEMU, mentre il kernel implementa sottosistemi di base tra cui mmap, esecuzione dei processi e I/O su file tramite l'inoltro degli hypercall.
È soprattutto utile per ricercatori di sicurezza e sviluppatori kernel che apprendono gli interni di KVM e la virtualizzazione basata su hypervisor da zero.
