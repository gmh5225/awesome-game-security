Una implementación mínima de hipervisor y kernel invitado basados en KVM que demuestra cómo usar las API de Linux KVM para crear una máquina virtual con hypercalls personalizados, gestión de memoria, manejo de syscalls y carga ELF.
El componente hipervisor actúa como un VMM similar a QEMU, mientras que el kernel implementa subsistemas básicos que incluyen mmap, ejecución de procesos y E/S de archivos mediante el reenvío de hypercalls.
Resulta especialmente útil para investigadores de seguridad y desarrolladores de kernel que aprenden los internos de KVM y la virtualización basada en hipervisor desde cero.
