Questo progetto è una proof of concept per il kernel Windows sulla comunicazione tramite hook di puntatori .data invece dei percorsi IOCTL standard. Mostra target di hook raggiungibili tramite syscall, inclusi flussi legati a NtConvertBetweenAuxiliaryCounterAndPerformanceCounter e all’interrogazione dell’integrità del codice. Il repository include client in user mode e progetti di driver kernel in C e C++, oltre a note di ricerca su come individuare siti di chiamata indiretta tramite artefatti Control Flow Guard. È pensato per ricercatori di sicurezza del kernel che studiano una comunicazione più furtiva dei driver mappati e i relativi compromessi di rilevamento.
