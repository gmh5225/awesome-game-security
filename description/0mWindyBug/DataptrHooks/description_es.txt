Este proyecto es una prueba de concepto del kernel de Windows para comunicación mediante hooks de punteros .data en lugar de rutas IOCTL estándar. Demuestra objetivos de hook alcanzables por syscall, incluidos flujos relacionados con NtConvertBetweenAuxiliaryCounterAndPerformanceCounter y la consulta de integridad de código. El repositorio incluye clientes en modo usuario y proyectos de controlador de kernel en C y C++, además de notas de investigación sobre cómo localizar sitios de llamadas indirectas mediante artefactos de Control Flow Guard. Está pensado para investigadores de seguridad del kernel que estudian una comunicación más encubierta de controladores mapeados y los compromisos de detección.
