Ce dépôt est une preuve de concept noyau Windows pour accrocher les notifications de processus via le mécanisme de table d'extension BAM. Au lieu de s'appuyer sur le tableau standard de callbacks de notification de processus, il cible le chemin de l'hôte d'extension et remplace le pointeur de callback BAM par une routine personnalisée. Le code du pilote illustre la recherche basée sur les offsets de ntoskrnl, le remplacement du pointeur de callback et la gestion temporaire du masque de notification. Il est destiné à la recherche avancée anti-cheat et EDR sur les contournements de callbacks, la visibilité défensive et les internals noyau non documentés.
