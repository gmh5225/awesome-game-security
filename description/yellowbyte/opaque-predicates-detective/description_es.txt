Este proyecto se centra en la introducción: los enfoques anteriores para identificar de forma genérica predicados opacos consisten en determinar si una rama condicional contiene una expresión invariante. El daño queda localizado a nivel de bloque básico (o a nivel de función), independientemente de cómo se construya la expresión invariante de un predicado opaco. Resulta principalmente útil para investigadores de seguridad en juegos e ingenieros de ingeniería inversa que estudian técnicas ofensivas en el ámbito de cheats / plugins de Binary Ninja.
