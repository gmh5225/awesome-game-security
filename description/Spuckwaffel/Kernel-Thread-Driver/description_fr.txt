Ce projet est un pilote de threads noyau Windows associé à un contrôleur en mode utilisateur pour les opérations mémoire.
Il utilise un modèle de communication par codes d'état entre le noyau et le mode utilisateur pour initialiser, suivre l'état de la connexion et traiter les commandes.
L'implémentation se concentre sur des tâches pratiques telles que la configuration du processus cible, la lecture mémoire et la récupération de la base des modules.
Il est destiné à la recherche sur le contournement d'anti-cheat et aux expériences d'architecture noyau-utilisateur dans les jeux protégés.
