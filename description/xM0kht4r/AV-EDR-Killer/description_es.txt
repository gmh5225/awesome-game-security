Este proyecto se centra en wsftprm.sys.
La vulnerabilidad se activa mediante el código IOCTL 0x22201C con un búfer de 1036 bytes en el que los primeros 4 bytes contienen el Process ID de destino como DWORD.
Resulta especialmente útil para investigadores de seguridad en juegos e ingenieros inversos que estudian técnicas ofensivas en el ámbito de cheats / controladores vulnerables.
