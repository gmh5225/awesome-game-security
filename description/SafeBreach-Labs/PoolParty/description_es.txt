Este proyecto es una colección de técnicas de inyección de procesos en Windows que abusan de los internos del thread pool para ejecutar código en procesos remotos.
Está escrito en C++ e implementa múltiples variantes, incluida la sobrescritura de la rutina de inicio del worker factory y la inserción de elementos TP_WORK, TP_WAIT, TP_IO, TP_ALPC, TP_JOB, TP_DIRECT y TP_TIMER.
El código incluye wrappers de API nativas, ayudantes de secuestro de handles e implementaciones estructuradas para cada primitiva del thread pool.
Está diseñado para investigación de red team y para probar la visibilidad de anti-cheat o EDR frente a técnicas de inyección de baja detección.
