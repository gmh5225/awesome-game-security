Dieses Projekt ist eine Sammlung von Windows-Prozessinjektionstechniken, die Thread-Pool-Interna missbrauchen, um Code in Remote-Prozessen auszuführen.
Es ist in C++ geschrieben und implementiert mehrere Varianten, darunter das Überschreiben der Startroutine der Worker Factory sowie das Einfügen von TP_WORK-, TP_WAIT-, TP_IO-, TP_ALPC-, TP_JOB-, TP_DIRECT- und TP_TIMER-Elementen.
Die Codebasis enthält Wrapper für native APIs, Hilfen zum Handle-Hijacking sowie strukturierte Implementierungen für jede Thread-Pool-Primitive.
Es ist für Red-Team-Forschung und zum Testen der Sichtbarkeit von Anti-Cheat- oder EDR-Systemen gegenüber schwer erkennbarer Injection-Tradecraft konzipiert.
