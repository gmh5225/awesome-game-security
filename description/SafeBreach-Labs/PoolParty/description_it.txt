Questo progetto è una raccolta di tecniche di injection di processi Windows che abusano degli interni del thread pool per eseguire codice in processi remoti.
È scritto in C++ e implementa diverse varianti, tra cui la sovrascrittura della start-routine del worker factory e l’inserimento di elementi TP_WORK, TP_WAIT, TP_IO, TP_ALPC, TP_JOB, TP_DIRECT e TP_TIMER.
Il codice include wrapper di API native, helper per l’hijacking degli handle e implementazioni strutturate per ogni primitiva del thread pool.
È pensato per la ricerca red team e per testare la visibilità di anti-cheat o EDR rispetto a tecniche di injection a bassa rilevabilità.
