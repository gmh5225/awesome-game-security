一个跨平台的 C/C++ 内存访问库，通过统一的读/写/模块枚举 API 对接多种后端：Windows 内核模式（经由 EPROCESS 遍历 ActiveProcessLinks）、用户模式（ReadProcessMemory/WriteProcessMemory）、Linux（/proc/pid/mem）、PCILeech（通过 VMMDLL/LeechCore DMA）、KVM（QEMU 客户机内省）、Proton（Linux 托管的 Windows 游戏），以及基于 EFI 变量的内核通信。
各后端实现同一套 vm.h 接口，涵盖进程查找、基于 CR3 的页表翻译、PEB/LDR 模块遍历与特征码扫描，使作弊或分析代码可在内核驱动、DMA 硬件与虚拟机监控程序传输之间切换，而无需改动面向游戏的逻辑。
它主要面向游戏安全研究者与作弊开发者，用于研究跨平台远程进程内存访问模式，以及基于 DMA/KVM 的读写原语。
