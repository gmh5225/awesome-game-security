Une bibliothèque d'accès mémoire C/C++ multiplateforme offrant des API unifiées de lecture/écriture/énumération de modules via plusieurs backends : mode noyau Windows (parcours d'ActiveProcessLinks via EPROCESS), mode utilisateur (ReadProcessMemory/WriteProcessMemory), Linux (/proc/pid/mem), PCILeech (via VMMDLL/LeechCore DMA), KVM (introspection d'invité QEMU), Proton (jeux Windows hébergés sous Linux) et communication noyau basée sur des variables EFI.
Chaque backend implémente la même interface vm.h pour la recherche de processus, la traduction de tables de pages basée sur CR3, le parcours de modules PEB/LDR et le scan de motifs, permettant au code de cheat ou d'analyse de basculer entre pilote noyau, matériel DMA et transports hyperviseur sans modifier la logique côté jeu.
Elle est surtout utile aux chercheurs en sécurité des jeux et aux développeurs de cheats qui étudient les modèles d'accès mémoire à distance multiplateformes et les primitives de lecture-écriture basées sur DMA/KVM.
