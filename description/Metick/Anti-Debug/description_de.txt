Anti-Debug ist ein kleines C++-Proof-of-Concept, das Debugger-Erkennung über das Suspend-Count-Verhalten von ResumeThread zeigt.
Es demonstriert, wie ein durch Debugger-Anhängen kurzzeitig suspendierter Thread erkannt werden kann, indem der von WinAPI-Aufrufen zurückgegebene Zähler geprüft wird.
Das Beispiel ist minimal und konzentriert sich auf ein einzelnes Anti-Debug-Signal statt auf ein vollständiges Schutzframework.
Es richtet sich an Windows-Sicherheitslernende, die Anti-Tamper- und Anti-Analyse-Techniken ausprobieren.
