Este proyecto es una prueba de concepto de un mecanismo de comunicación de Windows del kernel al modo usuario basado en el intercambio de un puntero .data del kernel. Está escrito en C++ con componentes separados de controlador de kernel y modo usuario, e incluye pattern scanning junto con rutinas auxiliares para operaciones de memoria y búsqueda de la base del proceso. El enfoque prioriza la comunicación de alta velocidad tipo mensaje y lectura-escritura, pero conlleva riesgos de estabilidad en sistemas protegidos. Está pensado para investigación del kernel y experimentación relacionada con anti-cheat, no para despliegue en producción.
