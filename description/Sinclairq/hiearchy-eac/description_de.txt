Dieses Projekt ist ein Windows-Kernel-Proof-of-Concept zum Umgehen der Selbstintegritätsprüfungen von Anti-Cheat-Systemen durch Manipulation der Aufrufhierarchie und von Speicherlesevorgängen. Es ist in C++ mit etwas Assembler implementiert und hookt ausgewählte Verifizierungsroutinen, um die Prüfung auf eine bereinigte Image-Kopie umzuleiten. Der Code überwacht Modul-Ladeereignisse, verfolgt die Speichergrenzen des EasyAntiCheat-Treibers und spoofed Stack- sowie Registerreferenzen bei integritätsbezogenen Zugriffen. Ziel ist Reverse Engineering von Anti-Cheat sowie ein defensives Verständnis von Integritätsprüfungspfaden.
