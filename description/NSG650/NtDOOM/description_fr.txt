Ce projet est un portage noyau Windows qui exécute DOOM depuis le contexte d'un pilote noyau.
Il s'appuie sur la gestion des appels système côté win32k, l'usurpation de contexte de thread, ainsi que des interactions graphiques et d'entrée côté noyau pour faire tourner la logique de jeu.
L'implémentation consiste principalement en code de pilote C et C++, avec un travail substantiel sur les internals NT et une base PureDOOM adaptée.
Il sert de démonstration de recherche pour des expérimentations extrêmes sur l'interface graphique et les appels système en noyau, plutôt que de développement de jeu pratique.
