Ce projet est une preuve de concept en C pour une injection de code orientée furtivité, basée sur NtQueueApcThreadEx et des gadgets NTDLL aléatoires. Il recherche dans les sections exécutables de ntdll des séquences pop-and-return adaptées et les utilise pour que l’exécution APC aboutisse dans le shellcode. L’approche fait paraître les pointeurs de routines plus légitimes qu’une injection APC directe classique, tout en documentant les vecteurs de détection probables. Son usage principal est la recherche en tradecraft offensif et l’évaluation de la télémétrie anti-cheat ou EDR autour de l’exécution basée sur les APC.
