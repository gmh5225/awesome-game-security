wsb-detect ist eine C-Bibliothek und ein Beispielprogramm zur Erkennung, ob Code in der Windows Sandbox ausgeführt wird.
Es implementiert mehrere Fingerprinting-Prüfungen, etwa sandbox-spezifische Prozesse, Benutzernamen, Gerätepfade, DNS-Suffixe, Registry-Artefakte und Timing-Hinweise.
Das Projekt stellt modulare Erkennungsfunktionen bereit, die je nach Toleranz gegenüber Fehlalarmen kombiniert werden können.
Der Hauptanwendungsfall ist Anti-Analyse-Forschung und Umgebungserkennung für Malware-Studien, Red-Team-Simulation und Tests defensiver Gegenmaßnahmen.
