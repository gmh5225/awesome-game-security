Ce projet fournit des sources de gestionnaires KVM modifiés qui interceptent et altèrent le comportement de timing RDTSC sur les chemins Intel VMX et AMD SVM.
Il explique comment patcher le code de virtualisation du noyau, ajuster les deltas de timestamp factices et configurer les indicateurs CPU de QEMU, notamment en désactivant RDTSCP.
L’implémentation est écrite en C pour le noyau Linux et se concentre sur le contrôle bas niveau du timing de l’hyperviseur.
Son cas d’usage principal est la recherche en virtualisation et anti-détection, y compris des expériences autour des contrôles anti-triche basés sur le timing.
