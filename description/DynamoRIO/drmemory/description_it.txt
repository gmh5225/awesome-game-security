Dr. Memory è un debugger dinamico della memoria che rileva errori di memoria runtime comuni nei programmi nativi. Scritto principalmente in C e C++ sopra la piattaforma di strumentazione DynamoRIO, può trovare letture non inizializzate, accessi fuori dai limiti, use-after-free, doppie free e leak, con controlli specifici per Windows come l'uso improprio di handle e GDI. Supporta binari non modificati su Windows, Linux, macOS e Android per le piattaforme IA-32, AMD64 e ARM. Il progetto è ampiamente usato per l'hardening del software e la ricerca di bug di sicurezza, inclusi i test di affidabilità del codice di giochi e anti-cheat.
