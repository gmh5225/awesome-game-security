Este proyecto es una prueba de concepto para ejecutar scripts Lua dentro del kernel de Windows. Incorpora un intérprete Lua 5.4 en un controlador de kernel, exponiendo a los scripts Lua que se ejecutan en ring 0 APIs del kernel NT como acceso a memoria física y virtual, enumeración de procesos y lectura/escritura de MSR. El controlador en C/C++ se comunica con un cliente en modo usuario que envía código Lua para su evaluación en el lado del kernel. Está dirigido a investigadores de kernel y desarrolladores de exploits que exploran la introspección del kernel mediante scripts y las implicaciones de ejecutar lenguajes interpretados en contextos privilegiados.
