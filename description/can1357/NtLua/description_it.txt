Questo progetto è una proof of concept per l'esecuzione di script Lua all'interno del kernel Windows. Incorpora un interprete Lua 5.4 in un driver kernel, esponendo agli script Lua eseguiti in ring 0 API del kernel NT come l'accesso alla memoria fisica e virtuale, l'enumerazione dei processi e la lettura/scrittura degli MSR. Il driver C/C++ comunica con un client in modalità utente che invia codice Lua per la valutazione lato kernel. È rivolto a ricercatori del kernel e sviluppatori di exploit che esplorano l'introspezione del kernel tramite script e le implicazioni dell'esecuzione di linguaggi interpretati in contesti privilegiati.
