이 프로젝트는 사용자 모드에서 직접 및 간접 시스템 호출(syscall) 호출을 탐지하기 위한 Windows 개념 증명입니다. instrumentation callback 또는 스레드 스택 검사를 사용해 시스템 호출이 ntdll에서 발생했는지(정상) 아니면 사용자 정의 스텁에서 발생했는지(의심) 판별하여, 잠재적인 시스템 호출 후킹 우회를 표시합니다. C 구현은 안티치트 및 EDR 제품이 활용할 수 있는 탐지 휴리스틱을 보여 줍니다. 직접 시스템 호출 탐지와 사용자 모드 무결성 모니터링을 연구하는 방어 보안 연구자를 대상으로 합니다.
