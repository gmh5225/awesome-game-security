Questo progetto crea thread di sistema trasparenti su Windows, quasi invisibili all'introspezione di sistema, rimuovendo un modulo caricato dalle tabelle delle pagine di sistema.
Imposta uno spazio di indirizzamento personalizzato completamente controllato con una propria Interrupt Descriptor Table (IDT) e impedisce l'ispezione individuale dei thread tramite interrupt non mascherabili (NMI).
Il driver del kernel Windows supporta la creazione di thread sia dall'esterno sia dall'interno dello spazio di indirizzamento personalizzato, con esecuzione diretta delle funzioni senza macro né chiamate wrapper.
È particolarmente utile per i ricercatori di sicurezza del kernel che studiano l'esecuzione occultata di thread, la manipolazione delle tabelle delle pagine e le tecniche anti-forensi nei driver Windows.
