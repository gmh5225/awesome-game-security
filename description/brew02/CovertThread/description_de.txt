Dieses Projekt erzeugt unter Windows transparente Systemthreads, die für Systemintrospektion nahezu unsichtbar sind, indem ein geladenes Modul aus den System-Seitentabellen entfernt wird.
Es richtet einen vollständig kontrollierten eigenen Adressraum mit eigener Interrupt Descriptor Table (IDT) ein und verhindert die Inspektion einzelner Threads über Non-Maskable Interrupts (NMIs).
Der Windows-Kerneltreiber unterstützt das Erstellen von Threads sowohl außerhalb als auch innerhalb des eigenen Adressraums mit direkter Funktionsausführung ohne Makros oder Wrapper-Aufrufe.
Er ist vor allem für Kernel-Sicherheitsforscher nützlich, die verdeckte Thread-Ausführung, Seitentabellen-Manipulation und Anti-Forensik-Techniken in Windows-Treibern untersuchen.
