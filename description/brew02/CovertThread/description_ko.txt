이 프로젝트는 로드된 모듈을 시스템 페이지 테이블에서 제거하여 시스템 인트로스펙션에 거의 보이지 않는 투명한 시스템 스레드를 Windows에서 생성합니다.
자체 인터럽트 디스크립터 테이블(IDT)을 갖춘 완전히 제어되는 커스텀 주소 공간을 구성하고, 마스크 불가 인터럽트(NMI)를 통한 개별 스레드 검사를 방지합니다.
이 Windows 커널 드라이버는 커스텀 주소 공간 외부와 내부 모두에서 스레드 생성을 지원하며, 매크로나 래퍼 호출 없이 함수를 직접 실행할 수 있습니다.
주로 Windows 드라이버에서 은밀한 스레드 실행, 페이지 테이블 조작, 안티포렌식 기법을 연구하는 커널 보안 연구자에게 유용합니다.
