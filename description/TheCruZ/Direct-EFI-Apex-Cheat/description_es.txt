Este proyecto combina un cliente de trampas de juego en modo usuario con un componente de tiempo de ejecución UEFI para acceder a rutas de memoria del kernel desde un contexto EFI. El código en C y C++ implementa operaciones de memoria basadas en comandos, resolución de la base del proceso y funciones de jugabilidad como glow y asistencia de apuntado. La comunicación se construye en torno a hooks de variables en tiempo de ejecución y punteros a funciones de bajo nivel del kernel de Windows puenteados desde el lado EFI. Su caso de uso principal es la experimentación con trampas asistidas por firmware y evasión de anti-cheat.
