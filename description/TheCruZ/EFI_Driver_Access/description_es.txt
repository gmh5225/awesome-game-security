Este proyecto demuestra la carga de un controlador EFI durante el arranque para proporcionar acceso privilegiado a la memoria en sesiones de Windows. Incluye un módulo en tiempo de ejecución del lado EFI y un cliente en espacio de usuario que emite solicitudes de lectura, escritura y base de proceso. La implementación combina C/C++ con componentes de GNU-EFI y Visual Studio, y documenta los detalles del flujo de compilación y arranque. Está pensado principalmente para investigación de kernel y evasión de anti-cheat que involucra rutas de ejecución pre-OS.
