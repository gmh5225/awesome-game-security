이 프로젝트는 instrumentation callback 메커니즘(NtSetInformationProcess와 ProcessInstrumentationCallback)을 사용해 시스템 콜 반환을 가로채는 Windows 개념 증명입니다. 설정되면 콜백은 커널에서 사용자 모드로의의 전환마다 호출되어, ntdll 스텁을 패치하지 않고도 시스템 콜 결과를 검사하거나 수정할 수 있습니다. C/C++ 구현은 가로챈 호출을 기록하며, 이 기법이 모니터링이나 변조에 어떻게 쓰일 수 있는지 보여줍니다. 대안 후킹 방법과 안티치트 또는 EDR 회피 기법을 연구하는 보안 연구자를 대상으로 합니다.
