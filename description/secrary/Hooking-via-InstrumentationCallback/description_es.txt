Este proyecto es una prueba de concepto para Windows que intercepta los retornos de syscalls mediante el mecanismo de instrumentation callback (NtSetInformationProcess con ProcessInstrumentationCallback). Una vez configurado, el callback se dispara en cada transición de kernel a usuario, lo que permite inspeccionar o modificar los resultados de las syscalls sin parchear los stubs de ntdll. La implementación en C/C++ registra las llamadas interceptadas y muestra cómo esta técnica puede usarse para monitorización o manipulación. Está dirigida a investigadores de seguridad que estudian métodos de hooking alternativos y técnicas de evasión de anti-cheat o EDR.
