Ce projet est un framework de fuzzing pour le serveur Apache HTTPD qui remplace la couche socket d'Apache par des filtres d'E/S personnalisés, injectant des octets bruts directement dans les mêmes chemins de code que le trafic HTTP réel. Le framework basé sur Python inclut des reproduceurs pour des CVE connues et une documentation détaillée sur les internals d'Apache, tels que les memory pools, les hooks, les filtres, les buckets et le pipeline de requêtes. Il s'adresse principalement aux chercheurs en vulnérabilités et aux ingénieurs sécurité qui étudient les techniques de fuzzing côté serveur et les surfaces d'attaque du parsing HTTP.
