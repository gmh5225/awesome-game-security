Dieses Projekt ist ein Proof-of-Concept-Android-Detektor zur Erkennung von Prozessinjektionsverhalten im Zygisk-Stil. Die Erkennungslogik ist in C++ und JNI umgesetzt: Es wird ein Kindprozess per fork erzeugt, per ptrace angehängt und PTRACE_GETEVENTMSG-Artefakte werden ausgelesen. Das Repository enthält eine Beispiel-APK sowie Laufzeitprotokolle, die zeigen, wie mehrere Zygisk-Forks erkannt werden können, während Baseline-Implementierungen abweichen können. Es eignet sich vor allem für Forschung zu Mobile Anti-Cheat und Root-Erkennung mit Fokus auf Laufzeit-Integritätsprüfungen.
