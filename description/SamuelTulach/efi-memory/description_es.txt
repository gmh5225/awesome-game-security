efi-memory es un controlador EFI runtime de tipo proof-of-concept para leer y escribir memoria virtual desde el contexto de firmware.
Utiliza un método de comunicación mediante hook de SetVariable inspirado en EfiGuard e incluye componentes acompañantes tanto del lado del firmware como en modo usuario.
El repositorio también contiene un cliente mapper derivado de un flujo de trabajo al estilo de kdmapper para escenarios de manual mapping en Windows.
Está orientado principalmente a la investigación de seguridad de firmware, experimentos de acceso a memoria de bajo nivel y estudios de seguridad en juegos que involucran interacciones pre-OS o con controladores runtime.
