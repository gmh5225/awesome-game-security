efi-memory ist ein Proof-of-Concept-EFI-Runtime-Treiber zum Lesen und Schreiben von virtuellem Speicher aus dem Firmware-Kontext.
Es nutzt eine von EfiGuard inspirierte SetVariable-Hook-Kommunikationsmethode und enthält sowohl firmwareseitige als auch User-Mode-Begleitkomponenten.
Das Repository enthält außerdem einen Mapper-Client, der aus einem kdmapper-ähnlichen Workflow für Manual-Mapping-Szenarien unter Windows abgeleitet ist.
Es richtet sich vor allem an Firmware-Sicherheitsforschung, Low-Level-Speicherzugriffsexperimente und Game-Security-Studien zu Pre-OS- oder Runtime-Treiber-Interaktionen.
