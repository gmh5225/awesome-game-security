efi-memory est un pilote EFI runtime de type proof-of-concept permettant de lire et d'écrire la mémoire virtuelle depuis le contexte firmware.
Il utilise une méthode de communication par hook SetVariable inspirée d'EfiGuard et comprend des composants compagnons côté firmware et en mode utilisateur.
Le dépôt contient également un client mapper dérivé d'un workflow de type kdmapper pour les scénarios de manual mapping sous Windows.
Il vise principalement la recherche en sécurité firmware, les expériences d'accès mémoire bas niveau, et les études de sécurité des jeux impliquant des interactions pre-OS ou avec des pilotes runtime.
