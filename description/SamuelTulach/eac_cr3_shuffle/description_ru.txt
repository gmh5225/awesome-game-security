Этот проект — компактный исследовательский пример, посвящённый поведению CR3 shuffling в средах, защищённых Easy Anti-Cheat.
Пример кода на C++ демонстрирует низкоуровневую логику пейджинга: обход диапазонов физической памяти, обнаружение directory base и проверки трансляции виртуальных адресов в физические.
К нему прилагаются справочные материалы, объясняющие, почему манипуляции с CR3 усложняют внешние сценарии инспекции памяти.
Основное применение — реверс-инжиниринг и исследование внутренней работы античитов, в особенности для понимания стратегий защиты, связанных с таблицами страниц.
