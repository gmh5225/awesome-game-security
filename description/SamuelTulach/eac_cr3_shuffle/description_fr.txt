Ce projet est un exemple de recherche compact centré sur le comportement de CR3 shuffling dans les environnements protégés par Easy Anti-Cheat.
Le code d'exemple en C++ illustre la logique de pagination bas niveau, comme le parcours des plages de mémoire physique, la découverte de la base de répertoire et les vérifications de traduction virtuel-vers-physique.
Il s'accompagne de documentation de référence qui explique pourquoi la manipulation de CR3 complique les flux d'inspection mémoire externes.
Son cas d'usage principal est la rétro-ingénierie et la recherche sur les mécanismes internes des anti-cheats, notamment pour comprendre les stratégies de protection liées aux tables de pages.
