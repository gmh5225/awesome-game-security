Этот проект — небольшой proof-of-concept инструмент anti-anti-debug, исследующий обход на основе состояний гонки (race condition). Он реализован на C++ для Visual Studio и использует нативные Windows NT API для проверки debug-портов, поведения скрытых потоков и связанных артефактов отладчика. Пример показывает, как тайминг и проверки состояния можно применять против распространённых механизмов сокрытия в сценариях отладки в userland. Основная аудитория — исследователи в области reverse engineering, экспериментирующие с техниками anti-debug и anti-anti-debug.
