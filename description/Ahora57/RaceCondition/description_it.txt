Questo progetto è un piccolo strumento anti-anti-debug di tipo proof-of-concept che esplora comportamenti di bypass basati su race condition. È implementato in C++ per Visual Studio e usa le API native di Windows NT per sondare debug port, comportamento di thread nascosti e artefatti correlati del debugger. L’esempio si concentra sul dimostrare come timing e controlli di stato possano essere usati contro i meccanismi di occultamento comuni negli scenari di debugging in userland. Il pubblico principale è costituito da ricercatori di reverse engineering che sperimentano tecniche anti-debug e anti-anti-debug.
