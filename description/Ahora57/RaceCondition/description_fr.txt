Ce projet est un petit outil anti-anti-debug de type proof-of-concept qui explore les contournements basés sur des conditions de course. Il est implémenté en C++ pour Visual Studio et s’appuie sur les API Windows NT natives pour sonder les ports de débogage, le comportement des threads masqués et les artefacts associés au débogueur. L’exemple vise à montrer comment le timing et les contrôles d’état peuvent être utilisés contre les mécanismes de masquage courants dans le débogage en mode utilisateur. Son public principal est composé de chercheurs en reverse engineering qui expérimentent les techniques anti-debug et anti-anti-debug.
