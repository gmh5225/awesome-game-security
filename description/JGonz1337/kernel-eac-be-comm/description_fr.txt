Ce projet implémente un framework de communication Windows entre le noyau et l'espace utilisateur, qui achemine les commandes via un pointeur de fonction win32k hooké. Le pilote et le client utilisateur sont écrits en C++ et exposent des opérations telles que la recherche de base de processus, la recherche de modules, la lecture et l'écriture mémoire, l'allocation, la libération et le changement de protection. Il utilise des structures de requête personnalisées, des chaînes obscurcies par XOR et un protocole de contrôle compact pour échanger des données entre le ring 3 et le ring 0. Le code est orienté vers le contournement d'anti-cheat et la recherche d'outils mémoire de jeux dans des environnements protégés.
